Gaps for Itemized Network & Disk Billing

What's missing today

  • Zero capture — no bytes, networkBytes, storageBytes on any usage model
  • No time-series usage table — all aggregation is monthly
  • Compute granularity = step level · no per-task line items
  • No artifact-upload metering hooks in nx-api

What's already in place to leverage

  • Plan modifier system supports new credit pools (Plan.kt:553)
  • MBillingRecord easy to extend with new credit fields
  • Stripe handler iterates credit types — auto-emits new line items
  • Aggregation pattern is repeatable per stream

Decisions needed before coding

  • Capture point — agent self-reports? server measures from artifact uploads? both?
  • Granularity — per-step (matches compute) or per-task (new dimension)?
  • Pricing model — flat $/GB · tiered · or included pool + overage like compute
  • Net vs gross — cache-hit downloads billable? probably yes, but PRO needs included pool or churn risk
  • Backfill — synthesize from existing artifact metadata? or forward-only after rollout?
