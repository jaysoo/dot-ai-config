# nx.dev GA Traffic Pipeline

Deterministic data-processing + visualization pipeline for nx.dev Google Analytics
traffic. Turns scraped raw GA4 JSON into merged chart data, a deterministic
analysis, and a self-contained chart app.

## Pipeline

1. **Scrape (manual, already done).** GA4 has no usable API here (needs auth, and
   Explore caps retention at ~14 months), so the series were pulled by hand via
   Chrome DevTools against the GA4 UI (standard Reports for Jan-Apr 2025, Explore
   free-form for May 2025 onward) and saved as `raw/*.json`. These are the source
   of truth and are never modified.
2. **Process.** Run `node process.mjs` from this directory. It reads `raw/*`,
   builds a merged daily table, computes the analysis, and writes `out/*`.
3. **View.** Open `out/chart.html` in a browser (works via `file://`; only the
   Chart.js CDN libs need network).

```bash
cd /Users/jack/projects/dot-ai-config/dot_ai/2026-06-19/tasks/ga-traffic
node process.mjs
open out/chart.html
```

`process.mjs` is a single Node ESM script with no external npm deps (only
`fs`, `path`, `url`). It is deterministic: same raw inputs -> identical outputs.

| `server-human-signal.json` | Documents the (failed) attempt to isolate a HUMAN server signal via the `Bot`+`AI Tool` dimensions: even `Bot=false AND AI-Tool=false` grows ~2-4x Feb->Apr, so GA cannot cleanly separate humans from undetected AI crawlers. Reference only (not charted). |
| `top-pages.json` | Top-25 pages per category over the full daily window (Views). Note Others is dominated by OLD root-path docs that pre-date the 2025-09-29 /docs move. |

| `segment-definitions.json` | The MECE page-type rules: home (`/`), docs (`/docs*`), blog (`/blog*`), marketing (Framer-page regex), others (= all - the rest). Documents coverage (blog/marketing daily from May 2025). |
| `events.json` | 10 config/platform/tracking events with dates and expected impact: GTM added, astro-docs /docs live (2025-09-29), server_page_view start (2026-02-02), Vercel->Netlify DNS cutover (2026-02-04), GTM-only cutover (#34384, 2026-02-10), Framer migration (2026-02-27), nx-blog standalone (2026-04-16), nx-dev pared down (2026-04-17), bot-probe 404s (2026-04-30), Cookiebot banner (2026-05-01). |
| `source-crosschecks.json` | Reports-vs-Explore agreement checks (May and Aug 2025 match exactly). Loaded by `process.mjs`, which recomputes them deterministically (Reports == Explore == monthly-segments) and emits `analysis.source_crosscheck`. |

`monthly-segments.json` is generated by `raw/_gen-monthly.mjs` from the verified
scrape array in `../ga-monthly-traffic.mjs` (run `node raw/_gen-monthly.mjs` to
reproduce it byte-for-byte). The daily `*.json` files are hand-scraped raw sources
of truth and have no generator.

`active_users` (daily and monthly) and `daily-all.json`'s `event_count_all` are
retained in the raw inputs but intentionally NOT analyzed or charted: AU is
contaminated by server-side MP events from Feb 2026 (not comparable across that
boundary), and event_count_all mixes client and server events. They are kept for
completeness only.

## Outputs (`out/`)

| File | Contents |
| --- | --- |
| `chart-data.json` | `{ meta, events, daily[], weekly[], weeklyNxdev[], weeklyServerCat[], monthly[], monthlyNxdev[], topPages, topPagesBeforeAfter }`. Each client row carries the MECE page-type segments (home/docs/blog/marketing/others views; others = all - the rest) + server_page_view. `*Nxdev` = the `Hostname=nx.dev` variant; `weeklyServerCat` = server_page_view by category with `*_bf` (bot=false) companions. |
| `analysis.json` | Deterministic computations: `mom` (month-over-month % for all/home/docs), `trend` (last 12 full months, OLS slope + total %, plus a `pre_banner` parallel trend through 2026-04 since the 12-month endpoint is consent-suppressed), `event_impacts` (14d-before/14d-after window on daily all_views per event; the after-window is inclusive of the event day, surfaced in `after_window`, and a `reason` is added when the before-window is truncated by the daily span start), `cookiebot_2026_05_01`, `server_page_view` (first/peak/monthly totals), `daily_vs_monthly_check` (per-series integrity for all/home/docs/server: monthly vs sum-of-daily, tolerance flag), `source_crosscheck` (recomputed Reports == Explore == monthly-segments agreement), `key_findings`. |
| `chart.html` | Self-contained chart app. Data is injected inline by `process.mjs` (not fetched), so it opens via `file://`. Top-of-page **"nx.dev hostname only" toggle** switches the client charts to the canonical-host series. THREE time-axis STACKED-AREA charts: (1) WEEKLY client Views by page type (full ISO weeks), (2) MONTHLY client Views back to Jan 2025 - both with all 10 milestones as annotated vertical lines + `server_page_view` as an orange right-axis line; (3) WEEKLY `server_page_view` by page type with an **include/exclude-bots toggle**. The white "Total (all types)" line recomputes to the sum of VISIBLE categories when you toggle legend items off. Below the charts: key findings, top-25 pages per category, and a **before/after the 2025-09-29 /docs switch** comparison. `CATEGORY-TRENDS-REPORT.md` (sibling file) is the written per-category MoM + steepest-decline + internal/external correlation analysis. |

## Key caveats

- **Views = client page_view** (GA4 `screenPageViews`). That is the only metric
  called "Views" here.
- **server_page_view is a separate event** (Measurement Protocol, server-side),
  NOT counted in Views, and is mostly AI-crawler traffic. It sits on its own
  right-hand axis.
- **No /2 normalization.** The double-count theory is disproven: pre-GTM
  (Jan-Apr 2025, ~650K/mo) matches post-GTM (May-Jul 2025, ~635K). Raw Views are
  directly comparable across the whole range.
- **The big May 2026 drop is the Cookiebot consent banner** (2026-05-01), a
  measurement effect (consent-gated client GA), not lost traffic - server_page_view
  stays flat/growing across it. There is also a real, separate pre-banner decline
  in Mar-Apr 2026. Because of this, the 12-month `trend` endpoint (2026-05) is
  consent-suppressed: read `trend.pre_banner` (through 2026-04) for the organic
  signal (~-52% all_views, ~-33% home), and treat the headline 12-month % as
  inflated by the measurement cliff. The home trend in particular is flat (~54K/mo)
  through Apr 2026 then a single May cliff, not a gradual decline.
- **Active users (AU) is contaminated** by server-side MP events from Feb 2026, so
  AU before vs after Feb 2026 is not comparable. The analysis does not trend AU.
- **The docs / non-docs split is only meaningful from ~Oct 2025**, when astro-docs
  moved to `/docs`. `process.mjs` zeroes pre-launch `/docs` Views (before 2025-09-29)
  into Others - those were 61 stray bot/scanner hits on URLs that did not exist yet.
- **Hostname matters.** The GA property tracks `*.nx.dev` versioned subdomains, redirect
  domains, `localhost`, and preview deploys alongside `nx.dev`. The non-nx.dev share is
  time-varying (2.5%-22%/mo, peak Jan 2026), so the all-hosts view overstates some of the
  late-2025/early-2026 numbers. The "nx.dev hostname only" toggle strips them.
- **The Bot/AI-Tool dimensions cannot isolate humans.** server_page_view is ~30% flagged-bot
  and ~7% flagged-AI-Tool, but even `Bot=false AND AI-Tool=false` grows ~2-4x Feb->Apr -
  undetected AI crawlers evade both flags. So server-side GA cannot prove the client decline
  is real vs measurement (see `server-human-signal.json`).
- **Decline read (calibrated).** The Mar-Apr 2026 pre-banner decline is content-agnostic:
  home/docs/blog all fell ~-33 to -43% (Oct 2025->Apr 2026) while marketing GREW +39 to +52%.
  Three unrelated page types declining by the same % is the signature of a shared client-capture
  regression (the GTM-only cutover #34384 making page_view depend on the GTM container config),
  not content-specific loss. Likely measurement-led with a smaller real AI/organic residual.
  NOTE: not an ad-blocking effect - gtm.js and gtag/js are blocked equally. The decisive
  unresolved test is GA4 page_views vs Google Search Console CLICKS (consent-independent).
