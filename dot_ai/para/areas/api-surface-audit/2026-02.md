## Medium Findings — Stale, mismatched, or misleading

### M1. `TaskHasher` deprecated overloads say "removed in v20" — still present in v22.5

- **Source**: `packages/nx/src/hasher/task-hasher.ts` lines 46-70
- Four overloads of `hashTask` and `hashTasks` marked `@deprecated ... removed in v20`
- Two major versions overdue for removal

### M2. `initTasksRunner` exported from `nx` package is deprecated with no replacement guidance

- **Source**: `packages/nx/src/index.ts` exports `initTasksRunner`
- The function is marked `@deprecated This function is deprecated. Do not use this`
- Still the only public export of the `nx` package
- No guidance on what to use instead

### M3. `defaultTasksRunner`, `RemoteCache`, `DefaultTasksRunnerOptions` — Still exported via devkit

- **Source**: `packages/nx/src/devkit-exports.ts`
- Custom task runners were deprecated in Nx 21, but these symbols are still in the public devkit API
- `NxJsonConfiguration.tasksRunnerOptions` marked deprecated but still exported

### M4. `@nx/vite:vitest` generator deprecated in favor of `@nx/vitest:configuration`

- **Source**: `packages/vite/src/generators/vitest/vitest-generator.ts` line 7
- Says "removed in Nx 23" but `@nx/vitest` has no docs page at all (see H1)
- Users migrating from `@nx/vite:vitest` to `@nx/vitest:configuration` have no API reference

### M5. `@nx/vite:test` executor deprecated in favor of `@nx/vitest:test`

- **Source**: `packages/vite/src/executors/test/vitest.impl.ts` line 7
- Says "removed in Nx 23" but `@nx/vitest` is undocumented

### M6. `NxAffectedConfig` and `implicitDependencies` — Deprecated without doc badges

- Both are in `NxJsonConfiguration` type, exported via devkit
- `affected` config marked deprecated since before Nx 19, no deprecation badge in API reference
- `implicitDependencies` deprecated, links to deprecation page but API reference doesn't surface warning

### M7. Docs list stale generators for @nx/webpack and @nx/jest

**@nx/webpack**:

**@nx/jest**:

These are all the same pattern: generators were renamed from `*-project` to `configuration` but docs were never updated.
