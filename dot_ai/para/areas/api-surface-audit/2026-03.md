## High Priority

### H1. `@nx/js` utility API has zero generated documentation

**Status**: Unchanged from Feb audit.

`@nx/js` exports ~25 re-export paths from `packages/js/src/index.ts` covering utility functions/types but is **not included** in the docs generation pipeline (`astro-docs/src/plugins/nx-reference-packages.loader.ts`). The loader only generates docs for: `nx`, `plugin`, `web`, `workspace`, `devkit`, CNW, and CLI.

Key undocumented exports:

- `createLockFile`, `getLockFileName` (re-exported from `nx/src/plugins/js/lock-file/lock-file`)
- `createPackageJson` (re-exported from `nx/src/plugins/js/package-json/create-package-json`)
- `readTsConfig`, `readTsConfigPaths`, `getRootTsConfigPath`
- `addTsConfigPath`, `getRelativePathToRootTsConfig`
- `libraryGenerator`, `initGenerator`, `setupPrettierGenerator`, `setupVerdaccio`
- `copyAssets`, `updatePackageJson`, `createEntryPoints`
- AST utilities: `insertImport`, `findNodes`, `findClass`, `addGlobal`, etc.
- TypeScript utilities: `printDiagnostics`, `runTypeCheck`, `loadTsTransformers`, `addTsLibDependencies`

Additionally, `@nx/js` generators (7 total) and executors (7 total) are severely under-documented: docs show 3 generators and 3 executors, missing `setup-verdaccio`, `setup-build`, `setup-prettier`, `typescript-sync` (hidden) generators and `copy-workspace-modules`, `prune-lockfile`, `release-publish` (hidden), `verdaccio` executors.

**Impact**: Users following official guides hit dead ends when looking for API details.

## Recommendations

### Immediate (this sprint)

1. **Fix the `*-project` -> `configuration` rename drift**: Update `docs/packages.json` for `webpack`, `jest`, `esbuild`, `rollup`, `cypress` (6 packages, same pattern). This is the lowest-effort, highest-impact fix.

2. **Remove stale Angular entries from docs**: Drop `downgrade-module`, `karma`, `karma-project`, `upgrade-module`, `change-storybook-targets`. Add `directive`, `pipe`, `ngrx-feature-store`, `ngrx-root-store`, `convert-to-rspack`, `convert-to-application-executor`. Reclassify `webpack-browser`, `dev-server`, `webpack-server` as builders. Add `application`, `browser-esbuild`, `extract-i18n`, `unit-test` executors.

3. **Fix `@nx/expo` docs**: Remove 8 deleted executors, add `serve` and `submit`.

4. **Fix `@nx/node` docs**: Remove `webpack` and `node` executors (no executors.json exists). Add `setup-docker` generator.

5. **Fix `@nx/next` docs**: Remove `export` executor (no longer in code). Add `cypress-component-configuration` generator.

6. **Add missing packages to `docs/packages.json`**: Prioritize `remix` (17 items), `rspack` (10), `playwright` (4), `vitest` (3), `eslint` (5).

### Near-term (this quarter)

7. **Add `@nx/js` to the doc generation pipeline**: Single highest-impact change for API reference completeness. Also update `docs/packages.json` to reflect actual 7 generators and 7 executors.

8. **Document `releaseTag` migration**: The 10 deprecated `releaseTagPattern*` properties need a migration guide before Nx 23.

9. **Add JSDoc to the 8 missing `nx` core exports**: Especially `logger`, `output`, `defaultTasksRunner`.

10. **Fix remaining drift**: `@nx/react-native` (generators removed, executors added), `@nx/workspace` (3 removed generators still in docs, executor categorization error), `@nx/storybook` (2 removed generators still in docs, 3 migration generators missing), `@nx/vue` (3 missing generators), `@nx/react` (missing `module-federation-static-server` executor), `@nx/web` (3 executors removed but still in docs), `@nx/plugin` (`e2e` executor removed but still in docs).

11. **Evaluate promoting top semi-public devkit utils to public API**: `forEachExecutorOptions` (26 users), `addPlugin` (23), `getNamedInputs` (22).

### Long-term

12. **Create a plugin author guide** covering recommended devkit APIs for common tasks.

13. **Establish a devkit-internals stability tier** for commonly-needed internal exports, including the new `PluginCache` and `safeWriteFileCache`.

14. **Remove deprecated items 2+ major versions past deadline**: 7 items targeting v20 removal are still present in v22.

15. **Clean up stale `docs/packages.json` entries**: Remove `angular-rsbuild`, `linter`, `tao` which have no matching package directories.
