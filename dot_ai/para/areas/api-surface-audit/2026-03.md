## Summary

| Severity | Count |
| -------- | ----- |
| High     | 5     |
| Medium   | 8     |
| Low      | 4     |

**Overall**: The `@nx/devkit` public API files (`devkit-exports.ts`, `public-api.ts`, `devkit-internals.ts`) are unchanged from the Feb 2026 audit (40 / 18 / 28 export statements respectively). The core API surface remains stable.

**Key gaps remain**: `@nx/js` still has zero generated API docs. 13 packages with generators/executors are still missing from `docs/packages.json`. The `releaseTag` deprecations (Nx 23 removal) remain undocumented in migration guides.

**Expanded this month**: Deep generator/executor drift analysis across ALL documented packages reveals widespread rename drift (`*-project` -> `configuration` pattern) and significant stale entries in `docs/packages.json` for `@nx/angular`, `@nx/expo`, `@nx/react-native`, `@nx/storybook`, and `@nx/workspace`.

**Delta from Feb 2026 audit**: No changes to public API barrel files. `docs/packages.json` now includes `angular-rspack` (new). `angular-rsbuild` still listed but no matching package in code. The primary new finding is the depth of generator/executor name drift across documented packages (see H4, M6-M8).

## High Priority

### H1. `@nx/js` utility API has zero generated documentation

**Status**: Unchanged from Feb audit.

`@nx/js` exports ~25 re-export paths from `packages/js/src/index.ts` covering utility functions/types but is **not included** in the docs generation pipeline (`astro-docs/src/plugins/nx-reference-packages.loader.ts`). The loader only generates docs for: `nx`, `plugin`, `web`, `workspace`, `devkit`, CNW, and CLI.

Key undocumented exports:

- `createLockFile`, `getLockFileName` (re-exported from `nx/src/plugins/js/lock-file/lock-file`)
- `createPackageJson` (re-exported from `nx/src/plugins/js/package-json/create-package-json`)
- `readTsConfig`, `readTsConfigPaths`, `getRootTsConfigPath`
- `addTsConfigPath`, `getRelativePathToRootTsConfig`
- `libraryGenerator`, `initGenerator`, `setupPrettierGenerator`, `setupVerdaccio`
- `copyAssets`, `updatePackageJson`, `createEntryPoints`
- AST utilities: `insertImport`, `findNodes`, `findClass`, `addGlobal`, etc.
- TypeScript utilities: `printDiagnostics`, `runTypeCheck`, `loadTsTransformers`, `addTsLibDependencies`

**Impact**: Users following official guides hit dead ends when looking for API details.

## Recommendations

### Immediate (this sprint)

1. **Fix the `*-project` -> `configuration` rename drift**: Update `docs/packages.json` for `webpack`, `jest`, `esbuild`, `rollup`, `cypress` (6 packages, same pattern). This is the lowest-effort, highest-impact fix.

2. **Remove stale Angular entries from docs**: Drop `downgrade-module`, `karma`, `karma-project`, `upgrade-module`, `change-storybook-targets`, `webpack-browser`, `dev-server`, `webpack-server`. Add `directive`, `pipe`, `ngrx-feature-store`, `ngrx-root-store`, `convert-to-rspack`, `application`, `browser-esbuild`, `extract-i18n`, `unit-test`.

3. **Fix `@nx/expo` docs**: Remove 8 deleted executors, add `serve` and `submit`.

4. **Fix `@nx/node` docs**: Remove `webpack` and `node` executors (no executors.json exists). Add `setup-docker` generator.

5. **Add missing packages to `docs/packages.json`**: Prioritize `remix` (18 items), `rspack` (11), `playwright` (5), `vitest` (3), `eslint` (6).

### Near-term (this quarter)

6. **Add `@nx/js` to the doc generation pipeline**: Single highest-impact change for API reference completeness.

7. **Document `releaseTag` migration**: The 10 deprecated `releaseTagPattern*` properties need a migration guide before Nx 23.

8. **Add JSDoc to the 8 missing `nx` core exports**: Especially `logger`, `output`, `defaultTasksRunner`.

9. **Fix remaining drift**: `@nx/react-native` (generators removed, executors added), `@nx/workspace` (3 removed generators still in docs), `@nx/storybook` (2 removed generators still in docs), `@nx/vue` (3 missing generators).

10. **Evaluate promoting top semi-public devkit utils to public API**: `forEachExecutorOptions` (26 users), `addPlugin` (23), `getNamedInputs` (22).

### Long-term

11. **Create a plugin author guide** covering recommended devkit APIs for common tasks.

12. **Establish a devkit-internals stability tier** for commonly-needed internal exports.

13. **Remove deprecated items 2+ major versions past deadline**: 7 items targeting v20 removal are still present in v22.
