## Systemic Issues (Persistent)

1. **Generator rename drift**: `*-project` -> `configuration` across 8+ packages. Docs still list old names (jest-project, rollup-project, esbuild-project, webpack-project, cypress-project).

2. **`convert-to-inferred` generators**: Present in 12+ packages, zero documented. These are critical for Crystal migration workflow.

3. **Angular executor modernization**: Docs still list webpack-based executors (`webpack-browser`, `webpack-server`, `dev-server`) while repo has `application`, `browser-esbuild`, `unit-test`.

4. **`init` generators**: Docs list `init` as visible for many packages, but repo marks them all as `hidden`. Contributes to stale generator count.

5. **13 packages** with user-facing generators/executors have no API reference documentation in `packages.json`. Many DO have doc pages in `map.json` (overview guides) but no auto-generated API reference.

6. **`linter` -> `eslint` rename** still not reflected. Docs have `linter` with `eslint` executor; repo has `eslint` package with `lint` executor.

7. **AI CLI commands undocumented**: 3 new commands (`ai`, `mcp`, `configure-ai-agents`) shipped without CLI reference docs despite being promoted in blog posts.

8. **Tailwind generator deprecation wave**: 4 packages (`angular`, `react`, `remix`, `vue`) have `setup-tailwind` generators deprecated for Nx 23 removal. No migration guide exists.
