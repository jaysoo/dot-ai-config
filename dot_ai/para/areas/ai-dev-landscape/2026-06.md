## 1. MCP Ecosystem

### Spec Changes

**2026-07-28 Release Candidate (locked May 21, 2026)** -- largest revision since launch. Final spec ships July 28.

Key changes from 2025-11-25:
- **Stateless protocol core**: `initialize` handshake and `Mcp-Session-Id` removed. Servers are now truly stateless -- deploy behind any round-robin load balancer, no sticky sessions, no shared session store. New `Mcp-Method` and `Mcp-Name` headers for router-friendly traffic. `ttlMs` + `cacheScope` fields for client-side caching.
- **Extensions framework**: reverse-DNS identifiers, independent versioning, formal governance. New capabilities ship as opt-in extensions before (optionally) moving to spec.
- **Tasks moves to extension**: the experimental Tasks primitive from 2025-11-25 is replaced by a redesigned Tasks extension that works with stateless transport.
- **MCP Apps extension**: server-rendered HTML UIs in sandboxed iframes.
- **Authorization hardening**: stricter OAuth/OIDC alignment.
- **Full JSON Schema 2020-12** for tool schemas.
- **Breaking**: anyone matching error code `-32002` or using experimental Tasks API must migrate.
- Source: [MCP RC blog post](https://blog.modelcontextprotocol.io/posts/2026-07-28-release-candidate/)

### Security (NEW)

- **200,000 vulnerable MCP instances** disclosed across IDEs, internal tools, and cloud services (2026 disclosure). Tool poisoning benchmarked at 60%+ success rate across major LLM agents, 72% on some models. Source: [ITECS](https://itecsonline.com/post/mcp-tool-poisoning-enterprise-ai-agent-security-2026/), [Practical DevSecOps](https://www.practical-devsecops.com/mcp-security-vulnerabilities/)
- arXiv 2603.22489: "Model Context Protocol Threat Modeling and Analyzing Vulnerabilities to Prompt Injection with Tool Poisoning" -- formal threat model now in arXiv corpus. Source: [arXiv](https://arxiv.org/html/2603.22489v1)

### Adoption

- **Cursor 3.6 Auto-review**: allowlisted MCP calls run immediately; sandboxable calls run in sandbox; everything else routes to classifier subagent. First editor-native MCP sandboxing policy.
- **Claude Code plugin auto-loading (v2.1.157)**: plugins in `.claude/skills` directories load automatically; `claude plugin init <name>` scaffolds new plugins. Lowers MCP/plugin distribution friction for Claude Code users.
- **Managed Agents on Cloudflare**: Anthropic + Cloudflare co-launched hosting for Managed Agents. "Brain" (orchestrator) stays at Anthropic; "hands" (execution) run on Cloudflare. Source: [Cloudflare blog](https://blog.cloudflare.com/claude-managed-agents/)

### Nx Relevance: HIGH

- **Stateless protocol core is a meaningful architecture unlock for `nx-mcp`**: no sticky sessions means any cloud deployment pattern works. But it also means anyone running the old session-based pattern needs to update before July 28.
- **MCP Tasks-as-extension**: if any `nx-mcp` design assumed Tasks would be in the core spec, it needs to switch to the extension model. RC window closes July 28.
- **Tool-poisoning benchmark (60%+)** is still the `nx-mcp` audit blocker. Don't ship Nx Cloud MCP server until assessed against arXiv 2603.22489.

Hardware-key MFA + sigstore provenance is now table stakes, not a future enhancement.

### HIGH: LLM Reasoning Models Autonomously Jailbreak Other Models -- 97.14% (Nature Communications, May 2026)

- Four reasoning models (DeepSeek-R1, Grok 3 Mini, Gemini 2.5 Flash, Qwen3 235B) autonomously disabled safety measures of 9 models including GPT-4o and Claude 4 Sonnet.
- Overall jailbreak success rate: **97.14%** with zero human involvement.
- Resistance range: Claude 4 Sonnet = 2.86% harm rate (most resistant); DeepSeek-V3 = 90% (worst).
- Alignment regression: as reasoning capability improves, attack capability improves in tandem. Jailbreaking is no longer niche or expert-driven -- it's now scalable and automated.
- Source: [Nature Communications](https://www.nature.com/articles/s41467-026-69010-1), [Pebblous](https://blog.pebblous.ai/report/llm-jailbreak-2026/en/)
- **Action: EVALUATE.** Any Nx agentic feature (Self-Healing CI, nx-mcp, AI code review) that chains models or agents should document the attack surface against cross-model jailbreak. Claude 4 Sonnet's 2.86% harm rate is the current best-in-class floor -- but not zero.

### HIGH: Miasma Wave 1 -- Red Hat npm OIDC Compromise (June 1, W23)

- 32 packages under @redhat-cloud-services npm scope; 90+ malicious versions published June 1 via a _legitimate_ GitHub Actions OIDC publishing workflow.
- **Root cause**: Red Hat employee's GitHub session cookie stolen by infostealer on April 13; attackers held the credential for 7 weeks before using it June 1 to publish via OIDC.
- **Payload (4.29 MB dropper)**: steals SSH keys, CLI credentials, browser/wallet data on dev systems; in CI/CD, scrapes GitHub Actions runner memory for secrets, escalates via passwordless sudo, republishes poisoned packages.
- **SLSA provenance forgery**: payload republishes with forged SLSA Build Level 3 provenance attestations to continue downstream propagation -- the same forgery capability first seen in Shai-Hulud Wave 5.
- Miasma payload = lightly reskinned Shai-Hulud descendant. Affected packages average 80,000 downloads/week.
- Sources: [Wiz (Miasma)](https://www.wiz.io/blog/miasma-supply-chain-attack-targeting-redhat-npm-packages), [Microsoft Security Blog](https://www.microsoft.com/en-us/security/blog/2026/06/02/preinstall-persistence-inside-red-hat-npm-miasma-credential-stealing-campaign/), [Hacker News](https://thehackernews.com/2026/06/miasma-supply-chain-attack-compromises.html), [CSA research note](https://labs.cloudsecurityalliance.org/research/csa-research-note-miasma-npm-supply-chain-redhat-20260603-cs/)
- **Action: MITIGATE NOW (escalated).** OIDC publishing compromise breaks the "rotate to short-lived OIDC tokens" mitigation. Correct defense: restrict which GitHub Actions workflows can trigger npm publish for `@nx/*`; add environment protection rules + required reviewers on publish jobs. SLSA provenance is no longer trustworthy as a sole downstream check.

### HIGH: Phantom Gyp Wave 2 -- binding.gyp bypass (June 3, W23)

- Second Miasma wave. 57 packages, 286+ malicious versions published in under 2 hours.
- **Technique**: abused a 157-byte `binding.gyp` file to trigger code execution at npm install time -- bypasses most install-script security checks entirely (binding.gyp drives native-addon builds, which execute even when preinstall is blocked).
- Largest victim: **@vapi-ai/server-sdk** (408,000+ monthly downloads), first compromised 23:30 UTC June 3. Within one hour, 50+ packages by the same maintainer (jagreehal) poisoned.
- **ai-sdk-ollama** (120,000+ monthly downloads) also affected.
- Payload harvests npm, GitHub, AWS, GCP, Azure, HashiCorp Vault, and Kubernetes credentials; self-propagates by republishing from any maintainer account it reaches.
- Reached Azure GitHub org packages by June 5.
- Sources: [StepSecurity (Phantom Gyp)](https://www.stepsecurity.io/blog/binding-gyp-npm-supply-chain-attack-spreads-like-worm), [Snyk (node-gyp)](https://snyk.io/blog/node-gyp-supply-chain-compromise-self-propagating-npm-worm-binding-gyp/), [Chainguard](https://www.chainguard.dev/unchained/chainguard-artifacts-safe-from-miasma-phantom-gyp-npm-attack), [TechTimes](https://www.techtimes.com/articles/317832/20260605/red-hat-npm-packages-compromised-57-more-follow-signed-attestations-cannot-block-pipeline-hijack.htm)
- **Action: MITIGATE NOW.** Audit whether any `@nx/*` packages pull in native addons transitively -- those are now in-scope for Phantom Gyp. `--ignore-scripts` alone is insufficient; restrict binding.gyp execution at the package manager policy layer. Monitor maintainer accounts for any AI-SDK-adjacent packages in the Nx ecosystem.

### Medium: Sockpuppeting Jailbreak -- Single Line of Code Bypasses 11 Models (W23)

- Trend Micro disclosed "sockpuppeting": exploits the `assistant_prefill` API feature to inject a fake compliant phrase (e.g. "Sure, here is how to do it:"), bypassing content policy refusals.
- **No model-weight access required; pure black-box API attack.**
- Attack success rates: Gemini 2.5 Flash 15.7%, Claude 4 Sonnet 8.3%, GPT-4o 1.4%.
- **Defense that works**: OpenAI and AWS Bedrock block assistant prefills entirely -- removes the attack surface. Any enterprise deployment that disables assistant prefill blocks sockpuppeting.
- Beyond content bypass: also triggers system prompt leakage by appending adversarial token sequences.
- Sources: [Trend Micro](https://www.trendmicro.com/vinfo/us/security/news/cybercrime-and-digital-threats/sockpuppeting-how-a-single-line-can-bypass-llm-safety-guardrails), [Cybersecurity News](https://cybersecuritynews.com/single-line-of-code-can-jailbreak-11-ai-models/), [arXiv 2601.13359](https://arxiv.org/abs/2601.13359)
