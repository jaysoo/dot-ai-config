## 1. MCP Ecosystem

### Spec & SDK

- **2026-07-28 Release Candidate locked May 21** -- the largest MCP revision since launch. Key changes:
  - **Stateless core**: drops the `initialize` handshake and session IDs. Remote servers can now sit behind a round-robin load balancer, routing on `Mcp-Method` header. No sticky sessions, no shared session store needed.
  - **Extensions framework**: formal process for independent extension repos with separate versioning. Extensions can evolve without touching core spec.
  - **MCP Apps**: server-rendered HTML interfaces exposed through MCP.
  - **Tasks extension**: long-running operations with retry and expiry semantics.
  - **Full JSON Schema 2020-12**: tool definitions now have first-class schema validation.
  - **Authorization hardening**: six changes aligned with OAuth 2.0 and OIDC (issuer validation, credential binding, etc.).
  - **Deprecations (12-month runway before removal)**: Roots, Sampling, Logging.
  - Final spec publishes **July 28, 2026**. SDK maintainers have a 10-week validation window.
  - Source: [MCP 2026-07-28 RC blog post](https://blog.modelcontextprotocol.io/posts/2026-07-28-release-candidate/)
- **TS SDK** still at 1.29.0 as of last scan. Expect major version bump when July spec lands.
- **2026 roadmap (March)** four priorities all tracked in this RC: stateless Streamable HTTP, Tasks, governance/extensions, enterprise readiness.

### Adoption & Tooling

- **WebMCP origin trial in Chrome 149 (announced Google I/O May 19)** -- proposed W3C open standard co-developed with Microsoft. Sites expose JS functions + HTML forms as MCP tools for browser-based agents. Only Gemini in Chrome consumes WebMCP today; spec incubated in W3C Web ML community group. Direct signal that MCP is extending beyond desktop coding tools into browser/web surfaces.
  - Source: [Chrome for Developers - WebMCP](https://developer.chrome.com/docs/ai/webmcp), [eWeek](https://www.eweek.com/news/google-webmcp-chrome-ai-web-standard-preview/)
- **Cursor team marketplace** (May 1, in previous report) -- Default Off / Default On / Required tiers for MCP servers.
- **Codex CLI goal mode GA** -- plugin marketplace improvements: version-aware sharing, share checkout, registry-backed remote environments.
- **Copilot CLI v1.0.40** -- `client_credentials` OAuth for headless MCP in CI (in previous report).
- **Anthropic acquires Stainless (May 18)** -- Stainless generated Anthropic SDKs (TS, Python, Go, Java), CLIs, and MCP servers from API specs. Used by OpenAI, Google, Cloudflare. Hosted Stainless products shutting down. Teams using Stainless-generated client libraries for `nx-mcp` integration should watch for sunset announcements.
  - Source: [Anthropic blog](https://www.anthropic.com/news/anthropic-acquires-stainless), [TechCrunch](https://techcrunch.com/2026/05/18/anthropic-has-acquired-the-dev-tools-startup-used-by-openai-google-and-cloudflare/)

## 5. AI Security Threats

### HIGH: npm Supply Chain Worm -- TanStack + node-ipc + AntV (May 11-19)

Three Mini Shai-Hulud waves in 9 days, all attributed to TeamPCP:

**TanStack (May 11)**:
- 84 malicious versions across 42 `@tanstack/*` packages (170+ total npm + PyPI packages, 404 malicious versions).
- Attack chain: GitHub Actions "Pwn Request" (pull_request_target on fork) -> cache poisoning across fork/base trust boundary -> OIDC token extraction from runner process memory (`/proc/<pid>/mem`).
- Published 19:20-19:26 UTC; detected within 20-26 min by StepSecurity researcher.
- `@tanstack/react-router` alone: 12.7M weekly downloads.
- Sources: [TanStack postmortem](https://tanstack.com/blog/npm-supply-chain-compromise-postmortem), [Wiz](https://www.wiz.io/blog/mini-shai-hulud-strikes-again-tanstack-more-npm-packages-compromised), [Orca Security](https://orca.security/resources/blog/tanstack-npm-supply-chain-worm/), [StepSecurity](https://www.stepsecurity.io/blog/mini-shai-hulud-is-back-a-self-spreading-supply-chain-attack-hits-the-npm-ecosystem)

**node-ipc (May 14)**:
- Three malicious versions (9.1.6, 9.2.3, 12.0.1) via expired domain re-registration (`atlantis-software.net`, re-registered May 7 via Namecheap). Attacker hijacked maintainer's email, then npm 2FA.
- Harvests SSH keys, API tokens, DB credentials, cloud CLI configs (AWS/Azure/GCP/k8s), Docker secrets.
- Exfiltrates via DNS TXT queries to `bt.node.js` using bootstrap resolver `sh.azurestaticprovider.net:443` (Azure lookalike). No HTTP/HTTPS -- evades egress firewalls.
- 822K weekly downloads.
- Sources: [StepSecurity](https://www.stepsecurity.io/blog/node-ipc-npm-supply-chain-attack), [Cyber Express](https://thecyberexpress.com/node-ipc-npm-package-credential-stealer/)

**AntV/Mini Shai-Hulud (May 19)**:
- 637 malicious versions across 323 `@antv/*` data-visualization packages in 22-minute burst (01:39-02:06 UTC).
- Compromised npm account "atool". Payload: ~499 KB obfuscated JS, harvests GitHub Actions secrets.
- Dual-channel exfil: commits encrypted AES-256-GCM data to branches in `antvis/G2` GitHub repo (primary); HTTPS to `t.m-kosche.com:443` (fallback).
- Sources: [Microsoft Security Blog (May 20)](https://www.microsoft.com/en-us/security/blog/2026/05/20/mini-shai-hulud-compromised-antv-npm-packages-enable-ci-cd-credential-theft/), [Wiz](https://www.wiz.io/blog/mini-shai-hulud-teampcp-hits-antv-supply-chain), [Snyk](https://snyk.io/blog/mini-shai-hulud-antv-npm-supply-chain-attack/)

**Action: MITIGATE NOW (elevated from April):**
- npm worm cadence is now approximately weekly. This is operational threat, not research.
- Verify `@nx/*` and `nx-cloud` published packages have no unintended preinstall/postinstall scripts.
- Audit GitHub Actions workflows for `pull_request_target` + `actions/cache` patterns (TanStack attack vector).
- All GitHub Actions OIDC tokens: minimal-scope, short-lived. Check for memory-extraction risk in CI runners.
- Enable npm 2FA for all maintainer accounts; use passkeys or hardware tokens (domain re-reg attack against email-based 2FA).
- Consider sigstore signing for all `@nx/*` packages.
- Block DNS-based exfil in CI network policy (node-ipc used DNS TXT to `bt.node.js`).

### HIGH: Semantic Kernel RCE -- CVE-2026-25592 (CVSS 10.0) + CVE-2026-26030 (CVSS 9.8)

- Disclosed May 7 via [Microsoft Security Blog](https://www.microsoft.com/en-us/security/blog/2026/05/07/prompts-become-shells-rce-vulnerabilities-ai-agent-frameworks/).
- **CVE-2026-25592 (CVSS 10.0, .NET)**: `DownloadFileAsync` accidentally tagged `[KernelFunction]` and exposed to the LLM with no path validation. Prompt injection -> agent calls it -> sandbox escape from Azure Container Apps.
- **CVE-2026-26030 (CVSS 9.8, Python)**: `InMemoryVectorStore` filter routes attacker-controlled fields into `eval()`. Prompt injection -> arbitrary code execution.
- Patched: semantic-kernel 1.71.0 (.NET), 1.39.4 (Python).
- Same class as CVE-2026-5603 (MCP RCE, Anthropic declined to patch). The "prompt injection -> RCE" pattern in agent frameworks is now fully validated.
- Sources: [Microsoft Security Blog](https://www.microsoft.com/en-us/security/blog/2026/05/07/prompts-become-shells-rce-vulnerabilities-ai-agent-frameworks/), [PointGuard AI](https://www.pointguardai.com/ai-security-incidents/semantic-kernel-lets-a-prompt-open-a-shell-cve-2026-25592-cve-2026-26030), [NVD CVE-2026-25592](https://nvd.nist.gov/vuln/detail/CVE-2026-25592)
- **Action: EVALUATE.** If Nx uses Semantic Kernel in any agent pipeline, patch immediately. Even if not: audit all internal agent frameworks for accidentally-exposed callable functions. This is the new `[KernelFunction]` annotation audit.

## Implications for Nx Roadmap

### VALIDATE

- **MCP centralized distribution** (Cursor Marketplace, Codex plugin marketplace, Copilot `client_credentials` OAuth, WebMCP). Confirms "where does `nx-mcp` get installed and authorized in an enterprise" is a real product question, not a docs problem.
- **Long-horizon agent loops** (Codex goal mode GA, Claude Code `/goal`, Antigravity parallel orchestration). Confirms Self-Healing CI "iterate until green with budget caps" is the correct UX pattern -- now table-stakes.
- **Multi-repo environments** (Cursor 3.4, Antigravity 2.0). Monorepo + AI agent context is the frontier challenge. `nx-mcp` project graph data is directly relevant here.
- **WebMCP** -- MCP expanding to browser surfaces validates MCP as the integration protocol for AI tooling, not a niche CLI primitive.

### CHALLENGE

- **AI vuln scanning is now a paid SKU** (Anthropic, Cursor). Any Nx Cloud AI code review feature needs clear monorepo-aware differentiation before this category fully solidifies.
- **Stateless MCP spec RC (July 28)** -- `nx-mcp` will need architectural work to go stateless or document sticky-session requirements. This is a near-term engineering constraint, not a roadmap item.
- **Cloudflare linguistic-stereotyping finding** -- enterprise AI features will be audited for bias. We need a published stance.
- **TeamPCP weekly worm cadence** -- ongoing threat group targeting npm at weekly frequency challenges assumption that published packages are safe between security audits. Continuous monitoring required.

### OPPORTUNITY

- **Publish "AI-Ready Workspace" narrative** with Mythos visible-CVE numbers (271 Firefox, 40 attributed Anthropic). Patch lag on flat repos is now measurable; structured workspaces with `affected` reduce patch-time. Strongest external data point of the quarter.
- **MCP hardening compliance mark** -- if `nx-mcp` passes audit against CVE-2026-5603, OWASP MCP Top 10, Unit 42 sampling vectors, Comment-and-Control, Semantic Kernel-class KernelFunction audit -- that's marketable in enterprise sales.
- **Headless OAuth (`client_credentials`) for `nx-mcp` in CI** -- align with Copilot CLI's pattern. Reduces friction for CI-installed MCP servers.
- **Stateless `nx-mcp`** -- proactively conforming to the July 28 spec before competitors could be a differentiator for teams running `nx-mcp` behind a load balancer.
- **GitHub Actions hardening guidance** -- Nx is positioned to publish "AI-era CI hardening for monorepos" covering: OIDC scope, `pull_request_target` audit, cache integrity, DNS egress control. Directly addresses TanStack-class attacks.

### WATCH

- **Grok 5 (Q2 window)** -- still not shipped. Open-frontier with offensive-cyber claims would re-shift threat model.
- **MCP July 28 final spec** -- SDK version bumps, Nx MCP server rework required.
- **Glasswing July accounting** -- full Mythos CVE list lands. Major narrative event for "AI-Ready Workspace."
