### W22 Implications for Nx (NEW)

1. **CI auto-fix is now an Anthropic-shipped product.** Self-Healing CI's wedge must be monorepo-aware affected-scope, dependency-graph-aware fix proposals, build-graph-aware test selection. The "we detect failing CI and propose a fix" core is no longer differentiated.
2. **Managed Agents = where production agents will run.** Any Nx Cloud agent story needs to answer "do we host or do we delegate to Anthropic Managed Agents?" If we host, we're competing with Anthropic's operational simplicity claim.
3. **MCP Tasks-as-extension breaking change.** If any `nx-mcp` design assumed Tasks would be in the core spec, switch to the extension model now. RC is May 21, final spec July 28 — narrow window.
4. **`claude agents` + background sessions normalize "agent as runtime"** (matches Windsurf Devin Local, Cursor cloud agents). Multiple long-running agents per repo is the new baseline UX. `nx-mcp` should expect to be invoked from background/headless contexts (relates to Copilot `client_credentials` from W19 — still the right pattern).
5. **Mini Shai-Hulud cadence (5 waves in 60d) + bun-hardened payload** = npm publishing for `@nx/*` and `nx-cloud` needs hardware-key MFA + provenance + sigstore. The AntV maintainer-account compromise (300 pkgs in 22min) is the exact threat model. Move from "should do" to "must do this quarter." Tie to `audit-supply-chain`.
6. **MCP sampling-vector research now in arXiv corpus** — `nx-mcp` audit blocker carries over from April + W19. Don't ship Nx Cloud MCP server until sampling-attack mitigations are explicit.
7. **Code w/ Claude "no new model" framing** — Anthropic is signaling that the model is plateauing and the product surface is where 2026 differentiation happens. Same read for Nx: monorepo-aware tooling around any model is more durable than betting on a specific model improvement.

### HIGH: Mini Shai-Hulud — SAP CAP npm Compromise (Apr 29–30)

- **TeamPCP** (different actor, same playbook) compromised four SAP CAP / MBT npm packages with malicious **preinstall scripts** (executes at `npm install`).
- Bun-based secret stealer harvests local dev credentials, GitHub/npm tokens, GitHub Actions secrets, AWS/Azure/GCP/Kubernetes cloud secrets. Exfil via public GitHub repos (encrypted).
- Published Apr 29 09:55–12:14 UTC. SAP issued Security Note 3747787 on Apr 30.
- Fourth Shai-Hulud-family wave in <60 days (Bitwarden CLI Apr, Shai-Hulud "Third Coming" Apr 22, Mini Shai-Hulud Apr 29). The pattern is now a recurring background operation.
- Sources: [Hacker News](https://thehackernews.com/2026/04/sap-npm-packages-compromised-by-mini.html), [Wiz](https://www.wiz.io/blog/mini-shai-hulud-supply-chain-sap-npm), [Sophos](https://www.sophos.com/en-us/blog/-mini-shai-hulud-supply-chain-attack-targets-sap-npm-packages), [Aikido](https://www.aikido.dev/blog/mini-shai-hulud-has-appeared), [The Register](https://www.theregister.com/2026/04/30/supply_chain_attacks_sap_npm_packages/), [Onapsis](https://onapsis.com/blog/sap-cap-mini-shai-hulud-supply-chain-attack/), [Dark Reading](https://www.darkreading.com/cloud-security/teampcp-sap-packages-mini-shai-hulud).
- **Action: MITIGATE NOW (continued from April).** Re-verify `@nx/*` and `nx-cloud` 2FA + trusted publishing. Audit `nx` repo's preinstall/postinstall scripts. Monitor npm token age and revoke long-lived tokens. Consider sigstore signing for all published packages.

### HIGH: Vercel Breach via Context.ai OAuth (disclosed Apr 20)

- **New attack class for AI-era SaaS**: malware (Lumma Stealer) at AI vendor (Context.ai, ~Feb) → Google Workspace OAuth tokens → Vercel internal systems → exposed customer env vars.
- Root cause: Vercel employee granted "Allow All" OAuth scope to Context.ai's "AI Office Suite" using their enterprise Google account. Vercel's internal OAuth config permitted this.
- Vercel + GitHub + npm + Socket confirmed no Vercel npm packages tampered.
- Sources: [Vercel KB](https://vercel.com/kb/bulletin/vercel-april-2026-security-incident), [Hacker News](https://thehackernews.com/2026/04/vercel-breach-tied-to-context-ai-hack.html), [Trend Micro](https://www.trendmicro.com/en_us/research/26/d/vercel-breach-oauth-supply-chain.html), [TechCrunch](https://techcrunch.com/2026/04/20/app-host-vercel-confirms-security-incident-says-customer-data-was-stolen-via-breach-at-context-ai/), [OX Security](https://www.ox.security/blog/vercel-context-ai-supply-chain-attack-breachforums/), [Halborn](https://www.halborn.com/blog/post/explained-the-vercel-hack-april-2026).
- **Action: MITIGATE NOW.** Audit Nx Cloud OAuth scope policies — what AI vendors have we (or our customers) granted broad Google Workspace / GitHub OAuth to? Document blast radius.
