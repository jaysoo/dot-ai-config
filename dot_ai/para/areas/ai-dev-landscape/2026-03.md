## TL;DR

The AI coding tools space has entered the **event-driven automation era**. Cursor Automations (Mar 5) lets agents trigger on Slack messages, Linear issues, GitHub PRs, and PagerDuty incidents -- shifting agents from reactive assistants to proactive infrastructure. Anthropic published its 2026 Agentic Coding Trends Report identifying multi-agent coordination as the defining pattern for 2026. Codex CLI 0.110 shipped a plugin system with local marketplace. **Turborepo 2.8 finally shipped AI features** (agent skill, AI docs, task descriptions) -- ending the "no AI support" gap documented in previous scans. Cursor hit $2B ARR. Claude Code hit $2.5B run-rate revenue. MCP Dev Summit North America (Apr 2-3, NYC) announced 95+ sessions. For Nx: Turborepo's AI entry validates the market but its implementation is shallow compared to Nx's graph-aware approach. Cursor Automations create a new integration surface for Nx Cloud event-driven CI.

## 3. Cursor

### Releases (Jan-Mar 2026)

**Cursor Automations (Mar 5)** -- BREAKING NEWS ([TechCrunch](https://techcrunch.com/2026/03/05/cursor-is-rolling-out-a-new-system-for-agentic-coding/), [Cursor blog](https://cursor.com/blog/automations))

- **Event-driven agent execution**: Agents trigger from Slack messages, Linear issues, GitHub PRs, PagerDuty incidents, webhooks, and schedules
- **Use cases in production**:
  - Code review: Bugbot scans every new change, opens threads with testable suggestions
  - Security scanning: Triggered on every push to main, audits diffs for vulnerabilities, posts high-risk findings to Slack
  - Incident response: PagerDuty triggers agent that queries Datadog MCP for logs, examines codebase for recent changes, sends Slack summary + proposed fix PR
- Cursor estimates hundreds of automations per hour already running
- **Significance**: Shifts agents from reactive (user prompts) to proactive (event-driven). This is the "agent factory" pattern becoming production infrastructure

**v2.6 -- MCP Apps and Team Marketplaces (Mar 3)** ([changelog](https://cursor.com/changelog/2-6))

- MCP Apps: Interactive UIs (charts, diagrams, whiteboards) from Amplitude, Figma, tldraw render directly in agent chats
- Team Plugin Marketplace: Admins on Teams/Enterprise share private plugins with central governance
- Debug mode improvements

**v2.5 -- Plugins and Cloud Agents (Feb 17-24)** ([changelog](https://cursor.com/changelog))

- Plugin marketplace launched (AWS, Figma, Stripe, Linear as partners)
- Cloud agents in isolated VMs -- test their own changes, produce merge-ready PRs with video recordings
- Bugbot Autofix: 35%+ of fixes merged
- Subagents with granular network controls

### Revenue

- **$2B ARR** as of Feb 2026, doubled from three months prior ([Bloomberg](https://www.bloomberg.com/news/articles/2026-03-02/cursor-recurring-revenue-doubles-in-three-months-to-2-billion), [TechCrunch](https://techcrunch.com/2026/03/02/cursor-has-reportedly-surpassed-2b-in-annualized-revenue/))

### Competitive Assessment

Cursor Automations is a category-defining move. Event-driven agent execution on commits, PRs, and incidents is exactly what Nx Cloud's Self-Healing CI does -- but Cursor is making it a general-purpose platform. The PagerDuty -> Datadog MCP -> fix PR pipeline demonstrates MCP as the connective tissue for automated incident response. Nx should consider whether Cursor Automations could be an integration surface: Nx Cloud CI failures as triggers for Cursor automation agents that have Nx MCP context.

## 11. Anthropic 2026 Agentic Coding Trends Report -- NEW

### Eight Trends

1. **Shifting engineering roles**: Coding moves toward agent supervision, system design, and output review

4. **Scaling beyond engineering**: Agentic coding extends to domain experts across departments
5. **Human-AI collaboration patterns**: Developers use AI in 60% of work but fully delegate only 0-20% of tasks
6. **Security as core design principle**: Not bolted on after
7. **AI-automated review systems**: Scaling human-agent oversight
8. **Context window management**: Parallel reasoning across separate context windows

### Key Data Points

- Rakuten achieved 99.9% accuracy on 12.5M-line codebase modifications in 7 autonomous hours
- Google/MIT published predictive framework for scaling multi-agent systems with tool-coordination tradeoff model ([InfoQ](https://www.infoq.com/news/2026/03/google-multi-agent/))

### Nx Relevance

The report's emphasis on multi-agent coordination and context window management directly validates Nx's project graph approach. The "scaling beyond engineering" trend suggests Nx should consider how non-developers (PMs, designers) interact with monorepo AI capabilities.

## 14. Implications for Nx Roadmap

### VALIDATE (Nx is on the right track)

1. **MCP has won**. Apple in Xcode, Shopify/Google UCP, MCP Apps, 20k+ servers, MCP Dev Summit with 95+ sessions and major sponsors. Nx's MCP-first strategy is correct.

2. **Turborepo AI entry validates the market but is shallow**. Turborepo 2.8's agent skill teaches "how to use Turborepo" (best practices, anti-patterns). Nx's skills provide project-specific intelligence (affected analysis, dependency graph, task pipelines). The gap between "documentation-aware" and "workspace-aware" AI is Nx's moat.

3. **Self-Healing CI remains uniquely differentiated**. Cursor Automations adds event-driven agents, GitHub has Agentic Workflows, Jules has Render integration -- none are monorepo-aware. The CI-to-local feedback loop has no competitor.

4. **Plugin/skill distribution is converging**. Claude Code, Codex, Cursor, and Turborepo all ship plugin/skill systems. Nx already has this with the AI Agent Skills plugin. Being part of this pattern is correct.
