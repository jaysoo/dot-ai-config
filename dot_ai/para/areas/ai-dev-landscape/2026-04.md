The **Claude Agent SDK** enables using Claude Code as a library with subagent orchestration. **Windsurf 2.0 shipped April 15** with **Devin cloud agent natively embedded** and a Kanban-style Agent Command Center -- Cognition's acquisition now fully productized. A2A protocol hit **v1.0** with 150+ organizations and production deployments. MCP ecosystem surpassed **10,000+ indexed servers** with Server Cards (`.well-known/mcp`) progressing and new **Skills discovery (`.well-known/agent-skills/`)** merged; a proposal to **deprecate Roots, Sampling, and Logging** is the first major surface-area retirement in MCP. **AGENTS.md** was donated to the Linux Foundation's Agentic AI Foundation. OpenAI **Codex 26.415** (April 16) added computer-use, in-app browser, chats, and thread automations. **Paperclip** hit 46,900+ GitHub stars in five weeks. Jules gained proactive task suggestions and scheduled tasks.

**For Nx:** The "agent orchestrator" pattern in Cursor 3, Copilot Autopilot, and Windsurf 2.0 Agent Command Center maps directly to Nx's task graph. ## 1. MCP Ecosystem

### Spec & Governance

- **MCP 2025-11-25 spec** shipped with major additions: OpenID Connect Discovery authorization, icons as metadata for tools/resources/templates/prompts (SEP-973), incremental scope consent via WWW-Authenticate (SEP-835), URL mode elicitation (SEP-1036), tool calling in sampling (SEP-1577), and experimental **Tasks primitive** for durable requests with polling and deferred retrieval (SEP-1686).
  - Source: [MCP 2025-11-25 Changelog](https://modelcontextprotocol.io/specification/2025-11-25/changelog)
- **Server Cards (`.well-known/mcp`)** progressing via SEP-1649 and SEP-1960 -- structured server metadata for discovery without connecting. Broad community support, active implementation by major clients, not yet merged into core spec.
  - Source: [SEP-1649](https://github.com/modelcontextprotocol/modelcontextprotocol/issues/1649), [Ekamoira Guide](https://www.ekamoira.com/blog/mcp-server-discovery-implement-well-known-mcp-json-2026-guide)
- **NEW -- Skills-over-MCP Working Group formalized** (SEP-2586, merged April 2026) -- upgraded from Interest Group to Working Group. Signals that Agent Skills integration with MCP is now a committed long-term spec area.
- **NEW -- Skills discovery mechanism** (SEP-2589, merged April 2026) -- establishes `.well-known/agent-skills/` endpoint as the discovery convention for agent skills alongside MCP server discovery. Mirrors `.well-known/mcp` server cards.
- **NEW -- Governance clarification** (SEP-2592, merged April 2026) -- simplified working group formation requirements; area-specific reviewers rather than requiring full core maintainer group for every SEP.
- **NEW -- Pluggable Transports SEP in draft** (SEP-2598) -- expanding MCP's transport layer flexibility; early signal that stdio/Streamable HTTP may not be the end of the transport story.
- **NEW -- Feature deprecation policy SEP in draft** (SEP-2596) -- establishing deprecation lifecycle rules.
- **NEW -- Roots/Sampling/Logging deprecation proposed** (SEP-2577, draft) -- the first major proposal to retire MCP features. If adopted, MCP servers relying on these (including nx-mcp if applicable) would need migration paths.
  - Source: [MCP Spec Pulls](https://github.com/modelcontextprotocol/modelcontextprotocol/pulls)
- **Transport scaling gaps continue** -- Streamable HTTP unlocked remote server deployments but stateful sessions fight load balancers, horizontal scaling requires workarounds, no standard server capability discovery without connecting (hence Server Cards push).
  - Source: [The New Stack](https://thenewstack.io/model-context-protocol-roadmap-2026/)
- **Tasks primitive** gaining traction -- early production use surfacing gaps in retry semantics and expiry policies.
- **SDK tiering system** established (SEP-1730) -- clear requirements for feature support and maintenance commitments across SDKs.

### Nx Relevance: HIGH

- **Server Cards + Skills discovery** are where Nx Cloud's MCP server and Nx agent skills should be discoverable. Implementing `.well-known/mcp` and `.well-known/agent-skills/` early gives first-mover advantage in the "build system" category.
- **Roots/Sampling/Logging deprecation (SEP-2577)** is a new risk to track. Audit nx-mcp for any use of these features and plan migration if adopted.
- **Pluggable Transports (SEP-2598)** could open new integration paths for Nx Cloud MCP (e.g., direct WebSocket from CI to agent).
- Enterprise adoption of MCP (Salesforce, Lucidworks, Slack servers) means enterprise buyers expect MCP integration -- validates Nx Cloud MCP strategy.
- Shadow IT concerns create demand for governed MCP servers -- Nx Cloud's authenticated MCP fits this need.
- Pinterest's production MCP architecture (central registry, domain servers, approval flows) is a reference architecture Nx Cloud could mirror.
- **`gh skill` + `.well-known/agent-skills/`** parallel the discovery surface Nx's agent skills should hit. If Nx's skills are registered in these conventions, they become discoverable from Copilot/Claude Code/Cursor without manual config.

### CHALLENGE (may need rethinking)

- **Agent-authored commits** -- Copilot signs commits, Autopilot runs unsupervised, Devin runs on its own VM. Nx Cloud's CI attribution, caching, and affected analysis need to handle agent vs. human commits (different trust levels?).

- **A2A readiness** -- A2A v1.0 with 150+ orgs is no longer "watch." Nx needs a position on whether/how to participate in agent-to-agent communication.
- **MCP Server Cards + `.well-known/agent-skills/`** -- both discovery surfaces are progressing. Nx Cloud MCP server and Nx agent skills should prepare to expose metadata for automated discovery.
- **MCP Roots/Sampling/Logging deprecation** (SEP-2577, draft) -- audit nx-mcp for any use of these features. If the deprecation lands, Nx MCP server will need a migration.
- **Claude Code native binary CLI (v2.1.113)** -- JS-bundled agent CLIs may be a trend away from standard soon. Implications for any Node-based agent tooling Nx ships.
- **Untrusted-input surface in MCP/tool integrations** -- Claude Code v2.1.113 added yet another round of Bash-tool hardening (macOS `/private/*` rm rules, `env`/`sudo`/`watch` deny-rule wrapping, `find -exec` auto-approve removal, `dangerouslyDisableSandbox` bypass fix).
