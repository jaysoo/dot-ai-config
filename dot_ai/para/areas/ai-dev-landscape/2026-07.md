## 5. AI Security Threats

### HIGH: npm v12 Ships (July 8, 2026) -- Install Scripts Blocked by Default

npm v12 shipped July 8, 2026 with the most significant security redesign in its 16-year history:

- **preinstall/install/postinstall scripts no longer execute automatically** -- require explicit `npm approve-scripts`.
- **Git dependencies blocked by default** (`--allow-git` to re-enable).
- **Remote-URL dependencies blocked by default** (`--allow-remote` to re-enable).
- Direct response to North Korean state-sponsored campaigns (Axios, March 2026; Mastra AI, June 2026) and Miasma/Shai-Hulud.

**New attacks post-v12 (still within the week)**:
- **jscrambler v8.14.0 (July 11)**: malicious preinstall hook drops and executes platform-specific native binary on Linux/Windows/macOS. npm v12 blocks this; teams on npm 11 remain exposed.
- **Injective blockchain SDK (July 8)**: wallet recovery phrases and private keys captured via fake analytics code; 18 related packages auto-infected within minutes.

**CI impact**: Teams running `npm install` in CI without migrating to `npm approve-scripts` will see broken builds. npm 11.16.0 has been warning; v12 enforces.

Cybernews: experts say v12 alone is not enough -- provenance attestation and dependency pinning still required.

Sources: [TechTimes npm v12](https://www.techtimes.com/articles/319890/20260708/npm-v12-ships-this-month-blocking-install-scripts-that-enabled-year-supply-chain-attacks.htm), [The Hacker News](https://thehackernews.com/2026/07/npm-12-disables-install-scripts-by.html), [defend.network briefing July 13](https://defend.network/briefings/npm-supply-chain-zimbra-rce-android-malware-2026-07-13.html), [Cybernews](https://cybernews.com/security/npm-security-overhaul-adress-supply-chain-attacks/)

**Action: MITIGATE NOW.** Nx workspaces using npm must audit CI pipelines for install-script dependencies before upgrading to npm v12 (mandatory when Node ships it as default). `@nx/*` packages should confirm they do not rely on preinstall hooks.

### HIGH: Miasma Supply Chain Attack (June 1-17, 2026)

- **32 packages** under `@redhat-cloud-services` npm namespace compromised; **96 malicious versions** published in two waves on June 1.
- Root cause: attacker compromised a GitHub account via a VS Code extension containing malware; used that account to inject code into packages and abused GitHub Actions OIDC trusted-publishing to mint short-lived npm tokens.
- **SLSA provenance attestation forgery for the third time in this campaign** (Wave 5 also forged SLSA). Signed provenance is no longer a sufficient defense on its own.
- Payload: preinstall script, credential harvesting (npm, git, CI/CD tokens, **GCP and Azure cloud identities added** in this variant), self-spreading to other packages the victim can publish.
- Campaign marker: "Miasma: The Spreading Blight."
- **Resolved**: Red Hat closed the incident June 17. No Red Hat products confirmed built with compromised versions.
- Sources: [Wiz blog](https://www.wiz.io/blog/miasma-supply-chain-attack-targeting-redhat-npm-packages), [Microsoft Security Blog](https://www.microsoft.com/en-us/security/blog/2026/06/02/preinstall-persistence-inside-red-hat-npm-miasma-credential-stealing-campaign/), [Red Hat RHSB-2026-006](https://access.redhat.com/security/vulnerabilities/RHSB-2026-006)

**Action: MITIGATE NOW.** SLSA attestation forgery via OIDC token abuse is now a confirmed repeatable technique. The attack chain -- compromised developer account -> OIDC token -> forged SLSA -> published npm -- is exactly the path that would compromise `@nx/*` packages. Hardware-bound publish keys (YubiKey/HSM) + independent attestation verifier (not GitHub Actions-self-signed) are the required mitigations.

## Implications for Nx Roadmap

### VALIDATE

- **Sonnet 5 near-Opus-4.8 quality at lower cost**: Self-Healing CI and nx-mcp can use Sonnet 5 as the production model without sacrificing much quality vs Opus 4.8. Cost modeling should use Sonnet 5 as default.
- **AI-native monorepo advantage confirmed**: 1M context windows mean structured monorepos (Nx's value prop) are now first-class citizens for AI agents. This is exactly what Nx should be marketing.
- **Background-agent autonomy with auto-PR**: Claude Code's auto-push/PR behavior validates Nx Cloud's "agentic CI" vision -- the ecosystem is building toward it whether Nx ships it or not.
- **Bloomberg Pomona**: real-world validation that AI-driven incremental automated changes work at large monorepo scale. Nx's automated migration story has a new academic anchor.

### CHALLENGE

- **Manual permission mode as Claude Code default**: Anthropic's correction to auto-PR behavior signals that "fully autonomous agent with repo write access" needs more user trust-building before it goes mainstream. Nx Cloud's Self-Healing CI needs the same explicit safety documentation.
- **Kiro CLI V3 + spec-driven development reaching terminal**: spec-driven agentic CI is now available in the terminal (not just the IDE). Kiro's approach could expand into CI pipelines before Nx ships its own answer.
- **Devin Desktop multi-agent Spaces**: Cognition's "Spaces" abstraction (shared context across sessions/PRs/worktrees) is architecturally closer to how Nx Cloud manages distributed agent work than any other tool. Watch for market overlap.
- **SLSA attestation forgery x2**: hardware-bound keys are now urgent. Any delay exposes `@nx/*` to the same attack used against Red Hat.

### OPPORTUNITY

- **nx-mcp + Kiro context drift above 200 files**: `nx-mcp` can solve Kiro's ~200-file coherence limit by providing structured project graph context. Position as the "Kiro context layer" for large Nx monorepos.
- **Glasswing accounting + Nx structured workspace**: time the "structured workspace reduces patch-lag" piece to the July Glasswing CVE release. If 1,000+ CVEs drop, the monorepo tooling angle is immediately relevant.
- **Team MCPs governance (Cursor pattern)**: nx-mcp distribution to teams via admin-configured channels is a natural next step. Cursor has proven the UX model.
- **Enterprise AI credit budgeting (Copilot pattern)**: AI credit pools per cost center is coming to every enterprise tool. Nx Cloud's usage metering story should account for model-cost attribution.

### WATCH

- **Glasswing full accounting** -- now 2+ weeks overdue. Drop is imminent and will be the major Q3 narrative event.
- **MCP 2026-07-28 final spec** -- 15 days out. SDK Tier 1 windows close July 28. Three new attack surfaces need Nx assessment before then.
- **npm v12 rollout** -- shipped July 8; CI pipelines need `npm approve-scripts` migration before v12 becomes the Node default. Watch for upstream breakage in Nx workspace CI templates.
- **Miasma campaign next wave** -- five-wave cadence means another wave is plausible in July/August. jscrambler (July 11) may be a copycat or wave 7 probe.
- **Kiro monorepo support at scale** -- currently degrades >200 files; watch for improvements.
- **Devin Desktop "Spaces" multi-agent patterns** -- first production "agent workspace" abstraction. If it gains traction, it shapes what users expect from any multi-agent Nx Cloud feature.
- **EU AI Act enforcement (mid-2026 onward)**: agentic CI adversarial testing requirements may apply. Monitor ENISA guidance.
