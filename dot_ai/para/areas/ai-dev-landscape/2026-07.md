## 5. AI Security Threats

### HIGH: Miasma Supply Chain Attack (June 1-17, 2026)

- **32 packages** under `@redhat-cloud-services` npm namespace compromised; **96 malicious versions** published in two waves on June 1.
- Root cause: attacker compromised a GitHub account via a VS Code extension containing malware; used that account to inject code into packages and abused GitHub Actions OIDC trusted-publishing to mint short-lived npm tokens.
- **SLSA provenance attestation forgery for the third time in this campaign** (Wave 5 also forged SLSA). Signed provenance is no longer a sufficient defense on its own.
- Payload: preinstall script, credential harvesting (npm, git, CI/CD tokens, **GCP and Azure cloud identities added** in this variant), self-spreading to other packages the victim can publish.
- Campaign marker: "Miasma: The Spreading Blight."
- **Resolved**: Red Hat closed the incident June 17. No Red Hat products confirmed built with compromised versions.
- Sources: [Wiz blog](https://www.wiz.io/blog/miasma-supply-chain-attack-targeting-redhat-npm-packages), [Microsoft Security Blog](https://www.microsoft.com/en-us/security/blog/2026/06/02/preinstall-persistence-inside-red-hat-npm-miasma-credential-stealing-campaign/), [Red Hat RHSB-2026-006](https://access.redhat.com/security/vulnerabilities/RHSB-2026-006)

**Action: MITIGATE NOW.** SLSA attestation forgery via OIDC token abuse is now a confirmed repeatable technique. The attack chain -- compromised developer account -> OIDC token -> forged SLSA -> published npm -- is exactly the path that would compromise `@nx/*` packages. Hardware-bound publish keys (YubiKey/HSM) + independent attestation verifier (not GitHub Actions-self-signed) are the required mitigations.

### HIGH: Agentic AI Security Posture -- Enterprise Gap

- **Cisco State of AI Security 2026**: 88% of organizations reported AI agent security incidents; only 29% prepared to secure agentic deployments.
- MCP prompt injection CVEs (CVE-2025-68143/68144/68145) in Anthropic's own official Git MCP server demonstrated that README-as-context is a live attack surface for code execution.
- **70+ AI offensive security tools** catalogued in 18 months (per Hadrian research, March 2026). AI pentesting agents now outperform 9/10 humans in live enterprise environments (ARTEMIS benchmark).
- Sources: [TechStoriess AI agent security](https://www.techstoriess.com/ai-agent-security-practices-2026-prompt-injection-mcp-risks-data-leaks/), [Hadrian blog](https://hadrian.io/blog/the-ai-offensive-security-boom-seventy-tools-in-eighteen-months)

**Action: EVALUATE.** nx-mcp's README-loading behavior (or any context sourced from repo files) is directly analogous to the CVE-2025-68143 attack vector. Audit before shipping.

- Standing figures: 10,000+ high/critical flagged; 1,587 confirmed true positives; 1,094 confirmed high/critical.
- Source: [Anthropic Glasswing](https://www.anthropic.com/glasswing), [VulnCheck tracking](https://www.vulncheck.com/blog/anthropic-glasswing-cves)

**Action: MONITOR.** When the full accounting drops (expected this month), assess whether any named CVEs affect packages in the Nx workspace. Prepare "AI-Ready Workspace patch-lag" narrative for the July news cycle.

5. **Glasswing July accounting**: monitor this week/month. If 1,000+ named CVEs drop, the narrative around AI-discovered vulnerabilities in OSS will be the biggest security story of Q3. Nx should be positioned with a clear statement on patch process.

## Implications for Nx Roadmap

- **Sonnet 5 near-Opus-4.8 quality at lower cost**: Self-Healing CI and nx-mcp can use Sonnet 5 as the production model without sacrificing much quality vs Opus 4.8. Cost modeling should use Sonnet 5 as default.

- **SLSA attestation forgery x2**: hardware-bound keys are now urgent. Any delay exposes `@nx/*` to the same attack used against Red Hat.
