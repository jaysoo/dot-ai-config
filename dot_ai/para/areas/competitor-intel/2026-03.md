## Turborepo

**Latest stable**: v2.8.20 (March 19, 2026) -- npm `turbo@latest` = 2.8.20
**Previous stable**: v2.8.19 (March 18), v2.8.18 (March 18), v2.8.17 (March 16), v2.8.16 (March 11), v2.8.15 (March 9), v2.8.14 (March 11), v2.8.13 (March 3)
**Major feature release**: v2.8 blog post (January 23, 2026)

### New Since Last Scan (v2.8.19, v2.8.20)

**`turbo-ignore` Deprecated** (v2.8.19 -- strategic):

- `turbo-ignore` officially deprecated in favor of `turbo query affected`
- Tailored deprecation notice for Vercel users (migration path from Vercel-specific CI skip to generic query engine)
- **This completes the turbo-ignore → turbo query affected migration.** `turbo-ignore` was Vercel's CI skip tool; replacing it with `turbo query affected` unifies the "what changed?" story under one query engine. Users now use `turbo query affected` for both local development and CI skip decisions.

**Package Binary Resolution Fixes** (v2.8.19):

- Detect scoped `@turbo/{platform}` packages in local binary resolution
- Support legacy unscoped binary package names in Node.js shim
- These are packaging/distribution fixes for the `turbo` binary resolution chain

**Persistent Task Fix** (v2.8.20):

- Prevent `stdin` from being dropped for persistent tasks in stream mode
- Hotfix release -- one commit, shipped next day

**Examples**: Upgraded Next.js to 16.2.0 across all examples and apps (v2.8.19)

### Previously Reported (v2.8.18)

**Task-Level Filtering** (v2.8.18 -- strategic):

- New `filterUsingTasks` future flag for task-level filter resolution -- extends the `affectedUsingTaskInputs` pattern to filtering
- `turbo query affected` CLI shorthand -- first-class affected query without full GraphQL syntax
- Filtered phantom tasks from `affectedTasks` query results
- Fixed `$TURBO_ROOT$` inputs with `affectedUsingTaskInputs` and prevented `$TURBO_DEFAULT$` from matching unrelated root files
- Ensured `pkg#task` CLI args always included in filtered packages
- **Trend: Turborepo is building a complete task-level intelligence layer** -- affected, filtering, and querying all converging

**Logging Architecture Overhaul** (v2.8.18):

- New `turborepo-log` crate with TerminalSink and TuiSink backends
- User-facing warnings routed through new logging system
- Task error/warning output migrated to new system
- `GroupingLayer` for per-task output buffering
- Child process bytes routed through TaskHandle with sink-owned prefix rendering
- **This is infrastructure for better DX** -- structured logging enables better TUI, better error reporting, and eventually better CI output

**Cache Metadata Enrichment** (v2.8.18):

- Git SHA and dirty hash added to cache metadata
- Cache SCM metadata surfaced in Run Summary
- Enhanced remote cache status visibility in run prelude
- **This converges with Nx Cloud's cache debugging** -- knowing which commit produced a cache entry

**Graph & Visualization** (v2.8.18):

- Native SVG rendering for `--graph` output
- Deprecated graphviz-dependent formats (dot, png, pdf, etc.)
- **Removing the graphviz dependency** -- SVG is self-contained, better for CI artifacts and agent consumption

**Pruning & Lockfile Hardening** (v2.8.18):

- `globalDependencies` included in prune (behind future flag)
- Preserved all bun lockfile overrides during prune
- Fixed package.json key order during prune with pnpm patches
- Support for private GitHub repos in `turbo gen workspace --copy`

**Performance** (v2.8.18):

- Replaced subprocess repo index with in-process `gix-index` -- another subprocess-to-in-process migration
- **Trend**: Turborepo systematically eliminating subprocess calls (git, graphviz) in favor of native Rust implementations

**DX & Hardening** (v2.8.18):

- Deprecated `--parallel` flag
- LSP infers monorepo root instead of trusting editor `root_uri`
- Suppressed noisy rustls TLS warning on stderr
- Fell back to bundled certificates when system cert store inaccessible
- `TURBO_*` env vars passed through to child processes in Strict Mode
- Flushed stale mouse tracking events from stdin during TUI cleanup

### Strategic Implications for Nx

**Threat level: Medium-High (unchanged, broadening).**

The v2.8.14-v2.8.20 cycle marks a significant and accelerating strategic shift:

1. **Task-level intelligence is becoming a platform.** `affectedUsingTaskInputs` (v2.8.17) + `filterUsingTasks` (v2.8.18) + `turbo query affected` (v2.8.18) = Turborepo building a complete task-level intelligence layer. No longer a single feature behind a future flag -- coordinated multi-release strategy converging toward Nx's analysis capabilities. The moat is narrowing faster than expected.

2. **Cache metadata enrichment converges with Nx Cloud.** Git SHA + dirty hash in cache metadata, surfaced in Run Summary (v2.8.18) = building toward cache provenance/debugging. Territory Nx Cloud currently owns ("which commit produced this cache entry?").

3. **Logging overhaul signals DX investment.** `turborepo-log` crate with sink-based architecture (TerminalSink, TuiSink, per-task grouping) is infrastructure for better error reporting, TUI, and CI output. Foundational work.

4. **Systematic subprocess elimination.** `gix-index` replacing subprocess repo index (v2.8.18), native SVG replacing graphviz (v2.8.18), pure Rust worktree detection (v2.8.13). Methodically removing all external process dependencies.

5. **`turbo-ignore` deprecated for `turbo query affected`.** (v2.8.19) This completes the consolidation of Turborepo's intelligence APIs. `turbo-ignore` was a standalone CI skip tool; its replacement by the query engine means one API surface for affected analysis, CI skip, and workspace intelligence. Clean API design.

6. **OTel + AI agent telemetry maturing.** Delta temporality, cardinality control, `x-ai-agent` header (v2.8.15-v2.8.17). No new additions in v2.8.18-v2.8.20 but prior investments stabilizing.

7. **Release cadence is aggressive.** Eight stable releases (v2.8.13 through v2.8.20) in ~17 days. This is a sustained sprint.

**Nx should consider**:

- **Respond to task-level intelligence convergence**: Turborepo now has affected + filtering + querying at task level. Emphasize Nx's maturity and depth.
- **Cache debugging/provenance gap**: Git SHA in cache metadata is early-stage. Nx Cloud far ahead but gap closing.
- Publishing agent-friendly doc formats (`.md` routes, structured sitemap)
- Monitoring Turborepo's OTel path toward "Vercel Build Insights"

## Moon

**Latest stable**: v2.1.0 (March 16, 2026)
**Previous**: v2.0.4 (March 6), v2.0.3 (Feb 26), v2.0.0 (Feb 18)

### Key Changes in v2.1.0 (March 16)

**Affected Files Enhancements** (significant):

- `affectedFiles.filter` -- glob patterns to filter affected files before passing to tasks
- `affectedFiles.ignoreProjectBoundary` -- include workspace-relative files beyond project boundary
- `affectedFiles.passDotWhenNoResults` -- use `.` instead of empty value when no files match
- These are granular affected-file controls that Nx doesn't expose at this level

**Execution Plans** (from v2.1 blog -- not in GitHub release notes):

- `moon exec` now supports execution plans -- declarative JSON files configuring targets, affected filtering, graph traversal, and pipeline behavior
- Dependency targeting by scope: `^production:build` syntax to depend on build tasks in production-level project dependencies only
- These are higher-level orchestration abstractions that Nx doesn't have direct equivalents for

**Task System**:

- `runInSyncPhase` -- run tasks during `moon sync` commands (new lifecycle hook)
- Duplicate aliases no longer a hard error (first match wins)

**Toolchain Configuration**:

- `inheritAliases` (default `true`) -- toggle alias inheritance per toolchain
- `installDependencies` (default `true`) -- toggle dependency installation per toolchain

**Go Toolchain**: Now runs `go list --deps` for project relationship discovery (expanding polyglot graph)

**Python Toolchain**: PEP 503 name normalization, fixed package manager settings inheritance

**TypeScript**: `pruneProjectReferences` setting for syncing non-moon-managed references

**Environment**: Improved remote devbox detection (Codespaces, Gitpod)

**Fixes**: Invalid JSON schema in MCP `generate` tool, token substitution fixes, `bash` fallback to `sh`

### Key Changes in v2.0.4 (March 6)

- JavaScript: Support `*` versions for project relationships, reworked pnpm/yarn dedupe detection
- Python: Read `pyproject.toml` dependencies for project relationships
- Updated proto to v0.55.4, Rust to v1.94.0

### Key Changes in v2.0.0 (Feb 18) -- Previously Reported

- WASM plugin toolchain system replacing hard-coded platforms
- CLI overhaul, multi-format config, task system rework
- Remote caching stabilized, Docker improvements, VCS v2

### Strategic Implications for Nx

**Threat level: Low-Medium (unchanged).** Moon v2.1.0 shows healthy post-major-release velocity and is filling in features that matter for polyglot monorepos:

- The `affectedFiles` filtering options are more granular than Nx's affected analysis at the task-input level. Worth noting for DX comparison.
- Go dependency graph support shows Moon expanding its polyglot story.
- `runInSyncPhase` is a lifecycle concept Nx doesn't have (running tasks during sync/codegen phase).
- MCP JSON schema fix shows continued investment in AI integration.

Moon remains niche (small market share, painful v2.0 migration) but is technically innovative.

## Strategic Implications for Nx

### Immediate Actions

1. **Respond to Turborepo's consolidated query engine.** `turbo query affected` now replaces both the GraphQL query API and `turbo-ignore` (deprecated v2.8.19). Turborepo is consolidating all intelligence under one API: affected analysis, CI skip decisions, filtering. Nx should:
   - Publish content emphasizing the maturity and depth of Nx's affected analysis (years ahead)
   - Benchmark Nx's file-level precision vs Turborepo's emerging task-level approach
   - Highlight that Nx already has what Turborepo is building toward, with deeper integration
   - Note: `turbo-ignore` deprecation means Vercel users migrating to `turbo query affected` -- opportunity to show Nx's superior affected analysis during their migration

2. **Monitor cache provenance convergence.** Turborepo v2.8.18 adds git SHA + dirty hash to cache metadata and surfaces it in Run Summary. This is early-stage cache debugging/provenance. Nx Cloud is far ahead but should ensure the gap remains visible.
