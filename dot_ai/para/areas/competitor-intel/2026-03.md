### New Since Last Scan (v2.8.19, v2.8.20)

**`turbo-ignore` Deprecated** (v2.8.19 -- strategic):

- `turbo-ignore` officially deprecated in favor of `turbo query affected`
- Tailored deprecation notice for Vercel users (migration path from Vercel-specific CI skip to generic query engine)
- **This completes the turbo-ignore → turbo query affected migration.** `turbo-ignore` was Vercel's CI skip tool; replacing it with `turbo query affected` unifies the "what changed?" story under one query engine. Users now use `turbo query affected` for both local development and CI skip decisions.

**Package Binary Resolution Fixes** (v2.8.19):

- Detect scoped `@turbo/{platform}` packages in local binary resolution

### Strategic Implications for Nx

**Threat level: Medium-High (unchanged, broadening).**

The v2.8.14-v2.8.20 cycle marks a significant and accelerating strategic shift:

1. **Task-level intelligence is becoming a platform.** `affectedUsingTaskInputs` (v2.8.17) + `filterUsingTasks` (v2.8.18) + `turbo query affected` (v2.8.18) = Turborepo building a complete task-level intelligence layer. No longer a single feature behind a future flag -- coordinated multi-release strategy converging toward Nx's analysis capabilities. The moat is narrowing faster than expected.

2. **Cache metadata enrichment converges with Nx Cloud.** Git SHA + dirty hash in cache metadata, surfaced in Run Summary (v2.8.18) = building toward cache provenance/debugging. Territory Nx Cloud currently owns ("which commit produced this cache entry?").

3. **Logging overhaul signals DX investment.** `turborepo-log` crate with sink-based architecture (TerminalSink, TuiSink, per-task grouping) is infrastructure for better error reporting, TUI, and CI output. Foundational work.

4. **Systematic subprocess elimination.** `gix-index` replacing subprocess repo index (v2.8.18), native SVG replacing graphviz (v2.8.18), pure Rust worktree detection (v2.8.13). Methodically removing all external process dependencies.

5. **`turbo query` graduated to stable + everything converging on query engine.** (canary.6-7) `turbo query` promoted from experimental to stable. `turbo ls` rebuilt on query internals with `turbo query ls` shorthand. Combined with `turbo-ignore` deprecation (v2.8.19) and `turbo scan` deprecation (canary.3), **the query engine is now the single canonical API for all workspace intelligence**: affected analysis, filtering, listing, CI skip. This is a clean, consolidating API design -- a unified interface similar to how Nx's project graph underpins multiple commands.

6. **Remote cache provenance arriving.** (canary.9) Git SHA + dirty hash now sent to remote cache, not just stored in local metadata (v2.8.18). **Turborepo now has end-to-end cache provenance** -- local and remote. This directly attacks Nx Cloud's cache debugging moat.

7. **Structured logging emerging.** `--json` + `--log-file` flags (canary.2) build on the v2.8.18 logging overhaul. Machine-readable build output is infrastructure for Vercel dashboard integration, AI agent consumption, and third-party tooling.

8. **`global` configuration key.** (canary.1) New configuration surface behind future flag. Turborepo expanding its config model toward Nx's workspace-level/project-level hierarchy.

9. **OTel + AI agent telemetry maturing.** Delta temporality, cardinality control, `x-ai-agent` header (v2.8.15-v2.8.17). No new additions in v2.8.18-v2.8.20 but prior investments stabilizing.

10. **Release cadence is aggressive.** Eight stable releases (v2.8.13 through v2.8.20) in ~17 days, plus 9 canaries in 4 days. This is a sustained sprint.

**Nx should consider**:

- **Respond to task-level intelligence convergence**: Turborepo now has affected + filtering + querying at task level, and `turbo query` is officially stable (canary.6). Emphasize Nx's maturity and depth.
- **Cache provenance gap closing faster than expected**: Git SHA + dirty hash now sent to remote cache (canary.9), not just local. Turborepo has end-to-end cache provenance. Nx Cloud's cache debugging is still far ahead (input comparison, cache miss analysis) but the foundational data is now available on Turborepo's remote caches.
- **Structured output parity**: `--json` + `--log-file` flags signal Turborepo building machine-readable output for CI/agent consumption. Ensure Nx's structured output is competitive.
- **Configuration model expansion**: `global` config key signals Turborepo closing gaps with Nx's workspace config hierarchy.
- **Query engine consolidation narrative**: Turborepo's "everything is a query" consolidation (`turbo ls` on query internals, `turbo query affected`, `turbo query ls`) is a clean API design story. Nx should consider whether a similarly unified intelligence API surface would be compelling.
- Publishing agent-friendly doc formats (`.md` routes, structured sitemap)
- Monitoring Turborepo's OTel path toward "Vercel Build Insights"

**Task System**:

- `runInSyncPhase` -- run tasks during `moon sync` commands (new lifecycle hook)
- Duplicate aliases no longer a hard error (first match wins)

**Toolchain Configuration**:

- `inheritAliases` (default `true`) -- toggle alias inheritance per toolchain
- `installDependencies` (default `true`) -- toggle dependency installation per toolchain

**Go Toolchain**: Now runs `go list --deps` for project relationship discovery (expanding polyglot graph)

**Python Toolchain**: PEP 503 name normalization, fixed package manager settings inheritance

**TypeScript**: `pruneProjectReferences` setting for syncing non-moon-managed references

**Environment**: Improved remote devbox detection (Codespaces, Gitpod)

**Fixes**: Invalid JSON schema in MCP `generate` tool, token substitution fixes, `bash` fallback to `sh`

### Key Changes in v2.0.4 (March 6)

- JavaScript: Support `*` versions for project relationships, reworked pnpm/yarn dedupe detection
- Python: Read `pyproject.toml` dependencies for project relationships

**Threat level: Low-Medium (unchanged).** Moon v2.1.1 is a healthy bug fix release 4 days after v2.1.0, showing responsive maintenance. The v2.1.x series demonstrates:

## Strategic Implications for Nx

### Immediate Actions

1. **Respond to Turborepo's now-stable query engine.** `turbo query` is officially stable (canary.6). `turbo ls` rebuilt on query internals (canary.7). Combined with `turbo-ignore` deprecation (v2.8.19) and `turbo scan` deprecation (canary.3), the query engine is now the single canonical API for all workspace intelligence. This is no longer "emerging" -- it's the shipped, stable API. Nx should:
   - Publish content emphasizing the maturity and depth of Nx's affected analysis (years ahead)
   - Benchmark Nx's file-level precision vs Turborepo's task-level approach
   - Highlight that Nx already has what Turborepo is building toward, with deeper integration
   - Note: `turbo-ignore` deprecation means Vercel users migrating to `turbo query affected` -- opportunity to show Nx's superior affected analysis during their migration
   - **NEW**: Consider whether Nx should articulate a similar "unified intelligence API" narrative

1b. **Watch the structured logging play.** `--json` and `--log-file` flags (canary.2) + the v2.8.18 logging overhaul = Turborepo building a structured output pipeline. This is infrastructure for: (a) CI dashboard integration (Vercel), (b) AI agent consumption (machine-readable build output), (c) third-party tooling integrations. Nx should ensure its own structured output story is competitive.

1c. **Monitor `global` configuration.** The new `global` config key (canary.1) behind a future flag signals Turborepo is expanding its configuration model. This could close a gap with Nx's workspace-level configuration.

2. **Cache provenance is no longer early-stage -- it's end-to-end.** Turborepo v2.8.18 added git SHA + dirty hash to local cache metadata. Canary.9 now sends this to the remote cache. Turborepo has end-to-end cache provenance. Nx Cloud's cache debugging (input comparison, miss analysis) is still far ahead, but the foundational data layer is now in place on Turborepo's side. The gap is closing faster than previously assessed.

3. **Address the AI agent telemetry gap.** Turborepo instruments AI agent usage via `x-ai-agent` headers. Vercel accumulating data on agent usage patterns. Nx Cloud should consider similar instrumentation.

4. **Counter Turborepo's "agentic coding" narrative.** Agent Skills, AI detection telemetry, and `.md` doc routes remain a marketing gap. Ship agent-friendly documentation endpoints.
