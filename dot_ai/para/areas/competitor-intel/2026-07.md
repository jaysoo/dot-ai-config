## Vite+ (VoidZero / now Cloudflare)

> **Highest-priority section this cycle -- escalated threat.**

**Previous status:** Alpha, watch lightly.
**Current status:** Beta (July 2, 2026), Cloudflare-backed. Active competitive threat.

### Cloudflare Acquisition (June 4, 2026)

VoidZero announced it was joining Cloudflare on June 4, 2026. All team members joined Cloudflare. Key commitments:

- **Open source maintained:** Vite, Vitest, Rolldown, Oxc, and Vite+ remain open source, vendor-agnostic, and community-driven.
- **$1M Vite ecosystem fund** committed by Cloudflare, administered by the Vite core team.
- Strategic rationale from Cloudflare: "build the future of the AI-native web." Vite integration with Cloudflare Workers/Pages/D1/R2 expected.
- The acquisition removes the startup survival risk. Vite+ now has enterprise-scale backing.

### Vite+ Beta (July 2, 2026)

**CLI:** `vp` (Vite+)
**Latest stable:** v0.2.4 (July 8, 2026)
**Components:** Vite 8, Vitest, Rolldown, tsdown, Oxlint, Oxfmt + built-in task runner
**Stats:** 180+ fixes since alpha; 1,300+ public repos depend on vite-plus

Key features relevant to Nx/monorepo competition:

- **`vp run` -- Monorepo-aware task runner with intelligent caching.**
  Powered by `vite-task` (Rust, 453 stars, 548 commits), a standalone monorepo task runner that fingerprints input files to determine cache eligibility -- no explicit `inputs` configuration required. If inputs are unchanged, task is skipped instantly from cache.
- **Dependency-aware scheduling.** Workspace ordering derived from `package.json` dependency relationships, not a separate task graph config. This is a simplification vs. Nx (`project.json` targets) and Turbo (`turbo.json` pipeline).
- **`vp dev`:** Starts a dev server with HMR via Vite 8.
- **`vp check`:** Format (Oxfmt) + lint (Oxlint) + typecheck in one command.
- **Framework-agnostic, MIT licensed.** Works for CLIs, libraries, and web apps.
- No explicit monorepo config required for basic caching -- this is the differentiating DX claim.

**What `vite-task` does NOT yet have:**

- No stable release (pre-release state); the Vite+ beta bundles it but it has no independent versioned artifact yet.
- No distributed task execution / cloud cache (analogous to Nx Cloud or Turbo Remote Cache). Not yet announced.
- No equivalent to Nx generators or Moon templates for scaffolding.

### Nx implications

**Threat level: HIGH (escalated from MEDIUM/WATCH).**

1. **The "will it survive?" concern is resolved.** Cloudflare backing eliminates the startup risk that made Vite+ easy to dismiss. It will keep shipping.
2. **Automatic input fingerprinting is a compelling DX story.** Nx requires explicit `inputs` configuration for fine-grained cache correctness. Vite+ claims to infer them automatically. If this works in practice for complex workspaces, it removes a common Nx friction point. **Recommended: evaluate vite-task's fingerprinting behavior on a mid-size Nx workspace and document where it falls short.**
3. **Vite+ targets the same "start a JS project" audience as `create-nx-workspace`.** The unified `vp` CLI (runtime + package manager + build + test + lint + task runner) is a compelling zero-config entry point. Nx's entry point requires more configuration to reach the same productivity baseline. **This is a positioning threat at the top of the funnel.**
4. **`vite-task` is Rust-based, matching Nx's own native approach.** Performance parity is plausible; benchmarks will matter.
5. **No cloud/remote cache yet.** This is Nx's window. Vite+ users who scale will need remote caching and DTE. Nx Cloud can capture that upgrade path if positioned early.
6. **Cloudflare integration risk.** Vite+ + Cloudflare Workers + Cloudflare R2 (cache storage) is a natural product bundle. If Cloudflare ships a hosted remote cache backed by R2 and integrated with their developer platform, it competes with Nx Cloud in a Cloudflare-centric stack.
7. **Enterprise features in beta.** Org templates and proxy-aware HTTP in v0.2.0 signal enterprise positioning. Vite+ is not just a developer tool -- it is building enterprise features from day one of beta.

**v2.10.0 (June 26, 2026) -- New minor: local cache eviction, deferred input hashing**

- **Local cache eviction:** New `cacheMaxAge` and `cacheMaxSize` turbo.json options. Nx has no equivalent local cache size controls.
- **Deferred input hashing:** `inputs` entries can now be structured objects that defer file hashing until upstream task dependencies have completed.
- **Composable `--affected` and `--filter`:** These flags now combine, allowing filtered + affected intersection queries.
- **TUI improvements:** Toggle between interactive TUI and streamed log output; click rows in the TUI task list to select tasks; auto-copy selection to clipboard on mouse release.
- **Graceful shutdown for watch mode:** Multiple reliability fixes for `turbo watch`.

**v2.10.1, v2.10.2, v2.10.3 (June 28 - ~July 3)**

- Transitive dependency fixes in engine graph pruning.
- Lockfile preservation during pruning for npm and pnpm.

- Additional TUI stability; Windows environment variable fixes.

### Released this period -- v2.10.4 (July 6, 2026)

**v2.10.4 (July 6, 2026) -- Cargo workspace support, performance**

- **Cargo workspace support (NEW ECOSYSTEM):** Turborepo now discovers Cargo crates as packages, enables task execution via `cargo`, hashes per-crate external dependencies, and supports pruning and watch mode for Cargo workspaces. This extends Turbo into Rust monorepos.
- **mimalloc as global allocator:** Performance improvement for memory-intensive operations.
- **Hardware-accelerated SHA-1 for git blob hashing:** Faster change detection on modern hardware.
- **Single-pass yarn v1 and Berry lockfile parsing:** Reduces lockfile parse overhead.
- **Parallelized pnpm lockfile section parsing and workspace discovery.**
- **`turbo query` now exposes resolved `experimentalCI` task config.** The undocumented `experimentalCI` object is now queryable -- first public surface area for this feature.
- **`@turbo/repository` package** (renamed from napi package): Signals a stable public API for programmatic workspace access.
- Bug fixes: bun ancestor dependencies after prune, non-reentrant libc calls in shutdown, raises open-file soft limits at startup.

**Canary progress (v2.10.5-canary.1 through canary.6, July 7-12)**

Active canary development continues; no single breakthrough feature visible yet.

- **Performance:** Deferred hashing, mimalloc, hardware SHA-1, parallelized pnpm parsing

### Released (July 6 report) -- v2.3.5

**v2.3.5 (June 26, 2026) -- Toolchain + remote cache fix**

- Deno v2.9 support.
- Docker pruning fix: projects with the same name/ID as their underlying toolchain package.
- Remote caching fix: remote cache writes were not updating unless `casOutputsCache` experiment was enabled.

### Released this period -- v2.4.0 through v2.4.3

**v2.4.0 (July 5, 2026) -- MAJOR: CAS cache, git hardening, Ruby, Poetry, daemon overhaul**

This is the most significant Moon release since v2.3.0 (June 1). Key features:

- **CAS Output Cache -- graduated beyond experiment:**
  - New storage API enabling true local/remote cache interoperability (same format, same content-addressable semantics).
  - OS reflink support (CoW file copies on supported filesystems -- zero-copy cache restores).
  - Cross-cache blob retrieval: a cache hit in remote can warm the local cache.
  - Local cache warming on remote hit -- automatic (was manual before).
  - **This is the "local/remote cache format unification" claim Moon can now make in full.** The CAS experiment from v2.3.0 (June 1) is no longer an experiment -- it is the cache layer.
- **Tasks -- new `checks` setting:**
  - Types: `fingerprint` (run only if inputs changed), `requirement` (dependency preconditions), `condition` (environment/flag gates).
  - `mergeChecks` option for inheritance control.
  - Top-level `taskOptions` in `moon.*` for workspace defaults.
- **VCS -- git command hardening (security):**
  - Argument injection prevention (same class as Turbo v2.9.18, June 10).
  - Credential prompt fail-fast (prevents hangs on interactive git prompts in CI).
  - `fsmonitor` disabled (avoids race conditions in watch mode).
  - Reworked merge base resolution and rename handling.
- **Ruby -- new unstable toolchain:**
  - Bundler support; tiers 1-3.
  - Moon now supports: JS (Node/Deno/Bun), Go, Python, Rust, Ruby.
- **Python -- Poetry package manager support** (unstable). Previously only pip/uv.
- **Daemon -- major overhaul:**
  - Log file rotation (up to 7 files).
  - Exclusive advisory file lock.
  - Daemon metadata in `daemon.json` (replaces `moond.pid`).
  - Combined connect+start operation.
  - Version-mismatch auto-restart.
  - Idle self-retirement.
  - Background webhook/output archiving.
- Processes: Byte-level output streaming (was line-level) -- lower latency for large outputs.
- Remote cache: Compression support for blob streams.
- Updated proto to v0.58.2 from v0.57.4.

**v2.4.1 (July 6, 2026) -- Daemon startup fix**

- Fixed daemon not starting if cache directory did not exist.
- Fixed some error messages being swallowed in logs.

**v2.4.2 (July 8, 2026) -- Workspace graph caching**

- Reworked workspace graph caching to avoid plugin calls on cache hits -- significant performance improvement in large workspaces.
- Go: Fixed `go list -deps` running on non-Go projects.

**v2.4.3 (July 12, 2026) -- CI reliability + Renovate**

- **Renovate support added.** Dependency update automation is now integrated.
- Updated BitBucket codeowners to new syntax/file location; `bitbucket-legacy` for old syntax.
- Fixed `runDepsInParallel: false` not serializing the full dependency subtree.
- Fixed `moon ci` failing when base branch is provided as a fully-qualified `refs/heads/<branch>` ref.
- Fixed task binaries failing with "command not found" in CircleCI (BASH_ENV PATH overwrite issue).
- Fixed read-only output files failing on cache hit when `casOutputsCache` experiment is enabled.
- Fixed non-UTF-8 bytes in captured task output being discarded (now replaced with `�`).
- Fixed `moon query changed-files` including uncommitted local index changes when an explicit `--head` revision was provided.
- Fixed VCS hooks being placed in `.moon/hooks` instead of alongside config when using `.config/moon`.

### Key changes by category (v2.4.x)

- **Features:** CAS cache (stable, no longer experiment), task checks, Ruby toolchain (unstable), Poetry (unstable), Renovate integration
- **Performance:** Workspace graph caching avoids plugin calls on hits; OS reflink for zero-copy restores; byte-level streaming
- **DX:** Renovate support, CircleCI fix, BitBucket codeowners update
- **Ecosystem:** Ruby tiers 1-3, Python Poetry, polyglot story strengthened
- **Security:** Git argument injection prevention, credential prompt fail-fast
- **AI:** No new MCP tools this cycle; existing `get_template`/`get_templates` carry forward

### Nx implications

**Threat level: HIGH (accelerated).**

1. **CAS cache is no longer an experiment -- it is the cache layer.** Moon v2.4.0 delivers on the June prediction. The claim: same content-addressable semantics locally and remotely; local cache warms from remote hits automatically; OS reflink enables zero-copy restores on modern filesystems. Nx's local cache does NOT have these properties. **This is a concrete, marketable differentiator for Moon.**
2. **Moon's daemon is now arguably better than Nx's.** Version-mismatch auto-restart, idle self-retirement, exclusive advisory lock, log rotation -- Moon's daemon is becoming production-grade infrastructure.

4. **Renovate support in v2.4.3** closes a practical workflow gap. First-class dependency update automation is a real-world differentiator.
5. **Ruby toolchain (unstable) strengthens polyglot story.** Moon spans JS/TS, Go, Python, Rust, Ruby. Turbo now adds Rust (v2.10.4). Nx has deeper JS/TS story but is falling behind on polyglot breadth.
6. Carry-forward: MCP generator discoverability gap; native file hashing 10-50% benchmark claim.

## Cross-Competitor Themes (updated July 13, 2026)

1. **Vite+ exits the "watch passively" tier.** Cloudflare backing + beta = this is now a real product with real funding. Three funded JS/TS monorepo tools compete for the same audience: Nx, Turborepo, Vite+.

2. **Automatic vs. explicit task configuration is the new DX battleground.** Vite+ claims zero-config input fingerprinting. Turborepo introduced deferred input hashing to reduce explicit config burden. Nx requires the most explicit configuration of the three. Nx needs either better inference or a stronger counter-narrative.

3. **Local cache management is a Turbo differentiator; CAS cache is a Moon differentiator.** `cacheMaxAge`/`cacheMaxSize` (Turbo) and local/remote CAS unification (Moon) both attack Nx's local cache story from different angles.

4. **Moon v2.4 delivered on CAS: local/remote cache format unification is real.** The June "watch v2.4" prediction materialized on July 5. Moon's local and remote cache now share the same content-addressable format, with automatic local warming and OS-level reflink for zero-copy restores. This is a concrete differentiator Nx cannot currently counter.

5. **Two of three JS competitors have now hardened VCS git calls.** Turbo (v2.9.18, June 10) and Moon (v2.4.0, July 5) both patched argument injection in git operations. The vulnerability class is confirmed-real. Nx audit is 5+ weeks overdue.

6. **Turborepo enters the Rust ecosystem.** v2.10.4 (July 6) adds first-party Cargo workspace support. Turbo is no longer JS/TS-only. For polyglot shops with Rust components, Turbo is now a viable option.

7. **Cloudflare + Vite+ = potential remote cache threat.** Cloudflare R2 is a natural backend for a hosted task cache. No announcement yet, but the product surface is obvious.

8. **Turbo 3.0 appears abandoned in favor of 2.x continuations.** 15+ weeks since March 25 "preparing for 3.0" blog. v2.10 is the milestone. Remove from watch queue.

### Where Nx Is Behind

- **Local CAS cache:** Moon v2.4.0 graduates CAS from experiment to core feature. Local/remote cache format unification, OS reflink (zero-copy restores), and cross-cache blob retrieval with local warming are all live. Nx has no equivalent.
- **Zero-config input inference:** Vite+ claims automatic fingerprinting with no `inputs` config needed. Most significant new DX gap.
- **MCP generator discoverability:** Moon's `get_template`/`get_templates` tools target the AI code-generation workflow. Nx generators are not yet MCP-queryable with schema introspection.
- **Cache security audit:** Two competitors have now patched git argument injection and cache path traversal. Nx audit of same classes is 5+ weeks outstanding (from June 10).
- **Cargo/Rust workspace:** Turbo v2.10.4 adds first-party Cargo support. Moon has Rust toolchain support. Nx is the only major JS monorepo tool without first-party Rust workspace support.

### Where Nx Is Ahead

- **Distributed Task Execution:** No competitor ships DTE. Turbo's `experimentalCI` is now surfaced via `turbo query` but still undocumented.
- **Nx Cloud:** Remote caching with security controls, observability, and managed CI orchestration. Vite+ has no cloud product. Moon has no cloud product.
- **Nx Console:** Turbo contracted scope (removed web UI). Vite+ has no IDE extension. Nx Console is the richest IDE-integrated monorepo experience.
- **Generator ecosystem breadth:** Nx generators vastly exceed Moon's templates and Vite+'s current scaffold story.

### Unassailable Moats

- **DTE + Nx Cloud:** Architectural decision requiring distributed coordination infrastructure. No competitor can replicate with a single release.
- **Workspace generator ecosystem:** Decade of community plugins and official generators.
