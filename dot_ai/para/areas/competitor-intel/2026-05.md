### proto v0.57 (May 12) -- OCI distribution + new backends

_(proto is moon's companion version manager, not moon itself -- but tightly coupled to the moon ecosystem)_

- **OCI plugin distribution**: Migrated official proto plugins from raw GitHub release .wasm files to OCI artifacts on ghcr.io. Eliminates GitHub CDN flakiness (download retries, timeouts) in the plugin resolution path. Directly addresses a reliability complaint in the moonrepo community.
- **cargo backend**: Install CLIs from crates.io via `cargo install` -- proto can now manage Rust-ecosystem tools
- **npm backend**: Install CLIs from npm registries -- proto can now manage JS-ecosystem tools without requiring Node to be separately bootstrapped

**Threat level: Medium-High (unchanged)**

2. **proto OCI distribution is infrastructure investment**, not user-facing feature. Signals moonrepo is treating the tools ecosystem (moon + proto) as a long-term platform, not just a build runner.
3. **proto cargo/npm backends** narrow the gap between proto and `mise` / `nvm` / `volta`. If Nx workspace setups commonly recommend one of those for tool management, proto is now a credible single-tool alternative for moon shops. Not a direct Nx threat, but relevant to the workspace management story.
4. **The `debug-task` AI skill (v2.2) remains the most significant unreplicated feature** since the May 4 scan. Nothing moon shipped in May changed that assessment.

## Cross-Competitor Themes (delta vs May 4)

1. **Security is now a universal theme.** Turborepo had 3 CVEs in one release (May 14). Turbo 2.9.9 canaries (May 4, now stable) hardened pipeline credentials and OG image signing. Bazel 8.7.0 added SOCKS proxy support partly for enterprise-network security. Pants cherry-picks Trivy and Pex security patches into LTS lines. **Every tracked competitor is treating supply-chain and auth security as table stakes.** Nx should ensure equivalent maturity in its VSCode extension, CLI auth flows, and release pipeline signing.
2. **IDE/editor integration is becoming a competitive front.** Turbo's v2.9.11 VSCode push (turbo.jsonc, $TURBO_EXTENDS$ LSP support, clickable log paths) signals Vercel sees the IDE plugin as a retention surface. Bazel has been investing in IDE integration for years. Nx Console is strong but should be benchmarked against Turbo's current VSCode feature set.
3. **Friction-reduction onboarding remains the 3-of-5 pattern** (unchanged from May 4). Turbo OAuth (stable May 1), Gradle --develocity-url (stable April 28), Nx Cloud. No new entrants; the gap between "connected" and "intelligent" is now where differentiation lives.
4. **AI features: no new entries since Moon's v2.2 `debug-task`** (April 12). None of the May releases from any competitor added AI-native capabilities. The window for Nx to establish AI-debug leadership over Moon is still open.
5. **Pants Rust rewrite is now testable** (`pants_ng` mode in 2.33 dev). Not a near-term threat but worth checking in again in Q3 2026 when it may hit RC.

**High urgency:**

1. **Audit Nx VSCode extension + CLI auth surfaces for the CVE classes Turbo patched.** Turbo's May 14 release patched command injection (VSCode extension), login callback CSRF, and local code execution during package manager detection. These are generic attack patterns that any build tool with a VSCode plugin and auth flow is susceptible to. Check Nx Console and `nx login` flow equivalents.
2. **Decide on AI-debug skill response to Moon's `debug-task`.** (Unchanged from May 4 -- still no action from Nx side confirmed.) Moon shipped first; the window is closing.

**Medium urgency:**

3. **Benchmark Nx Console against Turbo's v2.9.11 VSCode feature set.** Specific gaps to check: JSONC config support, config composition awareness in LSP, clickable log paths, relative path resolution. File gaps as Console issues if found.

6. **Watch for Turbo 2.10 or 3.0 RC.** No 3.0 tag; March blog signaled it. Window for Nx counter-positioning still open.
7. **Watch Moon for v2.3.** v2.2 stabilization complete (two bug-fix releases). v2.3 likely in June based on prior cadence.
