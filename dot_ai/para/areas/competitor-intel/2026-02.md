## Turborepo

**Versions in window**: v2.7.5 through v2.8.12 (15 stable releases)
**Current**: v2.8.12 (2026-02-27)
**Source**: `gh release list --repo vercel/turborepo`, turborepo.dev/blog

### v2.8.0 (2026-01-28) — Major Feature Release

**AI / Agentic Workflows** (This is their top investment area)
- **Agent Skill**: `npx skills add vercel/turborepo` gives AI agents expertise in monorepo patterns. This is a new category of tooling.
- **AI-enabled docs**: Every docs page has a `.md` route for agents, machine-readable sitemap at `/sitemap.md`, versioned docs on subdomains (e.g. `v2-7-6.turborepo.dev`), and an "Ask AI" chat.
- **`turbo docs` subcommand**: Search documentation from the terminal.
- **Task descriptions**: New `description` field in `turbo.json` provides context for humans and AI.

**Git Worktrees**: Local cache shared across worktrees so parallel agent sessions get cache hits.

**Composable Configuration** (started in 2.7): Package configs can `extends` from any workspace package, not just root. `$TURBO_EXTENDS$` keyword for appending to inherited arrays.

**Stats**: 18 features, 50 fixes in 2.8.0 alone.

### v2.7.5–v2.7.6 (in window, 2025-12-29+)

- **Devtools**: `turbo devtools` visual Package Graph and Task Graph explorer with hot-reload.
- **Biome rule**: `noUndeclaredEnvVars` in Biome 2.3.10+ for linting unsafe env vars.
- **Yarn 4 Catalogs**: Lockfile parser now understands Yarn 4.10 catalogs.
- Polyrepo support: removed root app package requirement.
- OOM-killed task handling improved.
- `errorsOnlyShowHash` future flag added.

### v2.8.1–v2.8.12 (patch releases through 2026-02-27)

Rapid cadence of bug fixes and incremental improvements. No major new features.

### Nx Relevance

- **HIGH**: Turborepo is making AI/agent-first a core differentiator. The agent skill, `.md` docs routes, and `turbo docs` CLI are ahead of what Nx offers. Nx should consider similar AI-friendly docs and agent tooling.
- **MEDIUM**: Git worktree cache sharing is relevant for CI parallelism. Nx Cloud already handles this via remote cache, but local worktree support could matter for developer workflows.
- **MEDIUM**: Composable configuration via `extends` is converging with Nx's targetDefaults pattern.
- **LOW**: Devtools is nice but Nx already has `nx graph`.

## Moon

**Versions in window**: v2.0.0-alpha.3 through v2.0.3
**Current**: v2.0.3 (2026-02-26)
**Source**: `gh release list --repo moonrepo/moon`, moonrepo.dev/blog

### v2.0.0 "Phobos" (2026-02-18) — MAJOR RELEASE

This is Moon's biggest release ever. Multi-year effort culminating in a full v2 with extensive breaking changes.

**WASM Plugin System** (Biggest architectural change)
- Replaced all hard-coded platform/toolchain support with WASM plugins.
- Community can now write toolchain plugins for any language/tool.
- Plugin APIs: `define_extension_config`, `extend_command`, `extend_project_graph`, `extend_task_command`, `extend_task_script`, `sync_project`, `sync_workspace`.
- Updated wasmtime to v37.

**Configuration Overhaul**
- Now supports JSON, JSONC, TOML, HCL, Pkl, and YAML (was YAML-only).
- Renamed `.moon/toolchain.yml` to `.moon/toolchains.yml` (plural).
- Added `.moon/extensions.*` config file.
- Added `inheritedBy` for configuration-based task inheritance.
- Many settings renamed/removed (see migration guide).

**CLI Restructuring**
- New commands: `moon exec`, `moon extension`, `moon hash`, `moon projects`, `moon tasks`, `moon query affected`, `moon template`.
- All options/flags now kebab-case (was camelCase).
- `moonx` stabilized.
- Supports `...` in task targets (Bazel-style `**/*` alias).
- `.config/moon` supported alongside `.moon`.

**Task System**
- `command`/`args` now simple commands only. Use `script` for compound commands.
- Deep merging for task inheritance (was shallow).
- `.env` loading moved to before execution (was at creation).
- Env var substitution reworked.

**Remote Cache**: All settings stabilized. gzip/zstd compression for HTTP.

**Performance**: Transitive reduction on action graph. Better parallelism with target resolution before partitioning.

**MCP**: Protocol updated to 2025-11-25. Added `generate` tool. Reduced payload sizes for `get_projects`/`get_tasks`.

**Migration tooling**: Includes `Migrate Nx` and `Migrate Turborepo` extensions for converting from competitors.

### v2.0.1 (2026-02-20)

- Fixed upgrade command for new distribution format.
- WASM serialization fixes.

### v2.0.3 (2026-02-26)

- Temporarily relaxed shallow checkout errors in CI.
- Docker prune logging improvements.
- Fixes for graph visualizer, shell task handling, CI run checks.

### Nx Relevance

- **HIGH**: Moon's WASM plugin architecture is a significant investment that could attract a plugin ecosystem. Nx's current plugin system is TypeScript-based generators/executors. The WASM approach is more language-agnostic.
- **HIGH**: Moon v2 includes explicit "Migrate Nx" tooling. They are actively targeting Nx users for conversion.
- **MEDIUM**: MCP integration puts Moon ahead on AI tool integration alongside Turborepo.
- **MEDIUM**: Multi-format config (JSON, TOML, HCL) is a DX win that Nx doesn't offer.
- **LOW**: The Bazel-style `...` target syntax is a nod to the Bazel ecosystem.

## Bazel

**Versions in window**: 8.5.1 (2026-01-12), 6.6.0 (2026-01-21), 9.0.0 (2026-01-20), 8.6.0 (2026-02-26)
**Current LTS**: 8.6.0 | **Current Major**: 9.0.0
**Source**: `gh release list --repo bazelbuild/bazel`

### 9.0.0 (2026-01-20) — MAJOR RELEASE

Bazel 9 is a landmark release that completes several multi-year migration efforts.

**WORKSPACE Removal** (Most impactful change)
- Bzlmod is now ALWAYS enabled. All WORKSPACE logic removed.
- `--enable_bzlmod` and `--enable_workspace` are no-ops.
- `bazel sync` removed; use `bazel fetch --all`.
- Migration tool available at bazel.build/external/migration_tool.

**Starlarkification Complete**
- All C++ rules removed from Bazel core, now in `rules_cc`.
- `--incompatible_autoload_externally` defaults to empty string.
- All language-specific rules must be loaded from their respective modules.

**Performance**
- Merkle tree rewrite: up to **30% wall time** and **70% peak heap** reduction for remote caching/execution.
- NestedSet memoization replaced with weak references: **20% retained heap**, **15% peak heap** reduction.
- Sandbox output copying up to **80% faster** for tree artifacts.
- Idle garbage collection for persistent action cache.
- Stale install bases also garbage-collected.

**Remote Execution**
- Zstd, Deflate, Snappy encoding for HTTP remote caches.
- `--remote_max_concurrency_per_connection` flag.
- Repository rule results can now be cached in remote cache (`--experimental_remote_repo_contents_cache`).

**Build Language**
- `set` data type added to Starlark.
- Legacy `struct` providers removed.
- Symbolic macro computation limits enforced.
- `attr.label_list_dict` type added.
- C++20 modules introductory support.

**External Dependencies**
- `REPO.bazel` supports `ignore_directories()`.
- `--inject_repository` flag for adding repos via CLI.
- `load_wasm` and `execute_wasm` methods added (experimental).
- Module mirrors via `--module_mirrors`.
- Python wheel `.whl` extraction support.

**Configurability**
- `config.none()` transition for data dependencies.
- `exec_group_compatible_with` for execution platform constraints.
- Starlark flag `scope` for target vs universal configuration.
- `executables()` function in query/cquery/aquery.

**Misc**
- macOS output base now defaults to `$HOME/Library/Caches/bazel`.
- `BAZELRC` environment variable for `.bazelrc` path.
- `--quiet` startup option.
- BEP now includes critical path time.

### 8.6.0 (2026-02-26) — LTS Patch

- Test result caching suppression via `--test_summary=short_uncached`.
- TLS failover to mirrors for repository downloader.

- `bazel mod show_repo` gains `--all_repos` and `--all_visible_repos`.

### 8.5.1 (2026-01-12) — LTS Patch

### 6.6.0 (2026-01-21) — Legacy LTS Patch

Maintenance release for 6.x LTS track.

### Nx Relevance

- **INFORMATIONAL**: The Merkle tree performance improvements (30% wall time, 70% heap) set a high bar for remote cache implementations. Nx Cloud should be aware of these benchmarks.
- **INFORMATIONAL**: Bazel adding WASM execution in repository context (`load_wasm`, `execute_wasm`) mirrors Moon's WASM plugin direction. This may become a pattern.

## Convergence/Divergence with Nx

### Converging
1. **Composable task configuration**: Turborepo's `extends` pattern is approaching Nx's targetDefaults.
2. **Remote caching stabilization**: Moon stabilized remote cache; all tools now treat this as table stakes.
3. **Graph visualization**: Turborepo devtools joins Nx graph as standard tooling.

### Diverging
1. **AI/Agentic workflows**: Turborepo is ahead. Agent skills, `.md` docs routes, `turbo docs` CLI are a new category. Nx has nothing comparable.
2. **WASM plugins**: Both Moon and Bazel are investing in WASM for extensibility. Nx's TypeScript plugin model is simpler but less language-agnostic.
3. **Self-contained distribution**: Pants scies, Moon's binary distribution -- trend toward zero-dependency executables that Nx (requiring Node.js) doesn't follow.

## Action Items for Nx

### Urgent (respond within 1-2 sprints)
1. **AI-friendly documentation**: Turborepo's `.md` routes, machine-readable sitemap, and versioned doc subdomains are a direct competitive threat for AI-assisted development. Nx docs should add similar machine-readable endpoints.

### Monitor
5. **WASM plugin ecosystem**: If Moon or Bazel WASM plugins gain traction, Nx may need to consider a more polyglot plugin system.
6. **Moon's "Migrate Nx" extension**: Track adoption and what pain points it targets.
7. **Self-contained binary distribution**: The trend toward no-runtime-dependency executables (Pants scies, Moon binaries) could matter if Node.js becomes a barrier.
