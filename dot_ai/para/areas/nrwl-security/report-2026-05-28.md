# Nrwl Security Vulnerability Report

**Report date:** 2026-05-28  
**Scope:** nrwl/nx · nrwl/ocean · nrwl/nx-labs · nrwl/nx-console · public nx npm packages  
**Sources:** GitHub Security Advisories, NVD, socket.dev, Snyk, CISA KEV, Nx official postmortems, vendor blogs

---

## Executive Summary

Two major **supply chain attacks** hit the Nx ecosystem within the past nine months — one targeting npm packages (August 2025), one targeting the VS Code extension (May 2026). Both are now patched, but teams that installed affected versions during the exposure windows must **treat those machines as fully compromised and rotate all credentials**.

Beyond the supply chain incidents, several **dependency-level CVEs** remain open or were recently patched across the repos, most notably in `undici`, `axios`, `tar-fs`, `storybook`, and `koa` (transitive via `@nx/module-federation`).

### Critical Action Items (as of 2026-05-28)

1. **Any developer who had `nrwl.angular-console@18.95.0` installed on 2026-05-18 between 12:30–13:09 UTC must rotate all credentials immediately.** (CVE-2026-48027 — added to CISA KEV)
2. **Any developer/CI that installed nx 20.9.0–20.12.0 or 21.5.0–21.8.0 on or after 2025-08-26 must rotate all credentials.** (CVE-2025-10894)
3. **Upgrade `undici` in `nrwl/ocean`** to ≥ 6.24.0 to close four active 2026 HIGH CVEs.
4. **Upgrade `tar-fs` in `nrwl/nx-labs`** to ≥ 2.1.3 (CVE-2025-48387).
5. **Upgrade `storybook` in `nrwl/nx-console`** to ≥ 10.1.10 and `@storybook/addon-docs` to ≥ 9.1.17 (CVE-2025-68429).
6. **Verify MongoDB server** version in ocean production is patched against MongoBleed (CVE-2025-14847).

## Part 1 — Supply Chain Incidents

### Incident 1 — S1ngularity npm Package Compromise

| Field | Value |
|---|---|
| **CVE** | CVE-2025-10894 (primary); also CVE-2025-59037, -59141 through -59144, -59162, -59330, -59331 |
| **GHSA** | [GHSA-cxm3-wv7p-598c](https://github.com/advisories/GHSA-cxm3-wv7p-598c) |
| **Severity** | **CRITICAL** |
| **Date** | 2025-08-26 22:32 UTC — 2025-08-27 02:44 UTC (~4 hours exposure) |
| **Root Cause** | GitHub Actions `pull_request_target` workflow with unsanitized PR title input allowed command injection → npm token exfiltration → malicious package publishing |

**What happened:** Attackers crafted a pull request against an outdated branch of `nrwl/nx` that still contained a vulnerable `pull_request_target` workflow (GHSL-2025-111). The injected shell payload extracted the npm publishing token; the attacker then published malicious versions of 8 packages containing a postinstall hook (`telemetry.js`) that:

- Scanned the filesystem for SSH keys, GitHub/npm tokens, `.env` files, API keys, and crypto wallets
- Exfiltrated everything by creating public GitHub repositories (`s1ngularity-repository-*`) under victim accounts
- Modified `~/.zshrc` and `~/.bashrc` to execute shutdown commands
- **For the first time in any known attack:** co-opted installed AI CLIs (Claude Code `--dangerously-skip-permissions`, Gemini `--yolo`, Amazon Q) for automated reconnaissance

Over **2,349 distinct secrets** were leaked. In March 2026, threat actor **UNC6426** was documented using S1ngularity-harvested credentials to gain AWS administrator access within 72 hours.

**Directly compromised packages:**

| Package | Malicious Versions |
|---|---|
| `nx` | 20.9.0, 20.10.0, 20.11.0, 20.12.0, 21.5.0, 21.6.0, 21.7.0, 21.8.0 |
| `@nx/devkit` | 20.9.0, 21.5.0 |
| `@nx/js` | 20.9.0, 21.5.0 |
| `@nx/workspace` | 20.9.0, 21.5.0 |
| `@nx/node` | 20.9.0, 21.5.0 |
| `@nx/eslint` | 21.5.0 |
| `@nx/key` | 3.2.0 |
| `@nx/enterprise-cloud` | 3.2.0 |

**Status:** All malicious versions removed from npm. Nx migrated to npm Trusted Publishers (OIDC) post-incident. **Current 22.7.x series is clean.**

**Remediation checklist:**
- [ ] Verify no malicious version is in lock files or caches (`npm cache clean --force` / `yarn cache clean --all` / `pnpm store prune`)
- [ ] Search CI logs for installs on 2025-08-26 or 2025-08-27 of any listed version
- [ ] Check for `/tmp/inventory.txt` and GitHub repos named `s1ngularity-repository*` in victim accounts
- [ ] Rotate: GitHub tokens, npm tokens, AWS keys, SSH keys, `.env` secrets accessible at install time
- [ ] Audit `~/.zshrc` and `~/.bashrc` for unexpected entries

### Incident 2 — Nx Console VS Code Extension 18.95.0

| Field | Value |
|---|---|
| **CVE** | CVE-2026-48027 |
| **GHSA** | [GHSA-c9j4-9m59-847w](https://github.com/nrwl/nx-console/security/advisories/GHSA-c9j4-9m59-847w) |
| **Severity** | **CRITICAL** (CVSS 4.0: 9.3) |
| **Date** | 2026-05-18 12:30–13:09 UTC (~18 min VS Marketplace, ~36 min OpenVSX) |
| **CISA KEV** | Added 2026-05-27 |
| **Root Cause** | Upstream TanStack supply chain compromise (`@tanstack/zod-adapter@1.166.15`, 2026-05-11) poisoned a contributor's dev machine; attacker stole their GitHub CLI token and published the trojanized extension |

**What happened:** Threat actor **TeamPCP** (also tracked as UNC6780) published a trojanized build of the Nx Console VS Code extension. The malicious version activated silently on any workspace open, fetched a 498 KB obfuscated payload from a **dangling commit inside the official `nrwl/nx` GitHub repository**, then executed credential harvesters targeting:

- GitHub tokens (keychain, config, and process memory)
- npm tokens
- AWS credentials and IAM roles
- HashiCorp Vault tokens (`~/.vault-token`)
- Kubernetes cluster credentials
- SSH private keys
- `.env` files on disk
- 1Password vault contents
- GCP credentials and Docker configs
- **Anthropic Claude Code configuration** (`~/.claude/settings.json`)

Exfiltration used three channels: HTTPS to attacker infrastructure, GitHub API, and DNS covert channels. A persistence backdoor was dropped at `~/.local/share/kitty/cat.py`. This attack subsequently led to the breach of **~3,800 GitHub internal repositories** and impacted GitHub, OpenAI, and Grafana.

**Affected version:** `nrwl.angular-console` v18.95.0 only  
**Estimated installs during window:** ~6,000+  
**Fixed version:** v18.100.0 and later

**Remediation checklist:**
- [ ] Update Nx Console to 18.100.0+ immediately
- [ ] If 18.95.0 was installed during the exposure window, treat machine as fully compromised
- [ ] Rotate: GitHub PATs, npm tokens, AWS keys, GCP credentials, SSH keys, Vault tokens, Kubernetes credentials, 1Password, Anthropic API keys, `.env` secrets
- [ ] Revoke GitHub CLI access (`gh auth logout`) and re-authenticate
- [ ] Check for `~/.local/share/kitty/cat.py` persistence artifact; remove if present
- [ ] Consider full system rebuild if machine is used for production deployments

**References:**
- [Nx Console v18.95.0 Postmortem](https://nx.dev/blog/nx-console-v18-95-0-postmortem)
- [GHSA-c9j4-9m59-847w](https://github.com/nrwl/nx-console/security/advisories/GHSA-c9j4-9m59-847w)
- [The Hacker News — Nx Console Compromise](https://thehackernews.com/2026/05/compromised-nx-console-18950-targeted.html)

### Incident 3 — Axios npm Supply Chain Compromise (North Korea / Sapphire Sleet)

| Field | Value |
|---|---|
| **Severity** | **CRITICAL** |
| **Date** | 2026-03-31 (~3 hours exposure) |
| **Attribution** | Sapphire Sleet (North Korea) / UNC1069 (Google) |
| **Affected Versions** | `axios@1.14.1`, `axios@0.30.4` |

**What happened:** North Korean state actors (Sapphire Sleet) compromised an axios maintainer account and published two malicious versions injecting `plain-crypto-js@4.2.1` (fake dependency), which downloaded a cross-platform RAT from C2. RAT commands: arbitrary PowerShell execution, file enumeration, in-memory binary injection. Estimated ~600,000 installs in 3 hours before removal.

**Impact on Nrwl repos:**
- `nrwl/ocean` pins `axios@1.16.0` via pnpm overrides — **not affected**
- `nrwl/nx` pins `axios@1.16.0` in dependencies — **not affected**
- Any repo with axios `1.14.1` or `0.30.4` in package manager caches should clear and re-install

**References:**

| 3 | `koa` (via `@nx/module-federation` → `@module-federation/node`) | CVE-2025-25200 | **CRITICAL** (CVSS 9.2) | koa < 2.15.4 | koa 2.15.4 | Fixed in Nx 22.7.0+ |

**CVE-2025-62718 — Axios `NO_PROXY` SSRF Bypass:**  
Axios proxy logic failed to normalize hostname casing / trailing dots before checking `NO_PROXY` allowlists. Attacker-controlled request URLs could bypass proxy controls and trigger SSRF against internal infrastructure.

**CVE-2025-25200 — koa Regex DoS:**  
Koa used an evil regex to parse `X-Forwarded-Proto` and `X-Forwarded-Host` headers, enabling memory-exhausting ReDoS attacks. Affected `@nx/module-federation` via transitive `@module-federation/node`.

### nrwl/ocean

**Package manager:** pnpm 10.26.1  
*Note: ocean already applies extensive `pnpm.overrides` patching qs, handlebars, jsonwebtoken, minimatch, ua-parser-js, axios (to 1.16.0), and others.*

| # | Package | CVE | Severity | Affected Range | Fixed In | Status in ocean |
|---|---|---|---|---|---|---|
| 1 | `undici` | CVE-2026-22036 | **HIGH** | < 6.23.0 | 6.23.0 | **VULNERABLE** — uses `^6.21.0` |
| 2 | `undici` | CVE-2026-1527 | **HIGH** | < 6.24.0 | 6.24.0 | **VULNERABLE** — uses `^6.21.0` |
| 3 | `undici` | CVE-2026-1525 | **HIGH** | Various | 6.23.0+ | **VULNERABLE** — uses `^6.21.0` |
| 4 | `undici` | CVE-2026-2581 | **HIGH** | Various | 6.23.0+ | **VULNERABLE** — uses `^6.21.0` |
| 5 | `mongodb` (server-side) | CVE-2025-14847 | **HIGH** (CVSS 8.7) | All pre-patch server versions | 8.2.3 / 8.0.17 / 7.0.28 / 6.0.27 | **Server-side** — verify deployed MongoDB server version |
| 6 | `tar` | CVE-2026-23745 | **HIGH** | ≤ 7.5.2 | 7.5.3 | **Partial risk** — uses `^6.2.1`; v6 may also be affected |
| 7 | `esbuild` (devDep) | CVE-2024-23334 | MEDIUM | ≤ 0.24.2 | 0.25.0 | **VULNERABLE** — uses `0.19.9` |

**CVE-2026-22036 — undici Decompression DoS:**  
Unbounded decompression chain in HTTP responses via `Content-Encoding`. A malicious upstream or MITM can cause fatal resource exhaustion (CPU/memory). Fixed in undici 6.23.0.

**CVE-2026-1527 — undici CRLF Injection:**  
CRLF injection via user-controlled `upgrade` option. Allows HTTP request manipulation and potential request smuggling. Fixed in undici 6.24.0.

**CVE-2025-14847 — MongoBleed:**  
Heap memory disclosure in MongoDB Server's zlib compression handling. Unauthenticated remote attacker sends crafted messages causing the server to return uninitialized heap memory (can expose auth tokens, session data, PII). Public exploit available since 2025-12-26; active exploitation confirmed. This is a server-side vulnerability — the npm driver (`mongodb@6.17.0`) is the client.

**Fix:** Add `"undici": "^6.24.0"` to `pnpm.overrides`. Bump `esbuild` devDep to `^0.25.0`.

`tar-fs` fails to sanitize `linkpath` entries for hard links and symlinks in `exports.extract`. A malicious tar archive can write files to arbitrary locations outside the intended extraction root.

**Fix:** Bump `tar-fs` to `^2.1.3` in devDependencies.

### Additional Risk: CVE-2025-36852 — CREEP Build Cache Poisoning

| Field | Value |
|---|---|
| **CVE** | CVE-2025-36852 / [GHSA-rrr2-jcr8-7q3x](https://github.com/advisories/GHSA-rrr2-jcr8-7q3x) |
| **Severity** | **CRITICAL** (CVSS 9.4) |
| **Disclosed** | 2025-06-26 |
| **Directly affected package** | `@nx/azure-cache` ≤ 4.0.0 |

A race-condition design flaw in bucket-based remote caches (S3, GCS, Azure Blob) allows untrusted PRs to **poison the build cache**, serving malicious build artifacts to developers. Any project using Nx remote caching where external contributors can write to the cache is at risk.

**Mitigation:** Restrict cache write access to trusted branches only. Use Nx Cloud's trust-boundary enforcement. Upgrade `@nx/azure-cache` to ≥ 4.0.1 if used.

- [ ] All devs: Verify Nx Console ≥ 18.100.0 in VS Code Extensions pane
- [ ] Anyone with `nrwl.angular-console@18.95.0` on 2026-05-18: rotate all credentials (GitHub, npm, AWS, SSH, GCP, 1Password, Anthropic, Vault, K8s). Check for `~/.local/share/kitty/cat.py`
- [ ] CI/dev machines that installed nx 20.9.0–21.8.0 after 2025-08-26: rotate all credentials. Search GitHub accounts for `s1ngularity-repository*` repos

- [ ] Add `"undici": "^6.24.0"` to `pnpm.overrides` (closes CVE-2026-22036, -1527, -1525, -2581)
- [ ] Verify MongoDB server in production is ≥ 8.2.3 / 8.0.17 / 7.0.28 / 6.0.27 / 5.0.32 / 4.4.30 (CVE-2025-14847)

- [ ] Audit `tar` usage and consider adding `"tar": "^7.5.3"` override (CVE-2026-23745)

### nrwl/nx-labs — High Priority

- [ ] Bump `tar-fs` from `^2.1.1` to `^2.1.3` in devDependencies (CVE-2025-48387)

### nrwl/nx-console — High Priority

- [ ] Bump `storybook` / `@storybook/web-components-vite` from `10.1.9` to `^10.1.10` (CVE-2025-68429)
- [ ] Bump `@storybook/addon-docs` from `9.0.18` to `^9.1.17` (CVE-2025-68429)

### nrwl/nx (build system) — Maintenance

- [ ] Patch stale branches `21.4.x` and `website-21` that still contain the vulnerable `pull_request_target` workflow (GHSL-2025-111)
- [ ] Verify `@module-federation/enhanced` ≥ 2.3.2 in all `@nx/module-federation` consumers (CVE-2025-62718)

- [ ] Enforce `minimum-release-age` npm safeguards on all developer machines
