## Proposed Process Rules

1. **All projects MUST have a target date.** 8 active In Progress projects have no target date (Gradle, Self-Healing CI, CLI AX, Nx Cloud changelog, Nx-api Iterative Improvements, Self-Healing CI Customer Success, Workspace visibility, Migrate DPE-tools to Lighthouse). Open-ended projects should use the end of the current initiative cycle as their target.

2. **Projects spanning 3+ initiative cycles need justification.** Gradle (8 cycles), Self-Healing CI (3 cycles), Maven (7 cycles), .NET (5 cycles), TUI (7 cycles, now closed). Each should have a written justification in the project description for why it remains open.

3. **All-milestones-complete = close the project.** If new work is needed, create a new project. Do not keep a zombie open "just in case."

4. **PoVs exceeding 3 months should be reclassified.** The 2-3 week PoV template doesn't match 5-month realities. Create a separate "Enterprise Implementation" project type.

5. **Paused + Overdue = triage within 2 weeks.** Either cancel, re-scope, or restart with a new target.
