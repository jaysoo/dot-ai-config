## Frameworks

### Angular

**Current stable:** 21.2.10 | **Next major preview:** 22.0.0-next.9 (2026-04-22)

**Releases in window:**
- v21.2.5 → v21.2.10 (stable patches, weekly)
- v22.0.0-next.4 → next.9 (six previews across Mar–Apr)
- v20.3.18/19, v19.2.20/21 (LTS patches)

**Angular 22 breaking surface (cumulative through next.9):**
- **Node 20 dropped** (min Node 22.22.0 / 24.13.1)
- **TypeScript 5.9 dropped** — TS 6+ required (next.7)
- `FetchBackend` is default HTTP backend; `withFetch()` deprecated no-op; migration via `withXhr()`
- `provideRoutes()` removed
- `ComponentFactoryResolver` / `ComponentFactory` removed from public API (next.7)
- `paramsInheritanceStrategy` default flipped from `'emptyOnly'` → `'always'` (next.9)
- `appRef.bootstrap` second arg no longer accepts `any` (non-nullable required)
- Compiler throws on duplicate input/output bindings and `data-` prefixed binding attempts
- `nullishCoalescingNotNullable` / `optionalChainNotNullable` diagnostics trigger on upgrade; `ng update` disables them by default (next.8)

**Angular 22 new features:**
- **`@Service` decorator** (next.9) — new DI primitive
- **Incremental hydration default-on** (next.8)
- **Shadow root bootstrap support** (next.8)
- **In-page AI tools** — DI graph visualization, AI runtime debugging tools (next.8)
- **`bootstrapApplication` via `ApplicationRef` with config** (next.7)
- Signal forms: late-bound input types, FormField CVA selection priority, deduplicated `writeValue`

**@nx/angular impact:** P0. Node/TS gating is mandatory; router default change needs docs+migration; factory removal requires generator sweep.

## Cross-Cutting Themes

### 1. Rspack 2.0 and the Rust bundler maturity moment
Rspack reaching GA (Apr 22) alongside Rolldown rc.17 marks both major Rust bundlers approaching production-ready state. The webpack-compat story is now cleaner (Rspack 2 sheds a lot of webpack-bridge baggage), but Nx plugin compatibility becomes the urgent piece.

### 2. Angular's AI turn
Angular 22 next.8 introduced in-page AI debugging tools (DI graph AI tool, AI runtime debugging) — the framework is embedding agent hooks directly. Nx MCP capabilities should align.

### 3. Node 18 is dead
Angular 22 dropped Node 20. Rspack 2.0 dropped Node 18. Vite 8 already required modern Node. Nx baseline should move to Node 22 in the next major.

### 4. ESM-only packages arriving
Rspack 2.0 is pure ESM. Anything internal to Nx that consumes `@rspack/core` via `require()` will break. `@nx/rspack` likely needs a mini-refactor for ESM interop.

### 5. TS 6 adoption is being forced
Angular 22 requires TS 6. That will force downstream generators (CNW, `@nx/angular`) to bump baseline TS.
