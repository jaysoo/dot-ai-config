## Frameworks

### Angular

**Current stable:** 21.2.8 (npm verified) | **Next major preview:** 22.0.0-next.7

**Releases in window:**
- v21.2.0 through v21.2.8 (stable patches)
- v22.0.0-next.0 through next.7 (major preview, weekly cadence)
- v20.3.17-v20.3.18, v19.2.19-v19.2.20 (LTS patches)

**Angular 22 breaking changes (cumulative through next.7):**
- **Node.js 20 dropped** -- minimum is now Node 22.22.0 and 24.13.1
- **TypeScript <6.0 no longer supported** (new in next.7)
- **`ComponentFactoryResolver` and `ComponentFactory` removed** -- pass component classes directly (new in next.7)
- `FetchBackend` is now the default HTTP backend; migration via `withXhr()`
- `provideRoutes()` removed (use `provideRouter()` or `ROUTES` multi token)
- `data-` prefixed attributes no longer bind inputs/outputs
- Compiler throws on duplicate input/output bindings
- `in` variables throw in template expressions
- `CanMatchFn` snapshot parameter now mandatory (new in next.7)
- Styles on dropped `host` elements now removed (new in next.7)
- `appRef.bootstrap` second arg no longer accepts null/undefined (new in next.7)

**Angular 22 new features (preview):**
- `ApplicationRef` bootstrap with config support (new in next.7)
- Host directive de-duplication (new in next.7)
- `@defer` prefetch warning without main trigger (new in next.7)
- `IdleService` with `IdleRequestOptions`
- `TestBed.getFixture` for testing
- `@defer` idle customization and optional timeout
- `browserUrl` input on router links
- Signal forms expansion continuing
- `CanMatchFn` migration schematic (new in next.7)

**Nx plugin impact:**
- `@nx/angular` generator must handle Node 22+ AND TypeScript 6.0+ requirements for Angular 22
- `ComponentFactoryResolver` removal may break older Nx migration code if referenced
- Init generator should add `withXhr()` migration for users upgrading
- Angular 22 weekly cadence continues -- stable likely May/June 2026

## Cross-Cutting Themes

### 1. Angular 22 Breaking Change Surface Widening
Angular 22 next.7 adds TypeScript <6 drop and `ComponentFactoryResolver` removal to the already significant Node 20 drop. This is now the highest-impact upcoming framework change for Nx -- three separate compatibility gates (Node, TypeScript, API removal) all in one release. Nx needs a coordinated Angular 22 support plan.

### 2. Rspack 2 and Vite 8 -- Dual Bundler Major Versions
Both `@nx/vite` and `@nx/rspack` need major version support work simultaneously. Rspack 2 RC.1's improved tree-shaking (side-effect-free function analysis) makes it more competitive with Vite 8's Rolldown. Nx users will benefit from both paths being well-supported.

### 3. Vite 8 Stabilization Accelerating
Eight patches in four weeks, including security fixes. The cadence is healthy -- most remaining issues are edge cases (DNS resolution, CJS interop). Safe for production adoption. Rolldown rc.15 inside Vite 8.0.8 is also maturing.

### 4. Security Focus in Bundler Updates
Vite 8.0.5 added path traversal prevention and `server.fs` checks. esbuild 0.28.0 added binary integrity verification. Bundler security is getting more attention -- Nx should surface these updates in migration guides.

### 5. AI-Native Framework APIs (Steady State)
Next.js 16.2's Agent DevTools and AI scaffolding shipped last month and are stable. No new AI-focused framework features this period, but the foundation is set. Nx's MCP integration should ensure compatibility.

### 6. Node.js 22 Becoming the Floor
Angular 22 requires Node 22.22+. Vite 8 requires 20.19+/22.12+. The trend toward Node 22 as baseline continues. Nx should plan for Node 22 minimum across all plugins by H2 2026.
