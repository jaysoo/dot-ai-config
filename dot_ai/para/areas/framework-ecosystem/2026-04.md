## Frameworks

### Angular

**Current stable:** 21.2.7 | **Next major preview:** 22.0.0-next.6

**Releases in window:**
- v21.2.1 through v21.2.7 (stable patches)
- v22.0.0-next.0 through next.6 (major preview, weekly cadence)
- v20.3.17-v20.3.18, v19.2.19-v19.2.20 (LTS patches)

**Angular 22 breaking changes (confirmed in next.0-next.6):**
- **Node.js 20 dropped** -- minimum is now Node 22.22.0 and 24.13.1
- `FetchBackend` is now the default HTTP backend; migration schematic via `withXhr()`
- `provideRoutes()` removed (use `provideRouter()` or `ROUTES` multi token)
- `getAngularLib`/`setAngularLib` removed from upgrade package
- `data-` prefixed attributes no longer bind inputs/outputs
- Compiler throws on duplicate input/output bindings
- `in` variables throw in template expressions
- `AnimationCallbackEvent.animationComplete` signature changed

**Angular 22 new features (preview):**
- `IdleService` with `IdleRequestOptions` support
- `TestBed.getFixture` for testing
- `@defer` idle customization and optional timeout
- `browserUrl` input support on router links
- `ChangeDetectionStrategy.Eager` migration schematic
- Resource params with special return statuses
- Forms: `touched` model split into input and `touch` output; untracked `setValue` in reactive forms

**Nx plugin impact:**
- `@nx/angular` generator must handle Node 22+ requirement for Angular 22 workspaces
- Init generator should add `withXhr()` migration for users upgrading who need upload progress
- Angular 22 preview cadence is weekly -- stable likely May/June 2026

## Bundlers

### Vite

**Current stable:** 8.0.6 | **MAJOR RELEASE: 8.0.0 (2026-03-12)**

**Releases in window:**
- v8.0.0-beta.14 through beta.18 (pre-release)
- **v8.0.0** (2026-03-12)
- v8.0.1 through v8.0.6 (rapid patches through 2026-04-07)
- v7.3.2, v6.4.2 (LTS patches, 2026-04-06)

**Vite 8 key changes (confirmed from blog):**
- **Rolldown is the production bundler** -- replaces esbuild+Rollup dual approach
- **10-30x faster builds** claimed; 38-64% real-world build time reductions
- Node.js 20.19+ or 22.12+ required (unchanged from Vite 7)
- Built-in devtools (`devtools` option)
- TypeScript path alias support (`resolve.tsconfigPaths`)
- Decorator metadata (`emitDecoratorMetadata`) without plugins
- SSR `.wasm?init` imports
- Browser console forwarding (`server.forwardConsole`)
- ~15 MB install size increase (lightningcss + Rolldown binary)
- Migration path: `rolldown-vite` on Vite 7 first, then upgrade to Vite 8

**Nx plugin impact:** HIGH priority.
- `@nx/vite` must update to support Vite 8 and Rolldown as bundler
- Existing Rollup plugin configs may need `rolldownOptions` migration
- Version detection in `@nx/vite` init generator needs Vite 8 awareness
- CNW templates using Vite need updating
- 6 patches in 4 weeks signals early stabilization -- safe to adopt now

## Cross-Cutting Themes

### 1. Rspack 2 RC -- Bundler Competition Heats Up
Rspack reaching RC is significant. Combined with Vite 8's Rolldown integration, the bundler landscape is in its most active transition period. Nx wraps both `@nx/vite` and `@nx/rspack` -- both need major version support work simultaneously. This is the highest-priority cross-cutting theme.

### 2. Vite 8 Early Stabilization
Six patch releases in four weeks signals Vite 8 is past the worst early-adopter bugs. The migration path (rolldown-vite on Vite 7 -> Vite 8) is well-documented. Nx should treat Vite 8 as ready for production adoption.

### 3. AI-Native Framework APIs Deepening
Next.js 16.2 shipped Agent DevTools, browser log forwarding for agents, and AI scaffolding in `create-next-app`. This is not experimental anymore -- it's in stable releases. Nx's MCP integration should ensure compatibility with these framework-level agent features.

### 4. Angular 22 Approaching
Angular 22 is on next.6 with weekly cadence. Breaking changes are well-documented and the migration schematic story is solid. Nx should be ready for Angular 22 support day-one (likely May/June 2026).

### 5. Remix 3 Architectural Reset
Remix 3 alpha shows a fundamentally different package architecture (modular middleware, data-table, etc.). This is early but signals that `@nx/remix` will eventually need a significant rewrite. No action needed now, but start tracking.

### 6. Node.js 22 Becoming the Floor
Angular 22 requires Node 22.22+. Vite 8 requires Node 20.19+/22.12+. The ecosystem is converging on Node 22 as the baseline. Nx should plan for this across all plugins by H2 2026.
