## Cross-Cutting Themes

### 1. Frameworks that were "coming soon" in March are now in active post-GA churn
Vite 8 shipped Mar 12 and is now on 8.0.9 in 6 weeks; Rspack 2 is on rc.3 with breaking refactors still landing; Angular 22 next.8 flipped incremental hydration to default-on; Next.js started a 16.3 canary line. Every framework we flagged last month as "about to ship" is shipping — Nx plugin work is **more urgent** across the board.

### 2. Coordinated security backporting
Apr 15-16 saw: Next.js 16.2.4 (Turbopack patches), Angular SSRF fix backported across 4 release lines (19.2.21 / 20.3.19 / 21.2.9 / 22-next.8) with public disclosure Apr 16 (GHSA-45q2-gjvg-7973), webpack 5.106.2. Nx should have a clear runbook for "emergency floor bump" migrations when upstream CVEs land — and for notifying users who still track older LTS lines.

### 3. TypeScript 6 is now mandatory for bleeding-edge Angular
Angular 22 drops TS 5.9, and next.8's service-worker fix confirms upstream is testing against TS 6.0. Nx workspaces targeting Angular 22 will need TS 6 bundled. This intersects with Nx's own TS version planning.

### 4. Rolldown / Vite coupling remains tight
Vite 8.0.8 → rc.15, Vite 8.0.9 → rc.16 — the Vite patch cadence is effectively Rolldown RC cadence. Rolldown rc.16 introduced a new module tagging system for code splitting. Expect more Vite patches as users report edge cases.

### 5. Bundler tree-shaking arms race continues
Rspack 2 shipped side-effect-free function call analysis + persistent SWC minimizer cache (rc.3); webpack 5.106 added CJS destructuring tree-shaking; Rolldown rc.16 added module tagging. Bundlers are converging on more aggressive optimization defaults. Nx generators should validate these don't regress user output.

### 6. AI integration surfacing across the ecosystem
Angular next.8 shipped `register AI runtime debugging tools` and an in-page DI graph AI tool. `create-rspack` rc.3 now offers optional "agent skills" in its starter template. Next.js 16.2 shipped `AGENTS.md` in `create-next-app`. This is becoming a standard framework capability — Nx should ensure its MCP and agent story stays visible.

### 7. React ecosystem governance stable; linting tier moving
Core React saw coordinated backport-only triple release (19.0.5 / 19.1.6 / 19.2.5 on Apr 8) — no API surface changes. The interesting action is `eslint-plugin-react-hooks@7.1.x`: ESLint v10 support, skip non-React files, all-errors-at-once. Forward-compat for Nx's ESLint story.
