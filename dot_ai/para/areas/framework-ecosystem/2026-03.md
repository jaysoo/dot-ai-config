# Framework & Bundler Ecosystem -- March 2026

_Scan date: 2026-03-04_
_Period: January 3 -- March 4, 2026_

## Summary

- **TypeScript 6.0 beta** (Feb 11) is the biggest cross-cutting event: ES5 target deprecated, `strict` defaults to true, `--outFile` removed, `--moduleResolution classic` removed. Last JS-based TS release before the Go rewrite (TS 7). Every Nx plugin and generator that emits tsconfig needs auditing.
- **Angular 21.2.0** (Feb 25) adds SignalFormControl, TypeScript 6 compat, arrow functions in templates, and `ChangeDetectionStrategy.Eager`. Nx 22.3+ already supports Angular 21; the 21.2 minor should be low-risk.
- **Rspack 2.0 now at beta.5** (Mar 3): drops Node 18, moves to pure ESM, renames webpack-prefixed globals, adds RSC lazy compilation compat. Nx `@nx/rspack` plugin will need a major update path.
- **Vite 8 beta.16** (Feb 27) continues maturing with Rolldown RC.6 integration. Rolldown RC.6 dropped built-in CSS bundling to explore alternatives. Once Vite 8 goes stable, `@nx/vite` needs testing.
- **Rolldown RC.6** (Feb 26) drops CSS bundling, adds `entriesAware` code splitting, approaches 1.0.
- **Remix 3.0 alpha** (Feb 28) is a full rewrite as a modular middleware-based framework (`@remix-run/*` packages). Early stage, not yet relevant for Nx plugins.
- **Webpack 5.105.0** (Feb 3) adds built-in `tsconfig` resolver option, potentially obsoleting `tsconfig-paths-webpack-plugin`.
- **Oxfmt beta** (Feb 24) reaches 100% Prettier JS/TS conformance at 30x speed.

## Remix (the framework, not React Router)

**Latest:** remix@3.0.0-alpha.3 (Feb 28, 2026)

### What's happening

Remix is being rewritten as a modular, middleware-based framework. The `remix` package now re-exports from many `@remix-run/*` packages:

- `@remix-run/data-schema` -- data validation
- `@remix-run/data-table` -- database table abstractions (MySQL, Postgres, SQLite)
- `@remix-run/fetch-router`, `@remix-run/route-pattern` -- routing
- `@remix-run/session-middleware`, `@remix-run/session-storage-redis` -- sessions
- `@remix-run/static-middleware`, `@remix-run/fs`, `@remix-run/mime` -- static file serving
- `@remix-run/component` -- UI components

This is a fundamentally different architecture from Remix v2 / React Router v7.

### Nx impact

- Very early alpha. No action needed for `@nx/remix`.
- Long-term, Remix 3.0 may require a new Nx plugin or significant rework of `@nx/remix`.
- Current `@nx/remix` targets React Router v7 framework mode, which remains stable.

## Vite

**Latest stable:** 7.3.1 (Jan 7, 2026)
**npm:** vite@7.3.1
**Latest beta:** 8.0.0-beta.16 (Feb 27, 2026)

### Vite 8 beta status (beta.10 through beta.16 in this period)

- **Replaces esbuild + Rollup with Rolldown** as the single bundler
- Linear: build times from 46s to 6s. Beehiiv: 64% reduction.
- **Built-in tsconfig `paths` support**
- **Automatic `emitDecoratorMetadata`** support
- **Plugin API remains compatible** -- most Vite plugins work out-of-the-box
- Beta.14 adds WASM SSR support (`.wasm?init`), upgrades to Rolldown 1.0.0-rc.4
- Beta.16 latest (Feb 27)

### Breaking changes for Vite 8

- **`build.rollupOptions` renamed to `build.rolldownOptions`** (rollupOptions deprecated)
- **`optimizeDeps.esbuildOptions` deprecated** -- converted to `optimizeDeps.rolldownOptions` automatically
- **`moduleType: 'js'` may be required** in plugin `load`/`transform` hooks when converting non-JS content to JS
- **`import.meta.hot.accept` no longer accepts URLs** -- must pass ids instead
- **Package resolution change:** when both `browser` and `module` fields exist in package.json, Vite no longer auto-selects ESM; uses `resolve.mainFields` order instead
- Framework users (Astro, Nuxt, Vitest) must override `vite` dependency
- Two migration paths: direct upgrade or gradual via `rolldown-vite` package

### Nx impact

- `@nx/vite` will need updates when Vite 8 goes stable.
- Plugin API compat means most Nx Vite configs should work, but testing required.
- The `rolldown-vite` gradual migration path could be useful for Nx workspaces.

- alpha.0 (Jan 22) -> alpha.1 (Jan 27) -> beta.0 (Feb 3) -> beta.1 (Feb 10) -> beta.2 (Feb 11) -> beta.3 (Feb 25) -> beta.4 (Feb 27) -> beta.5 (Mar 3)

### Rspack 2.0 breaking changes (critical for Nx)

- **Drops Node 18** -- minimum Node 20
- **Pure ESM package** -- no more CommonJS build

- **Removed:** `experiments.rspackFuture`, `experiments.lazyCompilation`, `experiments.parallelLoader`, `experiments.outputModule`, `.swcrc` reading, `output.charset`, legacy `output.library*` options, `WarnCaseSensitiveModulesPlugin`, `plugin.getHooks()`, `profile`/`stats.profile`, unsafe cache
- **SWC loader:** `verbatimModuleSyntax: true` by default
- **CSS enabled by default**
- **`@rspack/dev-server`** now optional peer dependency
- **`incremental`** moved from `experiments` to top-level config
- **`bundlerInfo`** moved from `experiments.rspackFuture` to `output`
- **`requireAsExpression`** disabled by default

### Beta.5 highlights (Mar 3)

- **RSC compatible with lazy compilation**
- **CJS require destructuring tree shaking**
- **Strict this context on imports**
- **`import.defer()` support** for statically-analyzed paths
- **`cache.portable` configuration** added
- Performance: replaced OverlayMap with RollbackAtomMap
- Migration guide from 1.x to 2.0 available
- **Target unification:** `target` option now affects SWC and LightningCSS defaults (configure once)

### Nx impact

- **HIGH PRIORITY:** `@nx/rspack` and `@nx/module-federation` will need major updates.
- Node 18 drop aligns with broader ecosystem but must be coordinated.
- Pure ESM transition may affect how Nx loads Rspack configs.
- Renamed chunk globals affect Module Federation setups.
- Plan migration support for Rspack 2.0 when it goes stable.

## Cross-Cutting: TypeScript 6.0 Beta

**Released:** February 11, 2026

### Critical changes for Nx

| Change                               | Impact                                                                                |
| ------------------------------------ | ------------------------------------------------------------------------------------- |
| `strict` defaults to `true`          | New projects fine; existing may break                                                 |
| `module` defaults to `esnext`        | Generators emitting tsconfig need updating                                            |
| `target` defaults to `es2025`        | Safe for modern projects                                                              |
| `types` defaults to `[]`             | Projects relying on ambient types (`@types/node`) will break without explicit `types` |
| `rootDir` defaults to tsconfig dir   | May change output structure                                                           |
| ES5 target deprecated                | Breaks legacy configurations                                                          |
| `--outFile` removed                  | Breaks projects using bundled output                                                  |
| `--moduleResolution classic` removed | Must migrate to `node`, `bundler`, or `nodenext`                                      |
| `--baseUrl` deprecated               | Use `paths` with `#/` imports instead                                                 |
| `esModuleInterop: false` forbidden   | Must be true or omitted                                                               |
| AMD/UMD/SystemJS deprecated          | Unlikely to affect modern Nx workspaces                                               |

### Nx action required

- Audit all tsconfig generators across plugins
- Ensure migration generators handle new defaults
- Test `@nx/js`, `@nx/angular`, `@nx/react`, `@nx/next` with TS 6 beta
- Plan for TS 7 (Go rewrite) -- API stability unknown
