## Frameworks

### Angular

**Current stable:** 21.2.6 | **Next major preview:** 22.0.0-next.5

**Releases in window:**
- v21.2.0 through v21.2.6 (stable patches)
- v22.0.0-next.0 through next.5 (major preview)
- v20.3.17-v20.3.18, v19.2.19-v19.2.20 (LTS patches)

**Angular 21.2 highlights:**
- TypeScript 6 support added
- `ChangeDetectionStrategy.Eager` alias for Default
- Signal forms major expansion: `SignalFormControl`, form directive, parse errors, submit options
- Arrow functions in template expressions
- `instanceof` operator support in templates
- Exhaustive `@switch` block checks
- Resource composition via snapshots
- Inline styles support in language server (completions, hover, folding)

**Angular 22 breaking changes (preview):**
- **Node.js 20 dropped** -- minimum is now Node 22.22.0 and 24.13.1
- `FetchBackend` is now the default HTTP backend (was `XhrBackend`); migration schematic provided via `withXhr()`
- `withFetch()` deprecated (now a no-op since Fetch is default)
- `provideRoutes()` removed (use `provideRouter()` or `ROUTES` multi token)
- `getAngularLib`/`setAngularLib` removed from upgrade package
- `data-` prefixed attributes no longer bind inputs/outputs
- Compiler throws on duplicate input/output bindings

**Nx plugin impact:**
- `@nx/angular` generator must handle Node 22+ requirement for Angular 22 workspaces
- Init generator should add `withXhr()` migration for users upgrading to Angular 22 who need upload progress
- Signal forms are developer-preview but growing fast -- docs and generators should reference them

## Cross-Cutting Themes

### 1. Rolldown Convergence
Vite 8 shipping Rolldown as its production bundler is the biggest ecosystem shift this period. The Rollup plugin ecosystem faces a compatibility inflection point. Nx must ensure `@nx/vite` handles this transition smoothly.

### 2. Node.js Version Floor Rising
Angular 22 drops Node 20. Vite 8 likely follows. Nx should plan for Node 22 as the new baseline across plugins by mid-2026.

### 3. AI-Native Developer Tooling
Next.js 16.2 ships `AGENTS.md` in `create-next-app` and an experimental `next-browser` for AI agents. This signals framework-level AI integration is becoming standard. Nx should ensure its MCP and agent capabilities work well with these framework features.

### 4. Performance as the Battleground
Next.js 16.2's 400% faster dev startup and 25-60% faster rendering, plus Rspack 2's continued Rust-native optimizations, show performance is the primary competitive axis. Nx's value add is in orchestration and caching on top of these fast tools.

### 5. Type Safety Deepening
Angular's signal forms with full typing, Next.js strict route types, and Nuxt's typed layout props all push type safety further into framework APIs. Nx generators should produce type-safe configurations by default.
