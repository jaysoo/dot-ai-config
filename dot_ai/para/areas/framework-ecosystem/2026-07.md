## Bundlers

### Vite

- **Releases**: 8.1.0 (June 23), 8.1.1, 8.1.2, 8.1.3 (July 2, latest). LTS channels: 6.x, 7.x maintained separately.
- **Key changes (8.1.0)**:
  - `server.hmr` options deprecated; renamed to `server.ws`. Generates warnings in 8.1.
  - Direct `.wasm` imports (WASM ESM Integration) with improved SSR support.
  - `server.fs.deny` extended with common sensitive file patterns.
  - `html.additionalAssetSources` option for extended asset discovery.
  - CSS path resolution via tsconfig paths in Sass/@import.
  - Vite Task: zero-config build caching integration (competitive with Nx caching).
  - Chunk importmap support for better code-splitting control.
  - Rolldown updated to 1.1.2 internally (note: Rolldown 1.1.4 now reverts lazyBarrel default -- see Rolldown section).
  - No breaking changes in 8.1.
- **@nx/vite impact**: P1. Audit all scaffold output / inferred target configs for `server.hmr` usage and replace with `server.ws`. Evaluate "Vite Task" caching against Nx task caching story -- could be a user confusion point or a competitive differentiator worth addressing in docs.

### Rspack

- **Releases**: 2.1.0 (June 26), 2.1.1 (June 27), 2.1.2 (June 30, latest). Node requirement: `>=20.19.0 || >=22.12.0`.
- **Key changes (2.1.0)**:
  - `maxGenerations` persistent cache option renamed to `maxVersions`; default increased to 3.
  - Pure functions analysis enabled by default in prod builds (`#__NO_SIDE_EFFECTS__` annotations respected).
  - React Compiler support in `builtin:swc-loader` (configure via loader options).
  - `import.meta.rspackPublicPath` and `import.meta.rspackHash` module variables (ESM-native equivalent to CJS globals).
  - `experimental.runtimeMode` infrastructure for runtime internals.
  - Module graph API: `getProvidedExports()` exposed, module error JS API added.
- **@nx/rspack impact**: P0 (carryover + 2.1 surface). `maxGenerations` rename will break any generated configs. Pure functions by default changes tree-shaking output -- validate against existing snapshots. React Compiler is a feature gap to document. ESM-only interop remains open. No indication of progress on the Nx-side integration.

### Rolldown

- **Current**: 1.1.4 (July 1). Was 1.1.2 in June.

- **1.1.4 changes** (July 1): **`experimental.lazyBarrel` DISABLED by default** (reverts the 1.1.0/1.1.2 behavior). Users must now explicitly opt in (`experimental.lazyBarrel: true`). Also: lazy barrel dev mode fix, JSON interface preservation under `preserveModules`, improved deconflicting.

## Cross-Cutting Themes

### 1. P0 Angular 22 compat resolved -- execution phase begins
GitHub #35910 is closed. The blocking peer-dep conflict is fixed. The June report listed 8 follow-on items for Angular 22 compat (OnPush migration, signal-forms generator, SSRF guards, etc.). The path is now clear to execute those. This is the most significant delta vs June.

### 2. Three concurrent new minors: Vite 8.1, Rspack 2.1, webpack 5.108
Three ecosystem minors in the same 18-day window is unusual. All three ship new APIs that affect `@nx/*` plugin output or configuration scaffolding. The `server.hmr` -> `server.ws` rename (Vite), `maxGenerations` -> `maxVersions` rename (Rspack), and `output.globalObject` default change (webpack) each require a config audit.

### 3. Rolldown lazyBarrel reversal
The June P1 risk (barrel-heavy codebases breaking) is self-corrected by Rolldown 1.1.4 disabling lazyBarrel by default. This reduces urgency on the Nx side but means the June docs note (if published) should be retracted.

### 4. Remix v3 beta cadence signals 2026 H2 scope decision needed
Five beta iterations in 2 months with significant API surface (UI components, route patterns, server config) means Remix v3 is moving toward stability. Nx needs a decision on when to start a plugin feasibility spike. Recommendation: trigger at beta.8 or RC1 -- whichever comes first.

### 5. Vite Task vs Nx caching: narrative alignment needed
Vite 8.1 ships "Vite Task" -- zero-config build caching inside Vite. This is positioned similarly to Nx task caching for Vite-native projects. Users in Vite-first monorepos may use it without Nx. Worth a competitive positioning review in Nx docs to clarify when each is appropriate.
