## Frameworks

### Angular

- **Releases this period**: 22.0.6 (July 8), 22.0.7 (July 16). Next: 22.1.0-next.0 through next.6.
- **Key changes**:
  - 22.0.6: Security fix -- `__proto__` outlet name handling + `hasOwnProperty` in query param parsing. Compiler: optional chaining type-check fix. Forms/Signals: `extractValue` reactive fix. Migrations: transitive NgModule reference preservation.
  - 22.0.7: **ReDoS vulnerability patched** in `platform-server` `SOURCEMAP_URL_REGEXP`. Core: static attributes for explicit input transforms. Forms: multiple async validators; intermediate number values in signal forms; stale CVA writeback blocked. HTTP: interceptor signal reads isolated from reactive contexts. Localize: null prototype for translations map.
  - 22.1.0-next.5: CSS variable namespacing in compiler; HTTP options for caching credentialed/non-cacheable requests; Chrome Performance panel deep links to Angular DevTools.
  - 22.1.0-next.6: Sync with 22.0.7 patch; no new features.
- **Nx 23.1 impact (July 15)**: **Full Angular 22 support shipped** -- Signal Forms stable, `resource()`/`httpResource()`, `@Service` decorator, `injectAsync`, OnPush new-component default, Angular Aria, TypeScript 6 support. **Angular v19 support dropped** (migration via `nx migrate`). ESLint v8 removed. This closes the carryover P0 Angular 22 items from July 6.
- **@nx/angular impact**: P0 carryover resolved by Nx 23.1. New: security peer range should specify 22.0.7+. 22.1 tracking continues (JSONP sweep, CSS variable namespacing).
- **Action needed**: Confirm Nx 23.1 migration is clean end-to-end. Write ESLint v8 -> v9 comms. Begin 22.1 tracking (JSONP sweep). Bump min peer to 22.0.7 for ReDoS fix.

## Cross-Cutting Themes

### 1. Nx 23.1 closes the major Angular 22 gap
Nx 23.1 (July 15) ships full Angular 22 support including Signal Forms, `resource()`, OnPush defaults, and TypeScript 6. This closes the carryover P0 from the June and July 6 reports. The new compat burden is the other side of the coin: Angular 19 support dropped and ESLint v8 removed -- both need migration guides before users upgrade.

### 2. Vue 3.6 enters RC -- Vapor Mode production approaching
Vue 3.6 moved from beta.17 to RC.1 in the same window. Vapor Mode is declared feature-complete (excluding Suspense). This signals stable is weeks away. `@nx/vue` has not yet begun RC validation or Vapor Mode generator planning. This is the fastest-moving pre-release story right now.

### 3. Nuxt 3 EOL July 31 -- plugin cleanup opportunity
Nuxt 3 reaches end-of-life in 11 days. Nuxt 4.5.0 ships Vite 8, Rspack 2, SSR streaming, and `unhead` v3. This creates a clean cut point to drop Nuxt 3 from `@nx/nuxt` in the next major, aligning the plugin with the ecosystem's direction.

### 4. TypeScript 7 entering the build toolchain
Next.js 16.3 preview.6 ships experimental TypeScript 7 CLI support (Go/Rust native toolchain). This is early but significant -- if Next.js ships with TS7 support at 16.3 GA, Nx will need a TS7 executor path for `@nx/next`. TS7 changes module resolution and compilation output.

### 5. Rolldown format convention shift (`esm` -> `es`)
Rolldown 1.2.0 changes the `[format]` token for ESM outputs from `esm` to `es`. This flows through Vite into any Nx project using format tokens in output filenames. A small but concrete breaking change that could silently rename build artifacts.
