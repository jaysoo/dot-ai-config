## Frameworks

### Angular

**Current stable:** 21.2.15 (May 28, 2026) | **Next major preview:** 22.0.0-rc.2 (May 28, 2026)
**LTS:** 20.3.23, 19.2.24 (May 28, 2026)

**Releases in window (May 25-Jun 1):**
- v21.2.12 -> v21.2.15 (four security patches)
- v22.0.0-rc.2 (May 28)
- v20.3.22, v20.3.23 (security patches)
- v19.2.23, v19.2.24 (security patches)

**Security CVEs (NEW - first appearance in this scan series):**

- **CVE-2026-27739** (SSRF): Angular SSR trusts `Host` and `X-Forwarded-*` headers to determine base origin without validation. An attacker can redefine the application base to an arbitrary external domain, enabling SSRF and relative URL manipulation. Fix: 21.2.15, 20.3.17+, 19.2.21+.
- **CVE-2026-22610** (XSS): Angular template compiler mishandles SVG `<script>` elements during sanitization, enabling arbitrary JS execution in the browser. Fix: 21.2.15+, LTS patches.

**Nx action:** Any `@nx/angular` workspace generating SSR apps should document the minimum safe patch level. If any Nx migration guide calls out specific Angular 21/20/19 versions, they must be at or above the fix versions.

**Angular 22 RC.2 (May 28):**
- Security hardening: SSRF prevention, sanitization hardening (same CVEs in RC line)
- Additional compiler + core bugfixes
- GA expected week of June 1, 2026 (per six-month cadence pattern and community signals)

**Cumulative Angular 22 breaking surface (unchanged from May):**
- Node 20 dropped (min Node 22.22.0 / 24.13.1)
- TypeScript 5.9 dropped - TS 6+ required
- FetchBackend is default HTTP backend; `withFetch()` deprecated no-op; escape via `withXhr()`
- `provideRoutes()` removed
- `ComponentFactoryResolver` / `ComponentFactory` removed
- `paramsInheritanceStrategy` default flipped `'emptyOnly'` -> `'always'`
- Compiler throws on duplicate input/output bindings, `data-` prefixed binding attempts
- `forms.min`/`max` no longer accept string values (numbers/null only)
- `reportProgress` deprecated for `reportUploadProgress`/`reportDownloadProgress`
- Signal forms graduated to public API

**@nx/angular impact:** P0. GA this week. Signal forms generator, security floor docs, TS 6 / Node 22 gating, FetchBackend + router migration docs, `ComponentFactoryResolver` sweep. All carry-forward from May.

## Cross-Cutting Themes

### 1. Security wave across Angular versions
The May 27-28 security releases (CVE-2026-27739 SSRF + CVE-2026-22610 XSS) hit all active Angular lines simultaneously. This is the first multi-CVE coordinated patch since this scan started. `@nx/angular` generated SSR workspaces are directly in scope for the SSRF CVE. Nx should communicate minimum safe version floors in its Angular plugin docs.

### 2. Angular 22 GA week
All signals point to Angular 22 going stable this week (June 1-7). Six months of preview work, two RCs, and community blog posts calling "week of June 1." The entire May P0 backlog (`@nx/angular` prep) becomes a live incident if GA lands without Nx support. Prioritize this week.

### 3. Rspack closes the Vite gap
`import.meta.glob` in Rspack 2.0.5 is the last major Vite-exclusive API that developers cited as a migration blocker. With `css/global`, CSS module parsing controls, and now glob imports, the Rspack -> Vite migration story has flipped. `@nx/rspack` is the beneficiary and should update migration docs immediately.

### 4. Next.js 16.3 GA imminent
Canary.36 in 6 weeks. Pattern matches prior Next.js major-minor GA timelines. `@nx/next` has ~1-2 weeks to complete pre-GA validation before a potentially unannounced stable drop.

### 5. Vue 3.6 Vapor Mode needs a plugin strategy
Beta.13 confirms Vapor Mode is feature-complete and opt-in. The question for `@nx/vue` is not IF to support it but HOW. A `--vapor` generator flag is the obvious surface, but the runtime differs enough that it may need its own template set. Design this before 3.6 RC.

### 6. Vite same-day double-patch (unusual)
8.0.15 + 8.0.16 on the same day (June 1) suggests a rapid regression fix. Until the CHANGELOG is accessible, treat 8.0.15 as unstable and template to 8.0.16.
