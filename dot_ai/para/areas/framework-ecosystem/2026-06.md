## Frameworks

### Angular

**Current stable:** 22.0.1 (2026-06-10) | **Next minor preview:** 22.1.0-next.0

**Releases in window:**
- v22.0.0 GA (2026-06-03) -- first stable after 12 preview drops
- v22.0.1 (2026-06-10) -- first patch

**Angular 22.0 highlights:**

- **Signal Forms stable.** `ReactiveFormsModule` signal-based API is production-ready. `formSignal()`, typed form groups with signals, reactive validators.
- **Resource API stable.** `resource()`, `rxResource()`, `httpResource()` all graduated to stable.
- **Angular Aria stable.** Accessibility primitives out of dev preview.
- **OnPush default for new components.** `ChangeDetectionStrategy.OnPush` is now the default. `ng update` auto-migrates existing components without an explicit strategy to `ChangeDetectionStrategy.Eager` to preserve behavior.
- **Zoneless architecture stable.** No Zone.js required for new apps; `provideExperimentalZonelessChangeDetection()` API graduated.
- **WebMCP APIs (experimental):** `provideWebMcpTools()`, `declareWebMcpTool()` -- Angular's MCP/agentic integration surface.

**Breaking changes:**
- **Node.js 22+ required** (drops Node 20; Node 26 supported)
- **TypeScript 6+ required** (drops TS 5.9 and below)
- `forms.min` / `forms.max` validators no longer accept string values; numeric or null only
- `reportProgress` on HTTP requests deprecated; use `reportUploadProgress` / `reportDownloadProgress`
- `ComponentFactoryResolver`, `ComponentFactory` removed (use `ViewContainerRef.createComponent` or `createComponent()`)
- `provideRoutes()` removed
- `FetchBackend` is default HTTP backend; `withFetch()` is now a no-op; use `withXhr()` to revert
- `paramsInheritanceStrategy` default changed from `'emptyOnly'` to `'always'`

**@nx/angular impact:** P0. GA has landed -- all previously-tracked RC risks now need immediate shipping:
1. Component generator: default `changeDetection` to OnPush
2. Peer dep gates: Node 22+, TS 6+
3. Signal Forms generator path
4. `forms.min/max` migration schematic (string -> number)
5. `reportProgress` -> `report{Upload,Download}Progress` migration
6. Already removed: `ComponentFactoryResolver` sweep should be complete
7. Docs: OnPush default explanation, Signal Forms getting-started

**Action needed:** Yes -- P0 blockers above.

## Cross-Cutting Themes

### 1. Cloudflare/VoidZero: infrastructure shift for the Vite ecosystem

The June 4 acquisition is the most significant strategic event since Vite 8's Rolldown integration in March. Cloudflare now owns the entire Vite/Rolldown/Vitest/Oxc stack. The $1M ecosystem fund and MIT/vendor-neutral pledges reduce governance risk, but the strategic direction is clearly edge-first (Workers). For Nx:
- Workers-first deployment generators (`@nx/vite` + Cloudflare Workers) become a priority opportunity
- Monitor whether Vite 8.1+ starts adding Workers-specific APIs or defaults that require `@nx/vite` adaptation
- Vitest (not yet Nx-native) may gain official Cloudflare Workers support -- watch for Nx test executor implications

### 2. Angular 22 GA: signal-first era is stable

The shift from "experimental" to "stable" for Signal Forms, Resource API, Zoneless, and OnPush-default is a consolidation. Angular 22 is not an experiment anymore -- it is the default way to write Angular. Nx generators that produce `Default` change detection or `FormsModule`-based forms now produce suboptimal scaffolding.

### 3. Vite 8.1 beta: full-bundle mode is a new surface

Full-bundle mode in 8.1 (experimental) changes Vite's output model. Where Vite has traditionally been dev-server-first with Rolldown for production, full-bundle mode suggests a single pipeline for both. This could affect `@nx/vite`'s inferred targets and build caching assumptions. Watch carefully.

### 4. React Router rapid cadence (7.17.0)

React Router has shipped 6 minors since December at a ~1/month pace. This is healthier than the prior low-cadence period. The `@nx/remix` peer range needs to be kept current or users on 7.16-7.17 will see peer conflict warnings.

### 5. Next.js 16.3 GA slipping

The 3-4 week canary cadence has now been running for 10 weeks without a GA. Something in the Cache Components architecture is causing delays. `@nx/next` teams should plan for August, not June.
