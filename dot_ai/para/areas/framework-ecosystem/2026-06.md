## Frameworks

### Angular

**Current stable:** 22.0.0 (June 3, 2026) | **LTS prior:** 21.2.11

**Releases since May W22:**
- v22.0.0 (June 3) -- GA
- v22.0.0-rc.0 (May 13), rc.1 (May 20), next.11 (May 7), next.12 (May 13) -- absorbed into GA

**Angular 22 GA headline features:**
- **Signal Forms: stable** (was experimental in v21)
- **OnPush: new default** for new components (migration sets `Eager` on existing)
- **@angular/aria: stable** -- accessible UI components with Signal Forms integration + test harnesses
- **MCP server: stable** -- Angular CLI ships `devserver.*`, code-gen, package-add tools; `onpush_zoneless_migration` MCP tool
- **Vitest: primary test runner** -- Jest and Web Test Runner experimental support removed; `migrate-karma-to-vitest` schematic in Angular CLI
- **Resource API: stable** -- signal-based async data loading

**Cumulative breaking surface (all now shipping):**
- Node 20 dropped (Node 22.22.0+ required)
- TypeScript 5.9 dropped (TS 6+ required)
- FetchBackend is default HTTP backend (`withXhr()` migration)
- `ComponentFactoryResolver` / `ComponentFactory` removed
- `paramsInheritanceStrategy` default: `'emptyOnly'` -> `'always'`
- `forms.min`/`max` no longer accept strings
- `reportProgress` deprecated -> `reportUploadProgress`/`reportDownloadProgress`

**@nx/angular impact:** P0. The GA makes all May P0/P1 action items overdue. Most urgent: switch CNW/generator default test runner to Vitest, update component generator for OnPush default, gate on Node 22+/TS 6. The `migrate-karma-to-vitest` and `onpush_zoneless_migration` Angular CLI migrations should be surfaced or re-exposed in `@nx/angular`. Angular Aria + MCP server are generator/docs opportunities.

**Action needed:** YES -- Vitest default, OnPush generator default, Node/TS peer gates, Signal Forms generator, migration schematic wiring. All overdue.

## Cross-Cutting Themes

### 1. Angular 22 GA resets the Nx action queue

Every P1 and P0 flagged for Angular in March/April/May is now live and user-facing. The most surprising addition -- Vitest as the primary test runner with Jest removed -- was not flagged in prior reports as a near-term default swap. This is the most impactful single change for `@nx/angular` generators this cycle.

### 2. AI/MCP is no longer experimental in Angular

Angular 22 ships a stable MCP server in the CLI. Combined with Angular Aria's AI accessibility testing tooling and the `onpush_zoneless_migration` MCP tool, Angular has a clear AI-agentic story. Nx's own MCP server should compare feature surface and consider Angular-specific tools.

### 3. React Router v8 pre-announcement is the React ecosystem's next forcing function

7.16.0's v8 future-flag warnings are the first concrete signal since React Router 7 that another major re-architecture is coming. The requirement for Vite 7+ (minimum) at v8 time means the entire React Router 7 -> React Router 8 -> `@nx/remix` deprecation story needs to be re-evaluated. This will affect `@nx/react` template choices as well.

### 4. Rolldown 1.1.0 crosses another "default behavior" line

With `lazyBarrel` enabled by default, Rolldown is maturing from "stable" to "opinionated about performance defaults". Each new default that ships in Rolldown flows through Vite to `@nx/vite` users without their knowing. Snapshot regression coverage needs to track Vite patch days.

### 5. Nuxt security hotfix underlines peer-range hygiene

A CVE-resolving patch (4.4.7) appearing in a framework that `@nx/nuxt` peers against is a reminder to keep peer ranges non-pinned so users naturally receive security fixes. If `@nx/nuxt` pegs to `^4.4.6` today, users don't get 4.4.7 automatically.
