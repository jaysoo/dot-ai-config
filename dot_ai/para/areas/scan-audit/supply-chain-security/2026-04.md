## New Supply Chain Hardening (this lookback window)

### 2026-04-07 — Transitive dep pinning via `expand-deps` (PR #35159)

Commit `2521f98b6d`: At publish time, the entire transitive dependency tree of `nx` is flattened into explicit pinned direct dependencies in `packages/nx/package.json`. This removes resolver freedom for ~76 transitive deps that previously could be silently swapped by a compromised dep-of-a-dep at install time.

**Impact:** significant hardening against Shai-Hulud-class transitive-dep attacks. Every version is now frozen at publish time based on what Nx CI tested.

Referenced in memory as `NXC-4197`. Verified shipped in `nx@22.6.5`.

### 2026-04-01 — Published-artifact hygiene (PR #35109)

Commit `19bac461a2`: `nx` package now uses `files` allowlist instead of `.npmignore` blocklist. Removes ~137 Rust source files, test fixtures, snapshots, and duplicate source dirs from the published artifact. Reduces attack surface for supply chain confusion (fewer files shipped = fewer places for an attacker to hide malicious changes).
