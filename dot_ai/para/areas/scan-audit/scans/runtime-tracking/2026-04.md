### Node 24.15.0 (Apr 16, LTS) — key changes

- **Stabilizations**: `require(esm)` **stable**, module compile cache **stable**. Both directly relevant to Nx plugin loader and daemon cold start.
- **New flags**: `--max-heap-size`, `--require-module` / `--no-require-module`.
- **New APIs**: `fs.stat()` / `fs.promises.stat()` gain `throwIfNoEntry` option; `net.Socket` gets `setTOS()` / `getTOS()`; HTTP/2 `http1Options` for H1 fallback; SQLite `DatabaseSync.limits`; C++ diagnostics channel support; test runner exposes worker ID and shows interrupted tests on SIGINT.
- **Stream**: `Duplex.toWeb()` option renamed `type` → `readableType` (minor cleanup, not breaking yet).
- **Deprecation**: `Module.register()` → **DEP0205**.
- **Module resolution**: `require.resolve()` now passes through `module.registerHooks()`; coverage fixes for mocked CJS modules imported from ESM.
- **Deps**: npm 11.12.1, SQLite 3.51.3, Ada 3.4.4, V8 cherry-picks, OpenSSL 4.0 compile support enabled (not linked yet).

### Node 25.9.0 (Apr 1, Current) — key changes

- **Test runner mock consolidation (SEMVER-MINOR)**: `MockModuleOptions.defaultExport` + `namedExports` merged into a single `exports` option. Codemod available: `npx codemod @nodejs/mock-module-exports`. Nx `@nx/node` test setups using `node:test` mocks need to update.
- **AsyncLocalStorage `using` scope** — explicit resource management integration. Cleaner pattern for request-scoped state; worth adopting in Nx Cloud server code.
- **REPL** drops `node:domain` dep and allows custom error handling.
- **Crypto**: TurboSHAKE, KangarooTwelve added; ML-KEM/ML-DSA PKCS#8 imports now validate seed.
- **New experimental**: `stream/iter`.
- **SEA**: code cache support for ESM entrypoints.
- **New flag**: `--max-heap-size` (same as 24.15.0).
- **Deprecations**: `Module.register()` DEP0205; `CryptoKey` in `node:crypto` deprecated in favor of Web Crypto.
- **Deps**: npm 11.12.1, OpenSSL 4.0 build compat, SQLite 3.51.3, ngtcp2 1.21.0, tzdata 2026a.

### Bun 1.3.13 (Apr 20) — the test-runner parity release

This is the release that tips `bun test` into credible Jest/Vitest replacement territory for CI:

- **`bun test --isolate`**: fresh global env per test file, same process, with VM-level transpilation cache for shared deps. This is the feature that makes Bun viable for codebases that rely on global mutation isolation (which is most Jest suites).
- **`bun test --parallel[=N]`**: distributes test files across worker processes, defaults to CPU count, auto-buffers output.
- **`bun test --shard=M/N`**: deterministic round-robin sharding for CI jobs — direct feature-parity with Jest/Vitest `--shard`.
- **`bun test --changed`**: git-aware filtering; accepts a commit/branch.
- **Snapshot fixes**: counter reset issues with `--rerun-each` and `retry` resolved.

Non-test highlights:
- Install: streaming tarball extraction → **17x less memory** on large repos. Isolated linker mode on peer-heavy monorepos (quoted 20.5s → 2.4s).
- Source maps: 20 → 2.4 bytes/mapping (8x memory reduction).
- Runtime footprint: -5% via mimalloc v2→v3 and libpas scavenger.
- Perf: gzip 5.5x faster (zlib-ng), array iteration 1.43x, JSC bumped +1316 commits (IC caching, SIMD).
- Node.js compat fixes: Worker lifecycle crash, `tls.connect` servername, `process.ppid` dynamic, `socket.setTimeout`, HTTP/2 `ENABLE_PUSH`, `os.freemem()` on Linux, `stat()` EINTR on macOS.
- Additions: Range requests in `Bun.serve()` (RFC 9110), `ws+unix://` schemes, SHA3 in WebCrypto, X25519 `deriveBits()`.
- **No breaking changes documented.**

### Bun 1.3.12 (Apr 10) — headline features

- **`Bun.WebView`**: native headless browser — WebKit (macOS) / Chrome (cross-platform). Methods `navigate()`, `click()`, `screenshot()`, `evaluate()` with OS-level event dispatch. Not Nx-relevant directly but a competitive signal on ambition.
- **Markdown**: `bun ./file.md` renders in terminal; `Bun.markdown.ansi()` API with hyperlinks + Kitty Graphics support.
- **`Bun.cron()` in-process**: callback-based scheduler, `--hot` reload integration, `using` disposal support. Different from 1.3.11's OS-level `Bun.cron`.
- **Explicit Resource Management** in JavaScriptCore (`using` / `await using`).
- **Perf**: `URLPattern` 2.3x, `Bun.stripANSI` 4x (SIMD), `Bun.Glob.scan()` 2x on `**/X/...`, `bun build` 1.43–1.47x on 2-core.
- **Node compat**: `process.env` in restricted dirs, `dns.getDefaultResultOrder()`, `fs.stat` large inodes, `Error.captureStackTrace` async frames, `vm.Script` mem leak fix, seccomp fallback for `statx`.
- **No breaking changes documented.**

- **Test runner**: Parity threat from 1.3.13 is now real. Scope a `@nx/bun` test executor or add `bun:test` support to `@nx/jest`.
- **Lockfile parsing**: Still the #1 friction point. [nrwl/nx#30607](https://github.com/nrwl/nx/issues/30607), [nrwl/nx#26565](https://github.com/nrwl/nx/issues/26565), [nrwl/nx#29494](https://github.com/nrwl/nx/issues/29494), [nrwl/nx#32853](https://github.com/nrwl/nx/issues/32853).

### Deno 2.7.13 (Apr 22 — today, new since last scan)

- **`node:repl`** module added.
- **`node:http` rewrite**: llhttp parser + native TCPWrap. Major step in Node parity.
- **Upgrade from PR**: `deno upgrade` can install from a PR branch.
- **HTTP/2**: fixes to constants, error codes, settings validation.
- **TLS**: validation, pipe, and JS-stream compat improvements.
- **`ChildProcess.kill()`** now returns false when process already exited (matches Node).
- **Module resolution**: fixes for `Module._resolveLookupPaths` and `require.resolve`.
- **Windows**: Playwright + file-ops fixes.
- **Crypto**: RSA-PSS in `crypto.verify` with null algorithm.
- **NAPI**: real V8 handle scopes, callback scopes, zero-copy external Latin-1 strings.
- **Perf**: Command.output() memory reduction; elided async_hooks emissions when unused; flush-free HTTP parser header fast path.

### Deno 2.7.12 (Apr 9)

- `fs.SyncWriteStream` added.
- **llhttp-based HTTPParser** with native cppgc binding (precursor to 2.7.13's http rewrite).
- **Native TLSWrap** implementation.
- **Native uv_pipe_t** with `NativePipe` and `FdTable`; TCP/pipe from FDs.
- Worker-threads memory leak fix (UvLoopInner drop).
- Non-blocking stdio `WouldBlock` handling.
- Permissions hardening: deny rules checked against resolved IPs; IPv4-mapped IPv6 normalized.
- `child_process.spawnSync` supports `timeout`, `killSignal`, `pid`.
- AWS-LC-SYS EVP API for ChaCha20-Poly1305.

### Deno 2.7.11 (Apr 1)

- **Alpha & Beta release channels** introduced.
- `fs.watch` now fires events on immediate writes.
- `deno watch` graceful shutdown cut from 5s → 500ms.

### Nx compatibility status

Still not a first-class Nx runtime. Node-parity work keeps accelerating — between Mar 18 (2.7.6) and Apr 22 (2.7.13) Deno rewrote TTY, TLS, HTTP, and pipes to match Node/libuv. No community demand signal strong enough to prioritize an `@nx/deno` runtime story this cycle.
