## Medium Priority

### M1. `json` input type shipped without docs (NEW)

**Status**: New 2026-04-22. Introduced by PR #35248 during this audit window.

The `inputs` configuration in `nx.json` / `project.json` now accepts a new object shape:

```jsonc
{ "json": "{projectRoot}/package.json", "fields": ["engines"] }
{ "json": "{workspaceRoot}/tsconfig.base.json", "fields": ["compilerOptions.*", "!compilerOptions.paths"] }
```

This enables selective JSON field hashing (previously only `tsconfig` had hardcoded selective hashing in the native hasher).

**Drift**: `astro-docs/src/content/docs/reference/inputs.mdoc` has no mention of `json` input type. It documents only `fileset`, `runtime`, `env`, `externalDependencies`, `dependentTasksOutputFiles`, and the root-tsconfig-subset behavior.

**Impact**: Users reading the input reference will not know this feature exists. Plugin authors setting up inputs for json-heavy tasks (release workflows, package.json-sensitive caching) will miss the granular invalidation option.

**Fix**: Add a "JSON field inputs" subsection to `inputs.mdoc`, with examples and `fields` pattern syntax (including negation via `!`).

### M2. `@nx/js` has ~29 exported functions with missing JSDoc

**Status**: Unchanged from 2026-02. Not affected by docs consolidation (JSDoc is on source, not docs).

Of ~35 public function exports in `@nx/js`, only ~3 have JSDoc comments (`getHelperDependency`, `resolveModuleByImport`, `isValidVariable`). Missing on all TSConfig utilities, AST utilities, `printDiagnostics`, `runTypeCheck`, `loadTsTransformers`, etc.

## Recommendations

### Immediate (this week)

1. **Document the new `json` input type** in `astro-docs/src/content/docs/reference/inputs.mdoc` — add an object-shape example with `fields` syntax, alongside the existing `fileset`/`runtime`/`env` sections. PR #35248 landed without docs; the window to add them before the next release is small.

### Near-term (this quarter)

2. **Add `@nx/js` to the TypeDoc API reference pipeline** (H1). This is unchanged from prior audits and remains the single largest API-documentation gap. Options: (a) add a `loadDevkitPackage`-style handler for `@nx/js` in `nx-reference-packages.loader.ts`, or (b) extend `plugin.loader.ts` to emit a second doc type for non-generator exports.

3. **Evaluate promoting top semi-public devkit utils to public API** (H2): `forEachExecutorOptions` (26 users), `addPlugin` (23), `getNamedInputs` (22). These are de-facto public.

4. **Populate `astro-docs/src/content/docs/reference/Deprecated/`** with the 8+ items past their removal deadlines. The directory exists but is unpopulated; this is a low-cost fix with high signal for users tracking migrations.

### Long-term

5. **Establish a `devkit-internals` stability tier** for commonly-needed internal exports (e.g. `PluginCache`, `safeWriteFileCache`, `registerTsProject`). Third-party plugin authors are already consuming these.

6. **Remove deprecated items 2+ majors past deadline** — `ProjectGraphProcessor`, `stripSourceCode`, `TypescriptImportLocator`, pre-v20 `TaskHasher` overloads. Include in the next major release cut.

## Methodology notes (for future runs)

- **Do not use WebFetch against raw.githubusercontent.com for line/export counts.** It strips comments and blank lines, producing undercounts (observed: 236 lines vs actual 261; 32 exports vs actual 39). Always use the local clone at `/tmp/nx-scan-*` for file content measurements.
- **Post-consolidation, `docs/packages.json` no longer exists.** Drift measurement is now: (a) `packages/*/generators.json` + `executors.json` vs `astro-docs/src/plugins/utils/plugin-mappings.ts` (must have an entry for each package), and (b) new public API (types, env vars, input schemas, CLI commands) vs corresponding `astro-docs/src/content/docs/reference/*.mdoc` pages.
- **The `plugin.loader.ts` auto-scan means per-package generator/executor drift is effectively zero** as long as `pluginToTechnology` has an entry for the package. Next audit should verify new packages (if any) get added to this map.
- **Treat devkit-exports.ts / devkit-internals.ts / public-api.ts as stable unless `gh api repos/nrwl/nx/commits path=<file>` returns commits in the window.** Don't re-count by extraction — `git log -p` the file if anything changed.
