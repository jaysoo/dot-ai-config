### H6. `nx/release/changelog-renderer` package export has no documentation

**Status**: New finding 2026-03-27.

PR #35033 added `./release/changelog-renderer` to the `nx` package.json `exports` field, making it a public API for importing/extending the default changelog renderer. This was added because "not having this causes issues with our documented approach on Node 24" (ESM resolution). The export maps to `./dist/release/changelog-renderer/index.js`.

No documentation page exists for this export path. Users creating custom changelog renderers need to know this entry point exists and what it exports. This is particularly important because:
- The Nx release docs reference custom changelog renderers
- Node 24 ESM resolution changes make explicit exports entries required
- Without docs, users won't know the import path changed from deep imports to `nx/release/changelog-renderer`

**Impact**: Users following the custom changelog renderer guide may use incorrect import paths, especially on Node 24+.

### M11. Vite 8 support landed but docs don't mention compatibility

**Status**: New finding 2026-03-27.

No new generators or executors were added, but the compatibility matrix in docs should be updated to reflect Vite 8 support. The `@nx/vite` and `@nx/vitest` docs pages don't mention supported Vite versions.

**Impact**: Users on Vite 8 may not know Nx supports it, or may encounter `rolldownOptions` vs `rollupOptions` confusion.

Add `application`, `browser-esbuild`, `extract-i18n`, `unit-test` executors.
