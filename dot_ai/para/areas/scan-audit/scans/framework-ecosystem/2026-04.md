#### Key facts -- Rspack 2.0 GA (Apr 22)

**Ecosystem scale:** 100K -> 5M weekly npm downloads since 1.0.

**Breaking changes (authoritative list):**

- **Pure ESM packages** -- `@rspack/core`, `@rspack/cli`, `@rspack/dev-server` have no CJS build.
- **Node 19+** (rspack.rs blog says Node 19+; config docs previously stated Node 20+ -- treat as "at minimum Node 20 for safety").

  - `experiments.SubResourceIntegrityPlugin`

  - `output.charset`
  - Legacy `output.library*` options
  - `WarnCaseSensitiveModulesPlugin`
  - `plugin.getHooks()`
- **Config renames:**
  - `output.chunkLoadingGlobal`: `webpackChunk*` -> `rspackChunk*`
  - `output.hotUpdateGlobal`: `webpackHotUpdate*` -> `rspackHotUpdate*`
  - `rspackExperiments.import` in swc-loader -> top-level `transformImport`
- **Default changes:**
  - CSS enabled by default.
  - `asyncWebAssembly` enabled by default.
  - `target` cascades to SWC + LightningCSS defaults.
  - `.swcrc` reading disabled by default.
  - `module.parser.javascript.exportsPresence` -> `'error'`.
- **Dev server** is now an optional peer dep; `@rspack/dev-server` v2 required separately.
- **Resolver:** multi-yarn-PnP project support dropped; added `pnp_manifest` option.

**New features:**

- **React Server Components** built-in (directive handling, compile-time validation, CSS collection across server/client components, HMR).
- **ESM enhancements:** `import.meta` preserves unknown fields; `import.defer()` + `import defer` syntax; new `library: { type: "modern-module" }` output.
- **Module Federation tree-shaking at export level** -- shared dependencies pruned based on actual imports. Big deal for MF bundle size.
- **Subpath alias imports** via package.json `imports` field.
- **`detectSyntax` option for swc-loader** -- auto-infers TS/JSX from extension (one rule handles multiple file types).
- **CSS `@import` control** via `resolveImport` option.
- Hashed module IDs + `enforceSizeThreshold` for code splitting.

**Performance / size:**

- 10% perf over 1.7; up to 100% vs 1.0.
- Dev-server 192 -> 1 dependency, 15 MB -> 1.4 MB install size.
- CLI: zero dependencies.

**Migration:** Upgrade guide published; `npx skills add` for agent-assisted migration.

#### `@nx/rspack` + `@nx/module-federation` impact (P0)

- **ESM-only is the single biggest required change.** Any `require('@rspack/core')` call in plugin code must become dynamic `import()` or the plugin must become ESM. Audit required.
- **Global renames** (`webpackChunk*` -> `rspackChunk*`, `webpackHotUpdate*` -> `rspackHotUpdate*`) affect any Nx-generated Module Federation remote that reads the global at runtime. Check generators AND runtime shims.
- **MF export-level tree-shaking** is a headline feature worth surfacing in `@nx/module-federation` generator options -- users will expect Nx to expose it.
- **RSC built-in** -- `@nx/rspack` should add a generator option for RSC projects (parallel to `@nx/next`'s RSC pattern).
- **`.swcrc` disabled by default** -- existing Nx-generated `.swcrc` files will be silently ignored on v2 unless generators flip the opt-in flag.
- **Removed experiments** (`rspackFuture`, `parallelLoader`, SRI plugin) -- grep Nx codebase + generator templates for these identifiers.
- **`detectSyntax` in swc-loader** -- opportunity to simplify generator output (single rule instead of separate `.ts`/`.tsx` entries).
- **Node 19+ / Node 20+** should align with current Nx Node floor already.

**Priority: bump to P0 from P1.** Rspack 2 stable is live *today*.

## Cross-Cutting Themes

### 1. TypeScript 6.0 floor hardening

Angular 22-next.8 (Apr 15) dropped TypeScript < 6.0 entirely. Rspack 2.0 swc-loader's `verbatimModuleSyntax` default (when swc enforces TS 6-style `type` imports) amplifies the pressure. TS 6.0.3 is now stable. Every Nx generator that emits `tsconfig.base.json` is on the clock to match TS 6 defaults (explicit `types: []`, `moduleResolution: bundler|nodenext`, no bare `baseUrl`).

### 2. ESM-only ecosystem lock-in

Rspack 2.0 shipped pure ESM. Vite 8 already effectively ESM-first. Remix 3.0 alpha is modular sub-path ESM. The CJS fallback era is ending fast. `@nx/rspack` cannot remain CJS.

### 3. Rolldown as the convergence layer

Rolldown powers Vite 8, ships in Rspack's toolchain (via swc/oxc primitives), and is Rspack 2's bundler soul. When Rolldown 1.0 stable lands, it's effectively the JavaScript bundler convergence point. Worth a `@nx/rolldown` feasibility spike.

### 4. RSC in the bundlers layer

Rspack 2 built-in RSC + Module Federation tree-shaking = bundler-level primitives Nx users will assume Nx exposes. `@nx/next` already handles RSC; `@nx/rspack` + `@nx/module-federation` now need equivalent affordances.

### 5. Supply chain signals

esbuild 0.28.0 adds hash verification on fallback install. Rspack 2.0 dropped 99% of dev-server deps. Both gestures match Nx's own supply-chain posture after the 2024 npm compromise incident. Worth citing in `audit-supply-chain` outputs.
