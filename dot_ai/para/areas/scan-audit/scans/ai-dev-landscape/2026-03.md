# AI Dev Tools Landscape -- March 2026

_Lookback window: 2026-01-23 to 2026-03-24_
_Last updated: 2026-03-24_
_Sources: GitHub releases, official blogs, web research (all claims sourced from live data)_

---

## TL;DR

The AI coding tools space is consolidating around three axes: **1M-token context windows** (Claude Opus 4.6 GA at standard pricing), **cross-IDE agent portability** (JetBrains ACP + Cursor), and **proprietary coding models** (Cursor Composer 2 beats Opus 4.6 on Terminal-Bench at 1/10th the price). **Cursor launched Composer 2** (Mar 19) but immediately faced a credibility crisis: **Composer 2 is built on Moonshot AI's Kimi 2.5** (Mar 22), an open-source Chinese model -- Cursor initially failed to disclose this, sparking backlash about transparency and US-China AI competition optics. Claude Code reached 2.1.81 with `--bare` flag for scripted calls and `--channels` permission relay. **GPT-5.2-Codex** launched as OpenAI's most advanced agentic coding model with context compaction and cybersecurity improvements. Codex CLI hit 0.116.0 with `userpromptsubmit` hooks and web search. **Copilot CLI v1.0.10** (Mar 20) added multiple concurrent sessions, SDK elicitation, and reduced memory. **Awesome GitHub Copilot** launched as a default plugin marketplace (175+ agents, 208+ skills, 48+ plugins). GPT-5.4 mini and nano launched (Mar 17) as OpenAI's most capable small models. **MCP faces a credibility challenge**: Perplexity publicly moved away from MCP (Mar 11), Cloudflare introduced Code Mode as an alternative to tool-calling, and 30+ CVEs were filed in 60 days. MCP SDKs are hardening (OpenAI Agents SDK v0.12.x ships MCP retry and error normalization). **Amazon's Kiro mandate caused a 6-hour shopping outage** (Mar 5) leading to mandatory senior review for AI-assisted deployments. Windsurf launched Codemaps -- AI-annotated code structure maps. Nx published "Agentic Experience Is the New Developer Experience" (Mar 5) and continues the skills-first pivot. For Nx: the skills-over-tools pivot is validated by both industry convergence AND the MCP backlash. The Composer 2/Kimi controversy highlights the fragility of the proprietary-model narrative -- tool vendors building on open-source bases adds supply chain risk awareness to model selection.

## 1. MCP Ecosystem

### Protocol Status

- **Current spec**: 2025-11-25 (Streamable HTTP transport) -- no new spec release since November 2025 ([spec](https://modelcontextprotocol.io/specification/2025-11-25))
- **2026 Roadmap published (Mar 9)**: Reorganized around priority areas instead of release dates. Four focus areas: transport scalability (horizontal scaling, stateless operation), agent communication, governance maturation (Working Groups as primary vehicle per SEP-1302), enterprise readiness (audit trails, observability). "On the horizon": triggers/event-driven updates, streamed results, deeper security ([blog](http://blog.modelcontextprotocol.io/posts/2026-mcp-roadmap/), [The New Stack](https://thenewstack.io/model-context-protocol-roadmap-2026/))
- **Active SEPs**: DPoP authentication extension, multi-turn SSE transport, Server Cards for discovery, Tasks abstraction (tracking async work), client security requirements (SEP-1024), default scopes (SEP-835) ([roadmap](https://modelcontextprotocol.io/development/roadmap))
- **MCP Apps**: First official MCP extension, launched Jan 26. Tools return interactive UI via `ui://` URI scheme, rendered in sandboxed iframes inside conversations. Supported by ChatGPT, Claude, Goose, VS Code, and Cursor ([blog](http://blog.modelcontextprotocol.io/posts/2026-01-26-mcp-apps/), [spec](https://github.com/modelcontextprotocol/ext-apps))
- **MCP Elicitation**: Structured form input for server-initiated user interactions. Now in Copilot CLI (Mar 3) and Claude Code (v2.1.76, with Elicitation/ElicitationResult hooks)

### Agent Client Protocol (ACP) -- NEW PROTOCOL

- **JetBrains + Zed launched ACP**: Open standard for AI coding agents in any IDE, analogous to LSP but for agents. ACP Agent Registry live since Jan 28 ([JetBrains](https://blog.jetbrains.com/ai/2026/01/acp-agent-registry/), [Zed](https://zed.dev/blog/jetbrains-on-acp))
- **Cursor joined ACP Registry (Mar 4)**: Cursor now available in IntelliJ IDEA, PyCharm, WebStorm via ACP. Full access to project with any frontier model ([Cursor blog](https://cursor.com/blog/jetbrains-acp), [JetBrains](https://blog.jetbrains.com/ai/2026/03/cursor-joined-the-acp-registry-and-is-now-live-in-your-jetbrains-ide/))
- **Kiro supports ACP**: CLI documentation for ACP integration ([Kiro docs](https://kiro.dev/docs/cli/acp/))
- **Significance**: ACP is emerging as MCP's complement -- MCP connects agents to tools/data, ACP connects agents to IDEs. Together they define the "agent infrastructure" layer

### MCP Backlash & Alternatives -- NEW (Mar 11-17)

- **Perplexity moving away from MCP** (Mar 11): At Ask 2026 conference, CTO Denis Yarats announced Perplexity is moving away from MCP internally. Reasons: tool schema overhead consumes up to 72% of context window before processing user input, authentication complexity (each server handles own auth), most MCP features go unused in production. Replacing with Agent API (managed runtime, multi-model routing). Still supports MCP for external integrations (e.g., Claude Desktop) ([Awesome Agents](https://awesomeagents.ai/news/perplexity-agent-api-mcp-shift/), [Junia](https://www.junia.ai/blog/perplexity-mcp-vs-apis-ai-agents))
- **Cloudflare Code Mode**: Keeps MCP for discovery but replaces tool-calling with code generation. Converts MCP tools into TypeScript API, LLM writes code against it. Their API MCP server covers 2,500 endpoints using 2 tools and ~1,000 tokens; native MCP would consume 244,000+ tokens. Cuts token usage by 32% (simple) to 81% (complex batch) vs standard MCP ([Cloudflare blog](https://blog.cloudflare.com/code-mode-mcp/), [WorkOS](https://workos.com/blog/cloudflare-code-mode-cuts-token-usage-by-81))
- **Y Combinator CEO Garry Tan** independently built CLI-based agent integration over MCP, citing reliability and speed
- **Significance for Nx**: This validates Nx's "skills over tools" approach. The MCP backlash is specifically about naive tool exposure (loading schemas into context). Nx's lean MCP server (CI-focused remote services) + skills architecture avoids the exact problems Perplexity and Cloudflare identified

### MCP Security Crisis -- NEW (Jan-Mar 2026)

- **30+ CVEs filed in 60 days** (Jan-Feb 2026): 43% exec/shell injection, 20% tooling infrastructure flaws, 13% auth bypass. Includes CVSS 9.6 RCE in a package with ~500k downloads ([HN](https://news.ycombinator.com/item?id=47356600), [DEV](https://dev.to/kai_security_ai/30-cves-later-how-mcps-attack-surface-expanded-into-three-distinct-layers-ihp))
- **OWASP published MCP Top 10**: Formalizing MCP-specific security risks ([OWASP](https://owasp.org/www-project-mcp-top-10/))
- **Exposed servers**: Knostic found 1,862 internet-exposed MCP servers; all 119 manually verified had no authentication. Bitsight found ~1,000 with zero authorization. Separate analysis: 41% of 518 servers had no auth at all ([Trend Micro](https://www.trendmicro.com/vinfo/us/security/news/cybercrime-and-digital-threats/mcp-security-network-exposed-servers-are-backdoors-to-your-private-data))
- **Root causes**: Not exotic zero-days -- missing input validation, absent authentication, blind trust in tool descriptions
- **SurePath AI**: Launched real-time MCP policy controls for governing AI agent actions ([PR Newswire](https://www.prnewswire.com/news-releases/surepath-ai-advances-real-time-model-context-protocol-mcp-policy-controls-to-govern-ai-actions-302709875.html))

### Competitive Assessment

Cursor's March was transformative but ended with a credibility hit. **Composer 2 made Cursor a model provider** -- but the Kimi 2.5 disclosure (Mar 22) revealed the model is built on a Chinese open-source base, undermining the "proprietary frontier model" narrative. The 10x cost advantage is partially explained by open-source foundations rather than pure efficiency gains. Enterprise customers with supply chain requirements will note the provenance. That said, the underlying product momentum is real: 30+ new plugins, JetBrains ACP entry, Automations, and cloud agents make Cursor the most complete AI coding platform. For Nx: the Kimi controversy adds nuance to the "tool vendors becoming model providers" trend -- it's more accurately "tool vendors fine-tuning open-source models," which lowers the moat. Nx plugin distribution in Cursor's marketplace remains important as Cursor captures more of the developer workflow.

## 15. Implications for Nx Roadmap

### VALIDATE (Nx is on the right track)

1. **Skills-first architecture is validated by MCP backlash**. Perplexity's CTO publicly cited 72% context waste from MCP tool schemas (Mar 11). Cloudflare's Code Mode shows tool-calling isn't the only pattern. Nx's "Why we deleted (most of) our MCP tools" blog was prescient -- the industry is now discovering what Nx articulated in February.

2. **MCP has won as infrastructure BUT faces growing pains**. Apple Xcode, Azure Visual Studio 2026, 20k+ servers, MCP Dev Summit. However: 30+ CVEs in 60 days, OWASP MCP Top 10, and Perplexity/Cloudflare/YC alternatives signal that naive MCP adoption is problematic. Nx's lean MCP server (CI-focused remote services only) avoids the exact problems others are hitting.

3. **Self-Healing CI is uniquely differentiated AND validated by Amazon's failures**. The Kiro outage saga (6-hour shopping outage, mandatory senior review) demonstrates that unverified AI-generated code in production is dangerous. Nx's "validate before apply" approach is the responsible pattern. No competitor has monorepo-aware CI + verification. The new interactive diffs, GitHub integration, and auto-apply features strengthen the product.

4. **Plugin/skill distribution is converging**. Claude Code, Codex (60+ plugins via @mention), Cursor (60+ marketplace), Copilot (`/extensions`), and Turborepo all ship distribution systems. Nx already has this with AI Agent Skills plugin.

5. **Agent config convergence validates `configure-ai-agents`**. AGENTS.md (60k repos), CLAUDE.md (read by Windsurf), Copilot CLI AGENTS.md + JetBrains support all confirm agent config files are a cross-tool standard.

6. **"Agentic Experience" framing is correct**. Nx's Mar 5 blog ("AX is the new DX") aligns with the industry shift from developer tools to agent-ready infrastructure. Copilot's semantic code search (Mar 17) and Codex's terminal reading show agents need ergonomic interfaces, not just data access.

### CHALLENGE (Assumptions to re-examine)

1. **Cursor's vertical integration is the most complete AI coding platform**. ## 16. Risks

1. **Cursor as CI platform via plugins + automations**. Datadog + GitLab + PagerDuty plugins feeding into event-driven Automations is an ad-hoc CI analysis stack. Not monorepo-aware today, but the plugin ecosystem grows weekly (60+ plugins now). Mitigation: Nx's graph-awareness moat remains, but the "good enough" threshold for non-monorepo users is getting lower.

2. **GitHub vertical integration deepening**. Copilot CLI 1.0 GA + `/extensions` + JetBrains agent GA + Memory (default) + AGENTS.md + Enterprise Controls + Agentic Workflows = full AI-native dev stack across VS Code AND JetBrains. If GitHub adds monorepo awareness (they have codeowners, dependency graph data), the gap narrows. Window: ~6-12 months.

3. **1M context windows reduce context-provision value**. When entire monorepos fit in a single context window, tools that provide context (MCP, skills) are less differentiated. Nx's value shifts entirely to *workflow intelligence* (what to build, what's affected, how to fix) rather than *context provision* (here's the project graph JSON).

4. **Amazon-induced enterprise AI-CI skepticism**. The Kiro outage saga may make enterprise buyers cautious about AI-in-CI. Nx must proactively differentiate "AI that deploys unverified code" (Amazon/Kiro) from "AI that proposes verified fixes for human review" (Self-Healing CI).

5. **Agent config fragmentation accelerating**. Cursor rules, Copilot Memory, AGENTS.md, CLAUDE.md, cascade hooks, Kiro spec configs, Turborepo task descriptions, ACP agent configs -- the number of output targets for `configure-ai-agents` is growing faster than the team can maintain.

6. **Revenue concentration in AI tools**. Claude Code ($2.5B) and Cursor ($2B ARR) are massive, well-funded competitors. They can invest heavily in CI/monorepo features if they choose to. Neither has done so yet, but the resources exist.

7. **Protocol AND pattern fragmentation risk**. MCP for tools, ACP for IDEs, A2A for agent-to-agent, plus Cloudflare Code Mode, Perplexity Agent API, and CLI-first approaches. The integration landscape is fragmenting beyond protocols into fundamentally different patterns (tool-calling vs. code-generation vs. direct API). Nx may need to support multiple integration patterns, not just protocols.

8. **MCP security reputation risk**. 30+ CVEs, OWASP Top 10, public backlash from Perplexity/Cloudflare could taint "MCP" as a brand. Nx MCP server may need to proactively communicate its security posture to avoid guilt by association with the broader ecosystem's problems.

9. **Cursor vertical integration moat (weakened by Kimi controversy)**. Composer 2 + marketplace + automations + JetBrains ACP + cloud agents = most complete AI coding platform. However, the Kimi 2.5 disclosure (Mar 22) weakens the model moat -- enterprise buyers may question Cursor's model provenance. If Cursor adds monorepo awareness (even shallow), it could capture the "good enough" tier of monorepo users. Cursor's $2B ARR funds rapid expansion. Mitigation: Nx's graph-awareness moat and enterprise CI features remain deep, and Cursor's credibility took a hit.

10. **Model cost race to bottom**. Composer 2 at $0.50/M input (10x cheaper than Opus 4.6) and GPT-5.4 mini at $0.75/M create downward pricing pressure. This benefits Nx (cheaper Self-Healing CI) but also enables competitors to build AI-CI features more cheaply.
