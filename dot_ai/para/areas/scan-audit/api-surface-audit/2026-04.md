## Methodology Notes

- Auto-generation pipeline traced through `astro-docs/src/content.config.ts` → `PluginLoader` / `NxReferencePackagesLoader` / `CommunityPluginsLoader`.
- `PluginLoader` skips `PLUGIN_IGNORE_LIST = ['add-nx-to-monorepo', 'cra-to-nx', 'create-nx-plugin', 'create-nx-workspace', 'make-angular-cli-faster', 'tao']`. Of these, only `create-nx-plugin` and `create-nx-workspace` exist in the current `packages/` directory. CNW is explicitly handled by `loadCnwPackage`; `create-nx-plugin` is not — hence finding #5.
- `parseGenerators`/`parseExecutors` in `plugin-schema-parser.ts` explicitly skip entries with `hidden: true`. `x-deprecated: <string>` entries are still included (with the deprecation note rendered).
- CLI extraction (`cli-subprocess.cjs`) skips commands with `describe: false` or no description. `hiddenCommands = ['$0', 'conformance', 'conformance:check']`.
- Devkit docs generated via TypeDoc from built `.d.ts`; categorization relies on `@category` tags in source.
- Deprecation wave counts are raw `grep -c` line matches, not distinct items (e.g. v22 has 9 distinct items but the 9 line count matches because one item per line in this window).
