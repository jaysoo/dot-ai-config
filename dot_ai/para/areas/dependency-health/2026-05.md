## Methodology Notes

- **Source of truth for "is the user-facing release stale?":** `time[<dist-tags.latest>]`. Republishing/deprecating bumps `time.modified` but not user behavior.
- **Ocean coverage caveat:** The full ocean repo clone failed this scan. Coverage came from npm registry metadata of 9 known cloud-published packages (`@nx/{graph,key,azure-cache,s3-cache,gcs-cache,conformance,powerpack-license}`, `@nx-cloud/runners`, `nx-cloud`). Ocean's internal services and packages without npm artifacts are not in this snapshot.
- **PeerDeps excluded:** Consistent with prior method, only `dependencies` + `devDependencies` are scanned. Notable peerDep-only packages skipped this round: `eslint-config-prettier` (in `@nx/eslint-plugin` peerDeps), pnpm-catalog refs like `@swc-node/register`.
- **Repo activity verified via `gh api repos/<owner>/<repo>`** for cytoscape plugins, signale, kill-port, http-server, node-machine-id, yarnpkg/yarn — confirms which dormant npm releases still have upstream movement.
