## Recommendations

### Immediate Actions (P0)

1. **Evaluate `enquirer` replacement** -- 31 months stale, used in nx core + both create-nx-* packages. Catalog pins `~2.3.6`. This is the single highest-risk unmaintained dependency. Candidates: `@inquirer/prompts` (MIT, actively maintained, ESM+CJS), `@clack/prompts` (MIT, modern DX).

2. **Upgrade `loader-utils`** from ^2.0.3 to ^3.x in @nx/rspack -- v2 had historical prototype pollution CVEs. Even though 2.0.4 patched them, staying on v2 is a bad signal.

### Short-term Actions (P1, Next Quarter)

3. **Remove `source-map-support`** from @nx/js -- 51 months stale, Node.js has built-in `--enable-source-maps` since v12.

4. **Replace `tcp-port-used`** (62 months) with `detect-port` (already a dep) in @nx/node.

5. **Replace `rollup-plugin-typescript2`** (29 months) with `@rollup/plugin-typescript` in @nx/rollup.

6. **Upgrade `copy-webpack-plugin`** from ^10 to ^14 in @nx/next -- 4 major versions behind and the package is actively maintained.

7. **Upgrade `fork-ts-checker-webpack-plugin`** from 7.x to 9.x in @nx/webpack.

### Medium-term Actions (P2)

8. **ESM migration unlocks**: Once Nx ships ESM, upgrade `figures`, `ora`, `string-width`, `open`, `cli-cursor`, `npm-run-path`, `cli-spinners`, `flat` -- all blocked by ESM-only majors.

9. **Upgrade `tar-stream`** from ~2.2.0 to ^3.x -- streaming API changed but v3 is actively maintained.

10. **Upgrade `dotenv`** from ~16.x to ~17.x and `dotenv-expand` from ~11.x to ~12.x.

### Low Priority (Monitor)

11. **Stable "done" packages** (`tree-kill`, `node-machine-id`, `front-matter`, `columnify`, `lines-and-columns`, `identity-obj-proxy`, `concat-with-sourcemaps`, `confusing-browser-globals`) -- No action unless security issues arise. These are small, purpose-built utilities that are effectively complete.

12. **`@yarnpkg/lockfile`** (89 months) -- Required for Yarn Classic support. No replacement possible while Yarn v1 lockfiles exist in the wild.

13. **`@ltd/j-toml`** (37 months) -- Single TOML parser dep. Alternative: `smol-toml` (active). Low priority since TOML parsing is a niche use case in Nx.
