## Recommended Actions

### Priority 1 -- Reduce risk from unmaintained packages

1. **`enquirer`** (Warning, 9 packages) -- Most impactful risk. Last publish Jul 2023. The maintainer (Jon Schlinkert) has been less active. Evaluate migrating to `@inquirer/prompts` (Inquirer.js rewrite) which is actively maintained and has ESM support.

2. **`tcp-port-used`** (Critical, @nx/node) -- Trivially replaceable with `detect-port` which Nx already uses in @nx/cypress, @nx/js, and @nx/web.

3. **`@ltd/j-toml`** (Critical, nx) -- Last publish Jan 2023. `smol-toml` is a modern, actively maintained alternative.

4. **`source-map-support`** (Warning, @nx/js) -- Node 20+ has built-in `--enable-source-maps`. Evaluate if this dep can be dropped for supported Node versions.

### Priority 2 -- Consolidation opportunities

5. **Terminal coloring**: Nx uses `chalk`, `picocolors`, AND `ansi-colors`. Consolidate to `picocolors` (already most widely used, smallest footprint).

6. **`identity-obj-proxy`** (Critical, @nx/jest) -- Stable but unmaintained. CSS module mocking has alternatives in Jest 29+ config.

7. **`figures`** and **`cli-cursor`** / **`cli-spinners`** -- All pinned to old CJS versions. These are Sindre packages that moved to ESM-only. When Nx moves to ESM, update these.

### Priority 3 -- Monitor

8. **`@phenomnomnominal/tsquery`** -- Used in 13 packages. Single maintainer (Craig Spence). No immediate risk but high bus factor.

9. **`@yarnpkg/lockfile`** -- Yarn v1 lockfile parser. Needed for legacy Yarn 1 support. Will eventually be droppable.

10. **`loader-utils`** -- Had critical CVE (CVE-2022-37601) in v2. Verify all usages are on v3.x.
