## Emerging Patterns

### 1. Rspack is the new frontier
Rspack integration has crossed the adoption threshold where edge cases are surfacing rapidly (Windows code splitting, peer dependency mismatches, angular-rspack handler accumulation, infinite reload with nested routes). This is typical of a technology moving from early adopter to mainstream.

### 2. Module Federation remains the highest-complexity surface
MF continues to generate the most multi-dimensional bugs (framework x bundler x configuration). The Angular + Rspack + MF combination is particularly fragile.

### 3. nx release needs independent projects hardening
Multiple issues suggest the independent projects workflow in nx release has gaps: incorrect version bumps from workspace-level changes, changelog generation for dependent projects, hardcoded npm detection, and git tag interpolation bugs.

### 4. Maven/Gradle is becoming a real workload
With 13 issues in 60 days and an active PR cadence (batch runner fixes, Gradle version bumps), JVM build tool integration is no longer a niche feature. The community is pushing into deeper integration scenarios (nested modules, Maven 4, custom project names).

### 5. AI integration is generating interest
Community discussions about MCP context awareness, Codex sandbox support, AI file prefixing, and pluggable Nx Agent executors signal that users are starting to think about Nx in the context of AI-assisted development workflows.

### 6. Build tool shift accelerating
@nx/vite (11.3M) has decisively overtaken @nx/webpack. @nx/rspack (5.2M) is catching up. The build tool pluralism creates maintenance pressure but reflects genuine adoption.

### 7. Triage gap is widening
45% unresponded rate (74 of 163) over 60 days, with some issues over 50 days old without any comment. This risks community perception of responsiveness.
