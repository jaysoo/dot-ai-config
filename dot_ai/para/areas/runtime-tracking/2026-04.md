### Notable Features (v24.15.0 LTS) — LANDED IN KRYPTON

**Stabilizations (big for Nx)**:
- **`require(esm)` marked stable** (#60959): CJS `require()` can now load ESM synchronously. This is the feature Joyee Cheung has been shepherding for 2+ years. Nx plugin system is mixed CJS/ESM — this should simplify `loadNxPlugin` and reduce dynamic `import()` workarounds.
- **Module compile cache marked stable** (#60971): On-disk V8 compile cache for startup perf. Nx CLI startup on warm cache could benefit — worth benchmarking against current `nx-cloud` compile cache implementation.

**New CLI flags**:
- **`--max-heap-size`** (#58708): Simpler alternative to `--max-old-space-size`. Update Nx OOM troubleshooting docs.
- **`--require-module` / `--no-require-module`** (#60959): Controls `require(esm)` behavior.

**API additions**:
- `fs.stat` / `fs.promises.stat` **`throwIfNoEntry`** option: Avoids try/catch for ENOENT checks. Nx file existence patterns simplify.
- `http2.http1Options`: HTTP/1 fallback config for HTTP/2. Relevant for Nx Cloud agents behind mixed-version proxies.
- `net.Socket.setTOS/getTOS`: QoS for TCP sockets. Minor relevance.
- `KeyObject` raw key formats: Crypto improvements.
- SQLite marked **release candidate** (from experimental).
- C++ diagnostics channels: Internal infra.

### Notable Features (v25.9.0 Current)

**Breaking (codemod available)**:
- **`MockModuleOptions.exports` consolidation** (#61727): `defaultExport` + `namedExports` merged into single `exports` option. If Nx test utilities use `node:test` module mocking, update via `npx codemod @nodejs/mock-module-exports`.

**Deprecations**:
- **DEP0205: `module.register()` deprecated**. This is the loader-hook registration API. Nx doesn't currently use loader hooks directly, but any plugin that does will need to migrate. Track replacement path.
- **CryptoKey use in `node:crypto` deprecated**: Move to SubtleCrypto flow.

**New APIs**:
- `stream/iter` experimental (James Snell, #62066): New stream primitive.
- `AsyncLocalStorage` using-scopes (Stephen Belanger, #61674): Works with `using` declarations (explicit resource management).
- `repl` customizable error handling and domain removal.
- SEA (single executable app) code cache for ESM entrypoints.
- Crypto: TurboSHAKE, KangarooTwelve Web Crypto algorithms.

### Highlights (v2.7.13)

**Major Node compat work**:
- **`node:http` full rewrite** with llhttp parser and native TCPWrap. This is significant — previous implementation was JS-on-top-of-Deno-HTTP. Aligns behavior with Node's HTTP module much more closely.
- **`node:repl` implemented**: First-class REPL support in Deno's Node compat layer.
- **NAPI**: Real V8 handle scopes and callback scopes, zero-copy external Latin-1 strings, deferred GC weak-callback finalizers. Native addon support tightening.
- **`ChildProcess.kill()`** returns false when already exited (bug fix matching Node).
- Multiple HTTP/2 bug fixes (constants, error codes, settings validation).
- `node:tls` compat improvements (validation, pipes, JS streams).
- `spawnSync` FD-indexed stdio inheritance.

### Nx Compatibility Assessment

- **`node:http` rewrite** narrows the Deno-Node gap further. Nx CLI's use of `http` for Nx Cloud API calls would benefit if users ran Nx via Deno, though still not a primary runtime.
- **NAPI improvements**: Deno now handles more native addons. Still not tested with Nx's native module (`@nx/nx-darwin-*`).
- **Monorepo support**: Unchanged from last month. Deno workspaces differ from npm/yarn/pnpm.

### Risk Level for Nx: LOW (unchanged)

Deno parity climbs steadily. Not a primary runtime, but the gap is narrowing faster than Bun's on pure Node API compat.
