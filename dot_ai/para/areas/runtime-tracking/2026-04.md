### v24.15.0 LTS Release (2026-04-15) -- Major Features for Nx

This is the first minor release of Krypton since v24.14.0 (Feb 24) and lands several features that matter for Nx:

- **`require(esm)` marked stable** -- Previously experimental, now stable in LTS. Allows CommonJS `require()` of ESM modules without `--experimental-require-module`. **Direct relevance to Nx plugin loading**: plugin authors can distribute ESM-only code and Nx can require it via CJS without extra flags. Review `packages/nx/src/plugins/js/utils/register.ts` for simplification opportunities once minimum supported version is Node 24.
- **`module compile cache` marked stable** -- V8 code cache for CJS modules. Nx cold-start could benefit; cache is controlled by `NODE_COMPILE_CACHE` env or `module.enableCompileCache()`. Worth benchmarking against Nx daemon startup and `nx graph` generation.
- **`--max-heap-size` CLI flag backported to LTS** -- No longer Current-only. Replaces `NODE_OPTIONS=--max-old-space-size` with a simpler flag. Document this for users hitting OOM in large graphs.
- **`--require-module`/`--no-require-module` CLI flags** -- Control whether `require(esm)` is enabled per-process. Users may disable it if it conflicts with transpilation setups.
- **`fs.stat`/`fs.promises.stat` `throwIfNoEntry` option** -- Matches `fs.statSync` behavior; can simplify Nx filesystem helpers that catch-and-return-undefined on ENOENT.
- **`http2.http1Options`** -- HTTP/1 fallback config; relevant for Nx Cloud client if HTTP/2 support changes.
- **`test_runner` worker ID exposed** -- `process.env.NODE_TEST_CONTEXT` or `t.workerID`. Lets Nx test runners reason about parallel shards.
- **`Socket.setTOS`/`getTOS`** -- Low-level socket Type of Service control. Not directly relevant.
- **Root certificates updated to NSS 3.121**.

### Security Release (2026-03-24) -- 7 CVEs Across All Active Lines

Coordinated release across v25.8.2, v24.14.1, v22.22.2, v20.20.2:

- **CVE-2026-21637** (High): `SNICallback` crash in TLS -- unhandled exception crashes process
- **CVE-2026-21710** (High): Prototype pollution via `headersDistinct`/`trailersDistinct` -- fixed with null prototype
- **CVE-2026-21713** (Medium): Non-timing-safe comparison in Web Crypto HMAC/KMAC
- **CVE-2026-21714** (Medium): Unhandled `NGHTTP2_ERR_FLOW_CONTROL` in HTTP/2
- **CVE-2026-21717** (Medium): **HashDoS via array index hash collision** -- V8 string table could be forced to O(n^2) with 2 MB payload causing 30s hangs. Fixed with seeded xorshift-multiply hash.
- **CVE-2026-21715** (Low): Missing permission check on `realpath.native`
- **CVE-2026-21716** (Low): Missing permission check in `lib/fs/promises`

**Nx Impact**: CVE-2026-21717 (HashDoS) is the most concerning -- any Nx Cloud endpoint or daemon accepting JSON input could be targeted. CVE-2026-21710 (prototype pollution in HTTP headers) affects Nx Cloud agents. Upgrade CI images to 24.15.0 (includes all March fixes).

### Notable Features (v25.9.0 Current)

- **`stream/iter` module**: Experimental iterator-based stream API (`import { fromReadable } from 'node:stream/iter'`). Could eventually simplify streaming patterns in Nx.
- **TurboSHAKE and KangarooTwelve crypto**: New Web Crypto algorithms for high-speed hashing.
- **Test runner mock module API change**: `MockModuleOptions.defaultExport`/`namedExports` consolidated to `MockModuleOptions.exports`. Breaking change with migration codemod available (`npx codemod @nodejs/mock-module-exports`).

### HashDoS Deep Dive (CVE-2026-21717)

The March blog post by Joyee Cheung details the vulnerability and fix:
- Array index strings (numeric strings <= 24 bits) used deterministic hashing: `hash = (length << 24) | numeric_value`
- Attacker could craft ~2 MB JSON payload causing 30s parse hang via hash collision chain
- Fix: 3-round xorshift-multiply permutation with random seed at startup
- Zero performance impact (within measurement noise on SunSpider, Kraken, Octane, JetStream)
- V8 embedder flag: `v8_enable_seeded_array_index_hash = true`

### Highlights

**v1.3.13 (April 20) -- shipped TODAY. Test runner push is the story.**

- **`bun test --parallel`**: Run tests in parallel within a single process
- **`bun test --isolate`**: Process-level isolation between tests
- **`bun test --shard`**: Shard tests across workers (matches Jest/Vitest sharding semantics)
- **`bun test --changed`**: Only run tests affected by changed files (direct analog to `nx affected -t test`)
- **`bun install` 17x memory reduction**: Streams tarballs to disk instead of buffering
- **Source maps 8x less memory**
- **5.5x faster gzip via zlib-ng**: Also affects package install speed
- **SHA3 in `node:crypto` and WebCrypto**: Matches Node 25.9.0 TurboSHAKE direction
- **Range request support in `Bun.serve()`**
- **`ws+unix://` WebSocket client**: Unix socket WebSocket support
- **82 fixes**: Continuing Node compat push

**Nx positioning note**: Bun's `--changed`, `--shard`, and `--parallel` flags land in a single runtime without workspace configuration -- overlapping with Nx's affected-test and distributed-test value. Bun doesn't do cross-project graph analysis, so Nx's edge remains multi-project awareness, caching, and Nx Cloud distribution, but "distributed tests" as a value prop is getting a baseline-competitive built-in alternative at the runtime level.

**v1.3.12 (April 10)**:
- **`Bun.WebView`**: In-process headless browser automation. Competitive signal vs Playwright/Puppeteer, though not directly relevant to Nx core.
- **cgroup-aware parallelism on Linux**: Bun auto-detects container CPU limits for parallelism. **Relevant pattern for Nx's own parallel task scheduling** -- Nx currently uses `os.cpus()` which doesn't respect cgroup quotas.
- **`Bun.Glob.scan()` 2x faster**: Native glob matching improvements. Potential benchmark target for Nx's file matching.
- **URLPattern 2.3x faster**: Relevant if Nx Cloud endpoints use URLPattern-based routing.
- **Async stack traces**: Better error reporting for native errors.
- **120 bug fixes**: Mostly Node.js compat improvements.

**v1.3.11 (March 18)**:
- **`Bun.cron`**: OS-level cron scheduling (Bun-specific, not Node compat)
- **105 bug fixes** including `dgram` UDP on macOS, `fs.openSync` numeric flags on Windows (unblocks tar@7), HTTP/2 WINDOW_UPDATE fix
- **4 MB smaller on Linux**
- **CRLF injection fix**: Security improvement

**v1.3.10 (Feb 26)**:
- **TC39 Standard ES Decorators**: Full support. Important for Angular ecosystem compatibility.
- **`--compile --target=browser`**: Self-contained HTML from Bun apps.
- **Native REPL**: No longer depends on Node's REPL.
- **Windows ARM64**: Expanding platform support.

### Nx Compatibility Assessment

- **Lockfile parsing**: Still broken -- Bun outputs malformed Yarn lockfile format ([bun#16252](https://github.com/oven-sh/bun/issues/16252)). **No change.**
- **Node compat**: Steady improvement. 462 combined bug fixes across v1.3.10-v1.3.13.
- **ES Decorators**: Bun now supports standard decorators, aligning with Angular/TC39.
- **cgroup-aware parallelism**: Pattern worth evaluating for Nx's own parallel runner on Linux CI.
- **Test runner (new)**: Bun 1.3.13 test runner adds sharding, parallelism, isolation, and `--changed`. Evaluate for competitive positioning.

### Risk Level for Nx: MEDIUM (unchanged)

Lockfile format remains the primary technical blocker. Competitive pressure increased with 1.3.13 test runner improvements.
