## Headline: Node 26 Released (May 5) - Next LTS in October 2026

**Node 26.0.0 (Current) shipped May 5, 2026.** Key breaking changes relevant to Nx:
- **`--experimental-transform-types` REMOVED** - projects using this flag to strip TypeScript types must migrate to `--strip-types` (stable since Node 24)
- **Extensionless CJS exception removed for `type:module` packages** - any Nx package or plugin with `"type": "module"` in package.json that uses extensionless `require()` will break on Node 26
- **`module.register()` runtime-deprecated** (DEP0205 formalized) - the loader hook API Nx plugins may use is now formally deprecated; migration path is `module.registerHooks()` (Node 24.3+)
- **Legacy private stream modules removed** (`_stream_wrap`, `_stream_readable`, etc.) - transitive dependencies using these will break
- **Temporal API now default** (no flag needed) - V8 14.6 (Chromium 146)
- **LTS transition: October 2026** (5 months out) - start testing now

Node 26.2.0 followed on May 20; Node 24.16.0 LTS on May 21 (QUIC re-enabled, `randomUUIDv7()`, OpenSSL 3.5.6).

Node 25 EOL is **June 1, 2026** - 7 days from this scan.

### Node 26 - Risks for Nx (NEW)

1. **`--experimental-transform-types` removed** - This flag was the pre-stable form of `--strip-types`. Any Nx CI script, docs, or generator template referencing it breaks on Node 26. `--strip-types` (stable since 24) is the replacement.
2. **Extensionless CJS in `type:module` packages removed** - Affects Nx packages that declare `"type":"module"` but rely on extensionless imports. The exception that allowed CJS files without extension in those packages is gone.
3. **`module.register()` runtime-deprecated (DEP0205 formal)** - This API was introduced as a replacement for the older loader hook API. Now it's itself deprecated in favor of `module.registerHooks()` (available since 24.3). Nx loader hooks / plugin loading may be affected.
4. **Legacy private stream modules removed** (`_stream_readable`, `_stream_writable`, `_stream_wrap`, etc.) - These were internal modules exposed via `require()`. Any transitive dependency referencing them breaks on Node 26. Check `npm ls` for deep deps using these.
5. **`http.Server.prototype.writeHeader()` fully removed** - Long-deprecated alias for `writeHead()`. Unlikely Nx uses it directly, but check plugins.
6. **V8 14.6 / Temporal API default** - Temporal landed without a flag. This is an opportunity (new datetime handling) but could affect any code that feature-detects `Temporal` existence.

### Node 26 - Opportunities for Nx

- **Temporal API default** - No longer needs a flag. Generator templates using `Date` could optionally use Temporal.
- **`randomUUIDv7()`** (backported to 24.16.0) - Timestamp-ordered UUIDs useful for cache keys or task IDs.
- **V8 `Map.getOrInsert()` / `getOrInsertComputed()`** - Replaces common Nx cache-map patterns.
- **`Iterator.concat()`** - Could simplify some generator/plugin code that merges iterables.

### V8 Engine

Node 26.0.0 ships V8 14.6 (Chromium 146). Key new JS features landing in V8 14.6: `Map.prototype.getOrInsert/getOrInsertComputed`, `Iterator.concat()`, Temporal API (default). Deno 2.7.x uses V8 147.x (Deno's internal versioning, not directly comparable to upstream V8 major).

### Highlights (v2.8.0)

**TypeScript 6.0.3** - Deno ships TS 6.0 before the broader npm ecosystem. This is a competitive signal: Deno users get TS 6 features (unknown at time of writing which major features TS 6 brings) while Node/npm-based projects wait for bundler/toolchain support. Nx may need a TS 6 migration guide ahead of when Node ecosystem catches up.

**Workspace catalog protocol** - Deno 2.8 adds a `catalog:` protocol for workspace dependencies, mirroring pnpm catalogs. This is Deno closing a real monorepo gap. Competitive with Nx's workspace dependency management story.

**Node.js compat additions:**
- `module.registerHooks()` - Deno implements the same API that replaces the deprecated `module.register()` (DEP0205). Deno tracks Node 26 deprecation path.
- `node:wasi` - WASI support lands. Niche but relevant for Nx plugin authors targeting WASM.
- Hoisted node_modules linker - better npm compat for packages that expect flat node_modules. Reduces Nx-on-Deno friction.

**TC39 import defer** - Deno 2.8 ships Stage 3 import defer proposal support. Ahead of V8/Node.

**Text imports stable** - `import text from "./file.txt"` (TC39 Import Text) stabilized in Deno 2.8. V8/Node still pending.

**New tooling subcommands** - `deno why` (why is this dep installed), `deno ci` (CI mode), `deno pack`, `deno transpile`, `deno bump-version`. Deno is building a full monorepo toolchain on top of its runtime. The `deno why` command mirrors what users do via `pnpm why` or `npm why` today.

### Nx Compatibility Assessment

- **Node compat**: Fastest gap-closing pace. `module.registerHooks()` + `node:wasi` + hoisted linker all reduce friction.
- **Monorepo story**: Workspace catalog + `deno why` narrows the gap vs Nx workspace features. Not a replacement but worth tracking.
- **NAPI**: Native addon support still unverified for `@nx/nx-darwin-*` natives.
- **TypeScript 6**: Deno is 1+ version ahead of Node ecosystem on TS. Users wanting cutting-edge TS have a reason to prefer Deno.

### Risk Level for Nx: LOW -> LOW-MEDIUM

Deno 2.8 is their self-described "biggest minor release to date." Workspace catalog + toolchain commands signals a deliberate monorepo play. Still not a primary threat to Nx's customer base but worth a quarterly reassessment.
