### Security Release (2026-03-24) -- ALL ACTIVE LINES

Coordinated security release across v25, v24, v22, and v20:

- **CVE-2026-21637** (High): `SNICallback` crash in TLS -- wrap in try/catch
- **CVE-2026-21710** (High): Prototype pollution via `headersDistinct`/`trailersDistinct` -- use null prototype
- **CVE-2026-21711** (Medium): Missing permission check in `pipe_wrap.cc`
- **CVE-2026-21712** (Medium): URL parser crash on certain formats
- **CVE-2026-21713** (Medium): Non-timing-safe comparison in Web Crypto HMAC/KMAC
- **CVE-2026-21714** (Medium): Unhandled `NGHTTP2_ERR_FLOW_CONTROL` in HTTP/2
- **CVE-2026-21717** (Medium): Array index hash collision
- **CVE-2026-21715** (Low): Missing permission check on `realpath.native`
- **CVE-2026-21716** (Low): Missing permission check in `lib/fs/promises`

**Nx Impact**: CVE-2026-21710 (prototype pollution in HTTP headers) could affect Nx Cloud agents. CVE-2026-21714 (HTTP/2 flow control) relevant for Nx Cloud connections. Recommend updating CI images promptly.

### Notable Features (v24.14.0 LTS)

- **`fs.watch` `ignore` option**: Native glob-based ignore for file watching. Could reduce Nx's userland ignore logic.
- **`http.setGlobalProxyFromEnv()`**: Auto-configures HTTP proxy from env vars. Simplifies proxy handling for Nx Cloud.
- **Subpath imports with `#/`**: Allows `#/utils/foo` style imports. Nx module resolution may need to handle this.
- **`cjs-module-lexer` replaced with `merve`**: Internal change to CJS/ESM interop -- watch for edge cases.

- **`async_hooks.createHook()` `trackPromises`**: New opt-in for promise tracking.
- **SQLite defensive mode by default**: Breaking for code relying on permissive SQLite.

### Notable Features (v25.8.0 Current)

- **SQLite `limits` property**: Configure database limits.
- **C++ diagnostics channels**: Internal infrastructure improvement.

## Bun

### Release Summary (since 2026-01-29)

| Version | Date | Key Changes |
|---------|------|-------------|
| **v1.3.11** | 2026-03-18 | `Bun.cron`, `Bun.sliceAnsi`, `--path-ignore-patterns` for tests, 105 bug fixes |
| **v1.3.10** | 2026-02-26 | Native REPL, `--compile --target=browser`, TC39 ES Decorators, Windows ARM64, 155 bug fixes |
| v1.3.9 | 2026-02-08 | Bug fixes |
| v1.3.8 | 2026-01-29 | Bug fixes |

### Highlights

**v1.3.10 -- Major Feature Release**:
- **TC39 Standard ES Decorators**: Full support for standardized decorators. Aligns with the Stage 3 proposal.
- **`--compile --target=browser`**: Creates self-contained HTML files. Novel deployment model.
- **Native REPL**: Built-in interactive shell (previously relied on Node's REPL).
- **Windows ARM64**: Expanding platform coverage.
- **Performance**: `structuredClone` up to 25x faster for arrays, `path.parse()` up to 7x faster.

**v1.3.11 -- Stability + Utilities**:
- **`Bun.cron`**: OS-level cron job support (Bun-specific API, not Node compat).
- **`Bun.sliceAnsi`**: ANSI-aware string slicing utility.
- **`fs.openSync` numeric flags fixed on Windows**: Unblocks `tar@7` -- important for package installation.

### Nx Compatibility Assessment

- **Lockfile parsing**: Known issue remains -- Bun outputs malformed Yarn lockfile format ([bun#16252](https://github.com/oven-sh/bun/issues/16252)). Nx lockfile parsing affected.
- **Workspace support**: Bun workspaces work but differ from npm/yarn/pnpm. Nx `detect-package-manager` handles Bun but edge cases persist.
- **Node.js compat**: Steadily improving. 105-155 bug fixes per release. `dgram`, `fs`, `HTTP/2` fixes in recent releases.
- **MCP compatibility**: v1.3.10 fixed Python asyncio MCP server compat with `Bun.spawn()`.

### Risk Level for Nx: MEDIUM

Bun adoption in Nx workspaces is growing but lockfile format issues remain the primary blocker for reliable Nx graph computation.

Total: 14 releases in the lookback window (rapid iteration).

### Deno 2.7 Highlights

**npm/Node Compatibility**:
- **`npm overrides` in package.json**: Deno now respects the `overrides` field. Critical for monorepo dependency management.
- **`jsr:` scheme in package.json**: Can use JSR packages without `deno.json`.
- **Node compat fixes**: `worker_threads`, `child_process`, `zlib`, `sqlite` improvements. `AsyncLocalStorage` preserved in `unhandledRejection` handlers.
- **`deno install --compile`**: Create standalone executables from npm packages.

**New Features**:
- **`deno create`**: Project scaffolding command (like `npm create`/`npx create-*`).
- **Temporal API stabilized**: First runtime to stabilize TC39 Temporal.
- **`deno compile --self-extracting`**: Enables native addons in compiled binaries.
- **`deno task` improvements**: `pipefail` and `shopt` configuration.

### Nx Compatibility Assessment

- **Monorepo support**: Deno workspaces exist but differ significantly from npm/yarn/pnpm. `npm overrides` support is a step toward compatibility but Deno is not a viable Nx package manager today.
- **Node compat**: Improving rapidly (~dozens of fixes per minor). `child_process` and `worker_threads` fixes help but gaps remain.
- **Primary concern**: Deno users running Nx via `npx nx` or `deno run npm:nx` -- these paths are fragile.

### Risk Level for Nx: LOW

Deno is not a primary Nx runtime. Monitor for users attempting Deno + Nx combinations, but no action needed.

## Nx `engines` Field Review

**Current state**: Nx `package.json` does not declare an `engines` field (verified via `npm view nx`).

### Recommended `engines` Policy

| Node Version | Status | Recommendation |
|-------------|--------|----------------|
| Node 18 | EOL (2025-04-30) | Already unsupported. No action. |
| Node 20 | EOL **2026-04-30** | Drop in Nx 23. Add deprecation warning in Nx 22.7+. |
| Node 22 | Maintenance LTS until 2027-04-30 | Primary supported version |
| Node 24 | Active LTS until 2026-10-20 | Fully supported |
| Node 25 | Current (EOL 2026-06-01) | Best-effort support |

**Suggestion**: For Nx 23 (next major), set `"engines": { "node": ">=22.0.0" }`. This aligns with keeping two LTS lines (22 + 24) supported. Node 20 EOL is imminent (2026-04-30).

If Nx 22 wants to add an `engines` field now: `"engines": { "node": ">=20.0.0" }` with a runtime deprecation warning for Node 20.
