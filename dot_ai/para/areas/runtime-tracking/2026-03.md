## Action Items for Nx

### Urgent (next 6 weeks)

1. **Node 20 EOL plan**: Node 20 reaches EOL April 30, 2026 (41 days). Nx 22.6.0 has no `engines` field — decide whether to set `>=22` and formally drop Node 20, or add deprecation warnings first.
2. **Mar 24 security patches (4 days)**: Apply Node security updates across CI when released (2 high, 5 medium, 2 low across v20–v25). All active lines affected.
3. **Bun lockfile compat**: Still the #1 Bun friction point. Track [nrwl/nx#30607](https://github.com/nrwl/nx/issues/30607).
4. **Nx engines field**: Shipping without `engines` is a risk. Consider adding `>=20.11.0` now, then bump to `>=22` after Node 20 EOL.

### Medium-term

5. **New release schedule impact**: Node 27 (Apr 2027) will be first under new model. Every major becomes LTS. Nx should plan `engines` bumps around April releases going forward. Alpha channel useful for early compat testing.
6. **Node 26 prep**: Node 26 releases April 2026 (old model). Will become LTS Oct 2026, EOL Apr 2029. Ensure Nx CI includes Node 26 on release day.
7. **`import defer` tracking**: V8 implementation progressing. Once Node.js picks it up, Nx CLI startup could benefit from deferred module evaluation. Monitor TypeScript PR [#60757](https://github.com/microsoft/TypeScript/pull/60757) and Rollup [#5932](https://github.com/rollup/rollup/issues/5932).
8. **ES2026 `using` keyword**: Nx codebase could adopt explicit resource management for file handles, database connections. Requires TypeScript 5.2+ and target ES2022+.
9. **Test against Node 25**: Ensure CI includes Node 25.x. Node 25.8.1 fixed CJS extensionless file handling in `"type": "module"` packages — relevant for Nx's CJS/ESM interop.

### Low priority / monitoring

10. **Deno compat**: Deno 2.7.x has significant node compat improvements. 2.7.7 adds dns compat, AES validation, constant-time GCM comparison, and worker_threads stubs. Not yet a priority runtime but gap narrowing fast.
11. **V8 compile hints**: Could improve Nx daemon/worker startup performance in future.
12. **Temporal API**: Stable in Deno 2.7, unflagged in Chromium 144+. No immediate Nx action needed.

### Release Schedule Overhaul (Mar 10 announcement)

Starting with **Node 27 (April 2027)**, the project moves to **one major release per year**:

| Phase       | Duration   | Description                                   |
| ----------- | ---------- | --------------------------------------------- |
| **Alpha**   | 6 months   | Oct–Mar: semver-major changes, signed builds  |
| **Current** | 6 months   | Apr–Oct: stabilization                        |
| **LTS**     | 30 months  | Long-term support                             |

Key changes:
- **Every release becomes LTS** (no more odd/even distinction)
- **Alpha channel** replaces odd-numbered Current releases — signed, CITGM-gated, not for production
- **Node 26** (Apr 2026) is the **last release under the old model**
- **Node 27** alpha begins Oct 2026, releases Apr 2027
- Calendar-aligned versioning: Node 28 = Apr 2028, Node 29 = Apr 2029, etc.

**Nx implications**: Simplifies `engines` planning — every April, a new major to support. Alpha channel is ideal for early Nx compat CI.

### Bun 1.3.11 Notable Details (Mar 18)

- **Native Windows ARM64 `.bin` shim**: Better Windows ARM64 support.
- **Key bug fixes**: dgram UDP fixed on macOS, `fs.openSync` numeric flags fixed on Windows (unbreaks `tar@7`), HTTP/2 `WINDOW_UPDATE` fix, Proxy-array crash fixes, CRLF injection fix, `bun install` hang fixes.

### Bun 1.3.10 Notable Details

- **TC39 standard decorators**: Now supports the finalized ES decorators proposal (in addition to legacy/experimental decorators).

- **Windows ARM64 support**: Native builds for ARM64 Windows.
- **155 bug fixes** addressing 642 upvoted issues.

### Nx Compatibility Notes

  - [nrwl/nx#26565](https://github.com/nrwl/nx/issues/26565): Failed to parse bun lockfile

  - [nrwl/nx#32853](https://github.com/nrwl/nx/issues/32853): @nx/rspack bun lockfile generation
- **Malformed Yarn-format output** from Bun when generating yarn.lock is a known upstream bug ([oven-sh/bun#16252](https://github.com/oven-sh/bun/issues/16252)).

### Deno 2.7.7 (Mar 19 — new since last scan)

- **macOS `select` fallback**: Implements fallback for systems where `select` syscall has issues
- **node:dns compatibility**: Improved DNS module compat with Node.js
- **AES key/IV length validation**: Added proper validation for `aes-128-cbc` and `aes-128-ecb` cipher modes
- **Constant-time GCM auth tag comparison**: Security fix — uses constant-time comparison for GCM authentication tag verification to prevent timing attacks
- **process.title improvements**: Better `process.title` support including new `--title` flag
- **worker_threads stubs**: Added disabled process function stubs in worker threads for better Node.js compat
- **Windows path fixes**: Skip normalization in `path.win32.join` for reserved device names
- **IPv6 multicast**: Support `interface` option for IPv6 multicast membership in dgram
- **WebIDL optimization**: Performance improvement for dictionary converters
- **Graceful shutdown**: `deno watch` now supports graceful shutdown via SIGTERM dispatch

### Deno 2.7.6 (Mar 18)

- **V8 updated to 146.8.0** with foreground task ownership
- **`--cpu-prof-flamegraph`**: Generate interactive SVG flamegraphs directly
- **Auto-detect CJS vs ESM in `deno eval`**: No more manual format specification
- **`MIMEType` support**: Added to `ext/node` along with `convertProcessSignalToExitCode` and `Symbol.dispose` for ChildProcess
- **Additional Windows signals**: More signal support on Windows
- **OTEL improvements**: Array values in OTEL attributes, fixed server span error marking for 4xx
- **TypeScript-Go updated** to 2026-03-09
- **Massive Node.js compat fixes** (~30+ fixes):
  - child_process: IPC message prototype, send validation, signal handling, AbortController, spawn stdio setup
  - crypto: `generatePrime` safe/add/rem options, encrypted PEM export, `generateKeyPair` validation, X509Certificate improvements
  - TTY: Rewritten on top of uv compat layer
  - TLS: JSStreamSocket resource leak fix preventing process hang
  - Buffer.concat fix, `markAsUntransferable` implementation
  - Multiple `deepEqual` comparison fixes
  - worker_threads path validation improvements
- **NAPI fixes**: Finalizer handling at shutdown, event loop kept alive for `uv_async_t`, `napi_coerce_to_object` null/undefined fix

### Deno 2.7.4–2.7.5

- **2.7.5** (Mar 11):
  - `KeyObject.toCryptoKey()` and `KeyObject.from(CryptoKey)` — bidirectional interop between Node.js-style and Web Crypto keys
  - Function coverage added to summary and HTML reports
  - `deno doc` now supports npm packages
  - `--cpu-prof` flags for CPU profiling
  - OTEL permission audit data integration
  - **V8 microtask policy fix**: Switched to V8 Explicit microtask policy for correct `nextTick` ordering — important for Node.js compat
  - NAPI: `node_api_create_object_with_properties` implemented
  - Fix: async NAPI work now runs execute callback on worker thread (was blocking main thread)

- **2.7.4** (Mar 5):
  - Deep `assert` compatibility with Node.js
  - Sync TTY writes for stdout/stderr (matches Node.js behavior)
  - npm `install -g deno` binary startup performance improvement
  - uv_compat cleanup and fixes

### Nx Relevance

- Deno is not a first-class Nx runtime, but the compatibility gap is narrowing rapidly — V8 146.8.0, TTY rewrite on uv compat, 30+ Node.js compat fixes in 2.7.6 alone, KeyObject↔CryptoKey interop, corrected microtask ordering, and constant-time GCM auth tag verification in 2.7.7 show serious Node.js parity effort.
- npm `overrides` support in Deno 2.7 helps users with complex dependency trees.
- No official Nx plugin for Deno runtime exists; community solutions remain the path.
