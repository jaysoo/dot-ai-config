# Runtime Tracking — March 2026

_Scan date: 2026-03-04_
_Period: January 3 – March 4, 2026_

## Summary

- **Nx 23.0.0 released** (Feb 26): New major version — no `engines` field set. Node 20 EOL is April 30, 2026; Nx 23 should formalize minimum Node version.
- **Node.js 25.8.0** (Mar 3): `--permission-audit` flag, SQLite limits, C++ diagnostics channels, test runner worker IDs. Node 24.14.0 is latest LTS.
- **Bun 1.3.10** (Feb 26): TC39 standard ES decorators, 25x faster `structuredClone`, native REPL, `--compile --target=browser`, barrel import optimization, Windows ARM64. Fixes 155 issues.
- **Deno 2.7.2–2.7.3** (Mar 3–4): V8 14.6, massive node:crypto/node:fs compat fixes, UDP dgram methods, HTTP over Windows named pipes.
- **Node 20 EOL**: 57 days away (April 30, 2026). Planning required.

---

## Action Items for Nx

### Urgent (next 2 months)

1. **Node 20 EOL plan**: Node 20 reaches EOL April 30, 2026. Nx 23 has no `engines` field — decide whether to set `>=22` and formally drop Node 20, or add deprecation warnings first.
2. **Bun lockfile compat**: Multiple open issues around `bun.lock` text format parsing. #1 Bun friction point for Nx users. Track [nrwl/nx#30607](https://github.com/nrwl/nx/issues/30607).
3. **Nx 23 engines field**: Nx 23.0.0 shipped without an `engines` field. Consider adding one to signal supported Node versions.

### Medium-term

4. **`import defer` tracking**: V8 implementation progressing. Once Node.js picks it up, Nx CLI startup could benefit from deferred module evaluation. Monitor TypeScript PR [#60757](https://github.com/microsoft/TypeScript/pull/60757) and Rollup [#5932](https://github.com/rollup/rollup/issues/5932).
5. **ES2026 `using` keyword**: Nx codebase could adopt explicit resource management for file handles, database connections. Requires TypeScript 5.2+ and target ES2022+.
6. **Test against Node 25**: Ensure CI includes Node 25.x. Node 26 will become LTS in Oct 2026.
7. **Bun ES decorators**: Bun 1.3.10 supports TC39 standard decorators. If Nx plugins emit decorator syntax, verify compat.

### Low priority / monitoring

8. **Deno compat**: Deno 2.7.x has significant node compat improvements (crypto, fs, worker_threads, dgram). Gap narrowing but not yet a priority runtime.
9. **V8 compile hints**: Could improve Nx daemon/worker startup performance in future.
10. **Temporal API**: Stable in Deno 2.7, unflagged in Chromium 144+. No immediate Nx action needed.

### Bun 1.3.10 Notable Details

- **TC39 standard decorators**: Now supports the finalized ES decorators proposal (in addition to legacy/experimental decorators).
- **Performance**: `structuredClone` up to 25x faster for arrays, `Buffer.slice()` 1.8x faster, `path.parse()` up to 7x faster, 168x faster deep rope string slicing via upgraded JavaScriptCore.
- **`--compile --target=browser`**: Self-contained HTML output from bundled code.
- **Barrel import optimization**: Faster resolution of barrel files (index.ts re-exports).
- **Windows ARM64 support**: Native builds for ARM64 Windows.
- **155 bug fixes** addressing 642 upvoted issues.

### Nx Compatibility Notes

- **Lockfile parsing remains the primary pain point.** Bun 1.2+ defaults to text-based `bun.lock` (JSON-like), but Nx has had issues parsing both formats. Relevant issues:
  - [nrwl/nx#30607](https://github.com/nrwl/nx/issues/30607): Text-based lockfile required for Bun >1.2.0
  - [nrwl/nx#26565](https://github.com/nrwl/nx/issues/26565): Failed to parse bun lockfile
  - [nrwl/nx#29494](https://github.com/nrwl/nx/issues/29494): New text-based lock file not supported
  - [nrwl/nx#32853](https://github.com/nrwl/nx/issues/32853): @nx/rspack bun lockfile generation
- **Malformed Yarn-format output** from Bun when generating yarn.lock is a known upstream bug ([oven-sh/bun#16252](https://github.com/oven-sh/bun/issues/16252)).
- **Breaking changes tracker**: [oven-sh/bun#20292](https://github.com/oven-sh/bun/issues/20292)

### Deno 2.7.x Node.js Compatibility (significant progress)

- **V8 14.6** in 2.7.2 (ahead of Node 25's V8 14.1)
- **node:crypto**: P-521 EC key export, DER-encoded keys, RSA PSS padding, Ed25519/X25519/P-521 in X509Certificate, CBC decipher `setAutoPadding`, GCM cipher padding, `KeyObject.equals`, `createCipheriv` with KeyObject/null IV
- **node:fs**: `recursive mkdir` returns first created path, `fsPromises.watch()` as proper AsyncIterable
- **node:worker_threads**: `threadName`, `worker.cpuUsage()`, BroadcastChannel ref/unref, CJS globals in eval mode
- **node:dgram**: Missing UDP methods implemented, synchronous socket ops
- **node:http**: HTTP over Windows named pipes
- **process.stdout/stderr**: Partial write handling fixed

### Nx Relevance

- Deno is not a first-class Nx runtime, but the compatibility gap is narrowing rapidly — especially with V8 14.6 and the node:crypto/fs/worker_threads fixes.
- npm `overrides` support in Deno 2.7 helps users with complex dependency trees.
- No official Nx plugin for Deno runtime exists; community solutions remain the path.
