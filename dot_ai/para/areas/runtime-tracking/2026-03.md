# Runtime Tracking — March 2026

_Scan date: 2026-03-02_
_Period: February 1 – March 2, 2026_

## Summary

- **Node.js 20 EOL approaching**: April 30, 2026 — Nx should plan to drop Node 20 support in the next major version. Node 24 (Krypton) is the primary LTS; Node 25.7.0 is the current line.
- **ES2026 finalized** (Dec 2025): `using`/`await using` (explicit resource management), Temporal API, `Iterator` helpers, `Set` methods, `Promise.try`, and `import defer` (Stage 3) all landed. Tooling should start leveraging these.
- **Bun 1.3.9** (Feb 26): Lockfile compat with Nx remains a known pain point (`bun.lock` text format vs `bun.lockb` binary). Bun's Yarn-format lockfile output can still be malformed (see [oven-sh/bun#16252](https://github.com/oven-sh/bun/issues/16252)).
- **Deno 2.7** (Feb 25): Temporal API stabilized, V8 14.5, improved `node:worker_threads`/`node:child_process` compat, npm `overrides` support. Growing viability as Nx runtime but still not first-class.
- **Chrome 145 / V8**: Chrome 145 stable shipped Feb 10 with Map/WeakMap upsert operations. V8 14.5 ships in Deno 2.7; Node 24 LTS bundles V8 13.6; Node 25 bundles V8 14.1.

## Bun

### Latest: v1.3.9 (February 26, 2026)

**Key changes in Feb releases:**

- **1.3.8** (Feb 8): Built-in `Bun.markdown` (CommonMark-compliant Markdown parser in Zig)
- **1.3.9** (Feb 26): Parallel/sequential script running (`--parallel`/`--sequential`), `bun:test` mock/spyOn get `Symbol.dispose` support, ESM bytecode compilation, faster `Bun.markdown`, RegExp/String perf improvements

**Canary:** 1.3.10-canary.20260227.1

### Nx Compatibility Notes

- **Lockfile parsing remains the primary pain point.** Bun 1.2+ defaults to text-based `bun.lock` (JSON-like), but Nx has had issues parsing both formats. Relevant issues:
  - [nrwl/nx#30607](https://github.com/nrwl/nx/issues/30607): Text-based lockfile required for Bun >1.2.0
  - [nrwl/nx#26565](https://github.com/nrwl/nx/issues/26565): Failed to parse bun lockfile
  - [nrwl/nx#29494](https://github.com/nrwl/nx/issues/29494): New text-based lock file not supported
  - [nrwl/nx#32853](https://github.com/nrwl/nx/issues/32853): @nx/rspack bun lockfile generation
- **Malformed Yarn-format output** from Bun when generating yarn.lock is a known upstream bug ([oven-sh/bun#16252](https://github.com/oven-sh/bun/issues/16252)) — affects Nx lockfile parsing.
- **Breaking changes tracker**: [oven-sh/bun#20292](https://github.com/oven-sh/bun/issues/20292)

- Zero-config frontend dev
- Unified SQL API, built-in Redis client
- Package catalogs
- Async stack traces
- VS Code test integration
- Improved Node.js compatibility

## Deno

### Latest: v2.7.0 (February 25, 2026)

**Key features:**

- **Temporal API stabilized**: `--unstable-temporal` flag removed. Temporal is now stable.
- **V8 14.5**: Latest V8 brings performance improvements and newest JS features.
- **Windows ARM support**: Native builds for aarch64-pc-windows-msvc (Surface Pro X, Snapdragon laptops).
- **`deno create` command**: Familiar alias for scaffolding (works with npm and JSR packages).
- **Compiled npm executables**: `deno install -g` can compile npm packages into standalone executables.
- **`--save-exact` flag**: Pin dependencies to exact versions.
- **npm `overrides` support**: `package.json` overrides field now respected.
- **SHA3 in Web Crypto API**, Brotli in CompressionStream/DecompressionStream.

### Node.js Compatibility Improvements

Dozens of fixes in 2.7:
- `node:worker_threads`: Added `threadName`, `worker.cpuUsage()`, BroadcastChannel ref/unref, error event fixes, stdout forwarding to parent
- `node:child_process`: Various fixes
- `node:zlib`: Compatibility improvements
- `node:sqlite`: Fixes
- First-class `package.json` support including `overrides` field

### Nx Relevance

- Deno is not yet a first-class Nx runtime, but compatibility gap is narrowing significantly.
- The npm overrides support and improved worker_threads compat make Deno more viable for running Nx-based projects.
- No official Nx plugin for Deno runtime exists; community solutions remain the path.

## TC39 Proposals

### ES2026 Finalized (December 2025)

The ECMAScript 2026 specification was published, including:

| Feature | Impact on Tooling |
|---------|-------------------|
| **`using` / `await using`** (Explicit Resource Management) | Bundlers/linters need support. ESLint tracking: [eslint/eslint#19792](https://github.com/eslint/eslint/issues/19792) |
| **Temporal API** | Date/time handling replacement. Unflagged in V8/Chromium 144+, stable in Deno 2.7 |
| **Iterator helpers** (`Iterator` global) | `.map()`, `.filter()`, `.take()` etc. on iterators. Useful in build tool internals |
| **Set methods** (union, intersection, difference, etc.) | Simplifies set operations in dependency resolution code |
| **`Promise.try`** | Wraps sync/async into Promise chain cleanly |
| **`RegExp.escape`** | Safe regex construction from user strings |
| **Float16Array + DataView helpers** | Niche, unlikely to affect Nx directly |
| **`Error.isError()`** | Reliable error checking across realms — useful for worker boundaries |
| **`Array.fromAsync()`** | Async iterable to array conversion |

### Stage 3 (Implementation Phase)

| Proposal | Status | Relevance |
|----------|--------|-----------|
| **`import defer`** | Stage 3 (since Feb 2025) | **High** — deferred module evaluation could significantly improve CLI startup time. Rollup tracking: [rollup/rollup#5932](https://github.com/rollup/rollup/issues/5932). TypeScript support PR: [microsoft/TypeScript#60757](https://github.com/microsoft/TypeScript/pull/60757). Modules with top-level await cannot be deferred. |
| **Immutable ArrayBuffer** | Stage 2.7 | Low — read-only ArrayBuffer variant |
| **Math.clamp** | Stage 1 | Low — convenience utility |

### February 2026 TC39 Meeting Outcomes

The 106th TC39 meeting advanced:
- **3 proposals to Stage 4**: RegExp Escaping, Float16Array, Redeclarable global eval vars (these are part of ES2026)
- **`import defer`** continued at Stage 3 with implementation progress in V8 and WebKit
- **9 proposals** advanced to various stages total

## Action Items for Nx

### Urgent (next 2 months)

1. **Node 20 EOL plan**: Node 20 reaches EOL April 30, 2026. Determine if Nx 22.x drops support or if this waits for Nx 23. At minimum, add deprecation warnings.
2. **Bun lockfile compat**: Multiple open issues around `bun.lock` text format parsing. This is the #1 Bun friction point. Track [nrwl/nx#30607](https://github.com/nrwl/nx/issues/30607).

### Medium-term

3. **`import defer` tracking**: Once V8 ships this and Node.js picks it up, Nx CLI startup could benefit significantly from deferred module evaluation. Monitor TypeScript PR [#60757](https://github.com/microsoft/TypeScript/pull/60757) and Rollup [#5932](https://github.com/rollup/rollup/issues/5932).
4. **ES2026 `using` keyword**: Nx codebase could adopt explicit resource management for file handles, database connections, etc. Requires TypeScript 5.2+ and target ES2022+. ESLint support is being tracked.
5. **Test against Node 25**: Ensure CI includes Node 25.x for forward compat testing before it potentially becomes LTS candidate (odd-numbered, so it won't become LTS, but Node 26 will in Oct 2026).

### Low priority / monitoring

6. **Deno compat**: Deno 2.7's node compat improvements are significant but Deno is not yet a priority runtime for Nx. Monitor community demand.
7. **V8 compile hints**: Could be explored for Nx daemon/worker performance in future.
8. **Temporal API**: No immediate action needed for Nx CLI, but documentation sites and user-facing tooling may want to reference it.
