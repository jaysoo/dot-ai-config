### Risks for Nx

- **Security release (HIGH)**: Any Nx CI matrix pinned to a pre-June-18 Node version is running against CVE-unfixed runtimes. The TLS hostname bypass (CVE-2026-48618) is particularly relevant to Nx Cloud connectivity.
- **`Stats.Date` now enumerable (v24.18.0)**: `fs.stat()` results now expose `atime`, `mtime`, `ctime`, `birthtime` as enumerable. Any Nx serialization code that spreads or JSON-stringifies `Stats` objects may produce unexpected extra fields.
- **`--permission` stable (v24.18.0)**: Nx task sandboxing docs that reference `--experimental-permission` need updating. The `--experimental-` prefix is removed.
- **Addon imports on by default (v26.5.0)**: Any Nx code that enabled native addons via a flag now gets that behavior unconditionally on Node 26.5+. Review conditional flag paths.
- **`validatePort` string `"0"` (v26.5.0)**: Subtle behavior change. Nx CLI port parsing should be verified against this.
- **Node 26 `node:vfs` (experimental)**: A virtual file system API is landing in Node 26. Relevant to Nx's in-memory project graph and daemon file watching -- track for potential native replacement of current VFS layer.
- **Node 26 Package Maps**: The loader now implements package maps (import maps for CJS/ESM resolution). Could simplify Nx plugin resolution or conflict with existing alias handling.
- **Node 20 at 81 days post-EOL**: No upstream CVE coverage. The two HIGH CVEs from June 18 are unpatched on any Node 20 installation.

### Opportunities

- **`--experimental-import-text` flag (v26.5.0)**: Aligns with TC39 Import Text (Stage 3). Nx generator template loading could eventually use native text imports rather than `fs.readFileSync`. Track for stabilization.
- **`blob.textStream()` (v26.5.0)**: Streaming text access on Blob objects. Relevant to large workspace file streaming in Nx daemon.
- **Perf hooks per-event-loop sampling (v26.5.0)**: Better granularity for profiling Nx task scheduling latency.
- **`readFile()` caller-supplied buffers (v26.4.0)**: Memory-efficient file reading for large workspaces. Relevant to Nx project graph serialization.
- **`Buffer.poolSize` 64K default (v24.18.0)**: Improved throughput for small buffer allocations common in CLI I/O.
- **Node 26 LTS in October 2026**: The Current line becomes LTS this October. Start planning Nx CI matrix update: add v26 as primary, v24 as secondary, v22 as minimum.

## Bun

### Major news: Zig -> Rust rewrite (July 2026)

**The 68-day release gap is explained.** Bun was entirely rewritten from Zig to Rust between approximately early May and May 14, 2026 (when the rewrite was merged). The blog post ["Rewriting Bun in Rust"](https://bun.com/blog/bun-in-rust) was published July 8, 2026 by Jarred Sumner.

**Key facts:**
- Rewrite used ~50 parallel Claude Code workflows, peaking at ~1,300 lines of code per minute
- ~960,000-1,000,000+ lines of Rust total; took ~11 days; cost ~$165,000 in API costs
- Binary size: 88 MB -> 70 MB (Linux), ~20% smaller
- Claude Code v2.1.181 (shipped June 17) was one of the first major deployments using Rust-based Bun internally
- v1.4.0 is the first stable Rust-based release -- available as canary (`bun upgrade --canary`) but NOT yet stable as of July 20, 2026
- Criticism: Zig language creator publicly called it "unreviewed slop" (The Register, July 14)

### Releases since 2026-06-01

No stable releases. v1.3.14 (May 13) is the most recent stable -- now **68 days** without a stable release.

| Version | Date | Key Changes |
|---------|------|-------------|
| v1.3.14 | 2026-05-13 | Last Zig-based release. `Bun.Image` (built-in image processing), 7x faster warm installs via isolated linker, HTTP/2 + HTTP/3 clients (experimental), `Bun.serve()` HTTP/3, rewritten `fs.watch()`, `process.execve()`, `Bun.Terminal` on Windows via ConPTY, FreeBSD + Android builds, ~92 bug fixes |

### Nx compat status

- **Lockfile bug #16252**: Still open, 0 assignees, no PR. 7+ months without activity. The Rust rewrite may change the underlying implementation -- verify against v1.4.0 canary when it ships stable.
- **v1.4.0 canary**: Performance improvements, smaller binaries, Rust borrow-checker memory safety, 24/7 coverage-guided fuzzing for parsers. No Nx-specific compatibility changes confirmed yet.
- **Test orchestration** (1.3.13 `--parallel/--isolate/--shard/--changed`): Still the competitive vector to track.

**Risk level for Nx: MEDIUM -> MEDIUM-HIGH** (nudge up: Rust rewrite is a major architectural change; compat behavior may shift in v1.4.0; lockfile bug needs re-validation against canary).
