### Risks for Nx

- **Node 25 EOL today**: Any CI matrix pinned to Node 25 is now testing against an unsupported runtime. Drop it in favor of Node 24 LTS + Node 26 Current.
- **Node 26 `module.register()` runtime-deprecated (DEP0205)**: Plugin loaders using `module.register()` will emit runtime warnings on Node 26. Confirm Nx plugin hook chains don't trigger this.
- **Node 26 `_stream_*` legacy modules removed**: Grep Nx packages and dependencies for `require('_stream_readable')` etc. Likely already clean but must verify.
- **Node 26 `Temporal` integration**: `Stats.mtime` can now return a `Temporal.Instant` in Node 26+. Code that calls `.getTime()` or passes `mtime` to `Date` constructor may behave differently.
- **Node 20 users in production**: Now 32 days without upstream CVE patches. Nx should surface a deprecation warning.

### Opportunities

- **Node 26 `stream.compose` stable**: Replace any manually composed pipeline helpers.
- **Node 24.16 `randomUUIDv7()`**: Monotonic, time-sortable UUIDs. Relevant for Nx Cloud task IDs, cache keys where ordering matters.
- **`require(esm)` stable in Node 24.15+ and Node 26**: ESM plugin loading via `require()` is now standard. Simplifies Nx plugin host code.
- **Module compile cache stable (Node 24.15+)**: `node --module-cache` via API. Benchmark potential Nx CLI startup savings.

## TC39 Proposals

### Stage changes since May 2026 scan

_Source: tc39/proposals README, fetched 2026-06-01. May TC39 plenary results now reflected._

| Proposal | Prior Stage | Current Stage | Nx Impact |
|----------|-------------|---------------|-----------|
| Explicit Resource Management (`using`) | Stage 3 | **Stage 4 (likely)** | Now official ECMAScript. TypeScript already ships `using`/`await using`. Polyfill audit. |
| Atomics.pause | Stage 3 | **Stage 4 (likely)** | Worker thread performance -- low direct Nx impact |
| Joint Iteration | Stage 3 | **Stage 4 (likely)** | Iterating parallel iterables -- low direct Nx impact |
| Iterator Chunking | Stage 2.7 | **Stage 3** | Batching generator output -- affects Nx task streaming |
| Iterator Includes | Stage 2.7 | **Stage 3** | Minor -- replaces manual `.find()` patterns |
| Iterator Join | Stage 2.7 | **Stage 3** | Minor -- string join over iterables |
| Error Stack Accessor | Stage 2.7 | **Stage 3** | Standardized error stack access -- relevant to Nx diagnostics |
| RegExp Buffer Boundaries | new | **Stage 3** | Word boundary fix -- low impact |
| **Decorators** | Stage 3 | **Stage 2.7 (REGRESSION)** | Angular + NestJS generators depend on Decorator support assumptions. Investigate if this affects TypeScript's Decorator impl. |
| **Decorator Metadata** | Stage 3 | **Stage 2.7 (REGRESSION)** | Same concern as Decorators. |

**Note on "Stage 4 (likely)"**: These three proposals no longer appear in the Stage 3 list in tc39/proposals README. Standard interpretation: graduated to Stage 4 (shipped in ES specification). Could not independently verify plenary minutes -- treat as high-confidence but confirm when TC39 May 2026 notes are published.

**Note on Decorators demotion**: Stage 3 -> Stage 2.7 regression is highly unusual. Stage 2.7 was introduced to mean "all spec text is final, no open issues" -- moving back suggests a reopened design concern. If confirmed, TypeScript's Decorator implementation could be at risk of diverging from the final spec. Watch for TypeScript team response.

### Stage 3 landscape (June 2026)

Legacy RegExp features, Source Phase Imports, Dynamic Code Brand Checks, Deferring Module Evaluation (Import Defer), Non-extensible Applies to Private, Import Text, Iterator Chunking, Iterator Includes, Iterator Join, Error Stack Accessor, RegExp Buffer Boundaries.

### Stage 2.7 landscape

**Decorators** (REGRESSION from Stage 3), **Decorator Metadata** (REGRESSION from Stage 3), ShadowRealm, ESM Phase Imports, Immutable ArrayBuffers, Import Bytes, Await Dictionary.

### Key carryovers

- **Import Text (Stage 3)**: Still actionable for Nx generator template loading once V8 ships.
- **Import Defer (Stage 3)**: Deno 2.8.0 ships unstable support. First runtime to ship it. Reduces pressure on Nx to polyfill lazy-loaded plugins.
- **Type Annotations (Stage 1)**: No movement. Node `--strip-types` remains the pragmatic path.

## Deno

### Releases since 2026-05-25

| Version | Date | Key Changes |
|---------|------|-------------|
| **v2.8.1** | 2026-05-27 | TLS cert fixes, `node:util` additions (`getSystemErrorMap`, `transferableAbortSignal`), `fs.exists` fix, `fs.watch` error events, browser field map in `package.json` bundles, `sideEffects` field reading, BYONM `node_modules/.bin` walk, reverted compile cache companion |

### Highlights (v2.8.1)

- **`package.json` browser field map**: Deno's bundler now reads the `browser` field for package resolution. Affects how Nx-built libraries consumed by Deno tooling resolve browser vs node exports. Could expose dual-package issues in Nx libs.
- **BYONM `node_modules/.bin` walk**: Better binary resolution in bring-your-own-node-modules mode. Nx workspaces using Deno in BYONM mode should see improved CLI tool resolution.
- **Reverted compile cache companion (`module.enableCompileCache`)**: The Node.js polyfill for this was reverted in 2.8.1, suggesting incompatibilities. Module compile cache is not reliably available in Deno.
- **Security: resolved IPs against net deny list for fetch/WebSocket**: Relevant for Nx Cloud connectivity in sandboxed Deno environments.

### Nx compat status

- **`catalog:` protocol** (2.8.0): Deno workspace dep centralization now mirrors pnpm catalogs. Competitive signal -- Nx workspace dep management has a new peer in Deno.
- **Hoisted node_modules** (2.8.0): Deno workspaces can now use hoisted linking. Makes Deno more viable for monorepos.
- **Node compat**: Continued improvement. `node:http`, TLS, `node:util`, `node:repl` all tightened in 2.8.x.
- **`deno pack`/`deno ci`/`deno bump-version`** (2.8.0): Deno is building out a full dev-workflow CLI. Still less capable than Nx for large monorepos but gap is narrowing.

**Risk level for Nx: LOW -> LOW-MEDIUM** (nudge up: Deno workspace + catalog protocol + hoisted linker = a legitimately viable Nx alternative for small-to-mid Deno-first monorepos as of 2.8.x).
