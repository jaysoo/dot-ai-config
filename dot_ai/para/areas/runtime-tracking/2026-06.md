### Risks for Nx

- **Node 25 EOL (June 1)**: CI matrix pinned to Node 25 tests against unsupported runtime. Drop in favor of Node 24 LTS + Node 26 Current.
- **Node 26.3.0 `Buffer.poolSize` x4**: Default pool up from 16KB to 64KB. Memory-sensitive paths in Nx Cloud servers or CLI task runners that rely on small allocations coalescing may see changed behavior. Benchmark on Node 26.
- **Node 26 `module.register()` runtime-deprecated (DEP0205)**: Plugin loaders using `module.register()` emit runtime warnings on Node 26. Confirm Nx plugin hook chains are unaffected.
- **Node 26 `_stream_*` legacy modules removed**: Grep Nx packages and deps for `require('_stream_readable')` etc.
- **Node 26 `Temporal` integration**: `Stats.mtime` can return `Temporal.Instant` in Node 26+. Code calling `.getTime()` or passing `mtime` to `Date` constructor may break.
- **macOS Universal Binary concern**: Node 26.3.0 noted Apple is "progressively dropping Intel support." Node may not maintain universal binaries for Node 26's full lifetime. Affects Nx CI on Intel Mac runners long-term.
- **Node 20 users in production**: 39 days without upstream CVE patches. Nx should surface a deprecation warning.

### Opportunities

- **Node 26 `stream.compose` stable**: Replace any manually composed pipeline helpers.
- **Node 24.16 `randomUUIDv7()`**: Monotonic, time-sortable UUIDs for Nx Cloud task IDs / cache keys.
- **`require(esm)` stable in Node 24.15+ and Node 26**: Simplifies Nx plugin host code.
- **Module compile cache stable (Node 24.15+)**: Benchmark potential Nx CLI startup savings.
- **Node 26.3.0 `permission.drop()`**: Could be used in Nx sandboxing or security-isolation features if Nx ever exposes permission management.

No polyfill work needed; check if Nx generator templates should use `using`.

### Key carryovers

- **Import Defer (Stage 3)**: Deno 2.8.0 ships unstable support. Reduces pressure on Nx to polyfill lazy-loaded plugins.
- **Type Annotations (Stage 1)**: No movement. Node `--strip-types` remains the pragmatic path.
