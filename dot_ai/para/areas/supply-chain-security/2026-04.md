## Major Posture Discovery This Week

**Ocean rolled out npm trusted publishing on 2026-04-08** (commit `91c620a30`, PR #10687). Prior reports flagged the Ocean publish flow as opaque; verifying live registry data this week confirms:

| Nrwl-owned package | Publisher identity | OIDC config | Provenance |
|--------------------|-------------------|-------------|:----------:|
| `nx@22.7.0` | GitHub Actions (OIDC) | nrwl/nx | YES (SLSA v1) |
| `create-nx-workspace@22.7.0` | GitHub Actions (OIDC) | nrwl/nx | YES |
| `create-nx-plugin@22.7.0` | GitHub Actions (OIDC) | nrwl/nx | YES |
| `nx-mcp@0.24.0` | GitHub Actions (OIDC) | nrwl/nx | YES |
| `nx-cloud@19.1.3` | GitHub Actions (OIDC) | **nrwl/ocean** | **NO** |
| `@nrwl/nx-cloud@19.1.3` | GitHub Actions (OIDC) | nrwl/ocean | NO |
| `polygraph@0.0.7` | GitHub Actions (OIDC) | nrwl/ocean | NO |
| `polygraph-cli@0.0.6` | GitHub Actions (OIDC) | nrwl/ocean | NO |
| `polygraph-mcp@0.1.9` | GitHub Actions (OIDC) | nrwl/ocean | NO |

**Implication:** No long-lived npm tokens are in use anywhere in the Nrwl publishing surface. Even Ocean-published packages now flow through GitHub Actions OIDC + a named environment with an actor allowlist. The token-theft attack class (e.g. the August 2025 `nx@20.x` malicious-publish incident pattern) is structurally blocked across both repos.

**Remaining gap:** Ocean's `releasePublish()` invocation in `tools/scripts/nx-release/npm-packages-publish.ts` does not pass `provenance: true`, and the workflow does not export `NPM_CONFIG_PROVENANCE=true`. Result: trusted-publishing succeeds but no SLSA attestation is generated. One-line fix.

**Polygraph-MCP correction:** Last week's HIGH bus-factor finding was wrong. `maxkless` is listed as a maintainer on the package, but the **publishing identity** is the Ocean GitHub Actions OIDC trusted publisher. Compromising Maxkless's npm credentials does not enable a malicious publish — only triggering the workflow from Ocean main with an allowlisted actor does. Downgrading to LOW (consistency: still no provenance, like all Ocean-published packages).

Notable security comment in workflow (verbatim from `release-nx-cloud-packages.yml`):

> SECURITY: This actor check is tamper-proof because rulesets prevent pushing to main, and workflow_dispatch runs the workflow file from the ref it's triggered on. An agent triggering this on ref:main gets the unmodified file with this check. If they trigger on their own branch (to bypass this check), the environment branch restriction blocks it. If they also remove the environment: key, the OIDC token's subject claim will no longer include "environment:npm-publish", and npm will reject the publish because the trusted publisher config on npmjs.com requires that environment.

This is a meaningfully strong layered defense — branch-protection ruleset + environment binding + OIDC subject claim verification at registry side.
