Notes:
  • AI agent flag (aiAgent:true) shipped Feb 2026 — pre-Feb AI volume = 0.
  • Cloud opt-in metric definition (chart): Strict nxCloudArg == "yes", matching the goal table. Excludes github/gitlab/etc CI provider selections — those came from leading prompts pre-22.5.3 and inflate the rate without representing user intent. Pre-Dec 2025 (CSV era) uses doc-level useCloud as a fallback since nxCloudArg wasn't emitted; those numbers aren't directly comparable to the modern target.
  • Dec 2025 → Apr 2026 trajectory: Direct prompt era. Dec/Jan still had older single-step "yes/skip" prompts (no Never option) → 23-25% yes. Late Feb introduced 22.5.3's direct yes/skip/never → rate dropped to 8-12%. Mar 8.6% Human-only matches the project update on Mar 30 ("9-10% in 22.6.x"). Apr 11.1% Human-only matches the Apr 8 update ("CNW rate is 10.2%").
  • The CI-provider channel collapse: Pre-22.5.3 (through Feb 2026), the CLI's first prompt was "What CI provider do you use?" — picking github/gitlab/etc implicitly enabled Cloud. 22.5.3 (late Feb) replaced this with a direct yes/skip/never prompt. 22.5.4 briefly re-tried the leading CI prompt (enable-caching2) and got 2,508 picks before being removed again. Net effect: total cloud opt-ins per day dropped from ~800 (Nov 2025) to ~65 (Apr 2026). The strict-yes rate chart shows the optimization wins on the new prompt; the by-source chart shows the absolute funnel loss the redesign caused.
  • Reading the two cloud charts together: Strict-yes rate (line chart) tracks our goal metric and shows incremental A/B wins (8% → 14.7%). The by-source chart (stacked bars) shows that the absolute funnel got smaller because the implicit-opt-in CI channel was removed. Even at 10% true activation rate, the pre-22.5.3 CI channel produced more activated workspaces than today's strict-yes channel. Open question: is the trade-off (explicit consent vs absolute volume) worth it? Needs Cloud activation data to answer.
  • CI + Contentful overlap (Contentful runs are largely automated CI builds), so stacks are not strictly additive.

  • Init error rate only observable since 22.6.4 (Apr 2026) when init started emitting JSON funnel events.
